# Chapter 20: 强化学习

## Core Idea
**强化学习 (RL)** 是通向通用人工智能的可能路径: 当"最佳输出未知"或"标注数据难收集"时(如下围棋),让**智能体 (agent)** 通过与环境互动获取**奖励 (reward)**,学习最大化**累积奖励**。RL 与监督学习的区别: 监督学习有人给标准答案,RL 只有奖励信号。**策略梯度算法**是基础: 用神经网络(策略网络)输出动作概率,采样执行;通过评价动作好坏(即时/累积/折扣累积/减基线四种 A 值)定义损失,梯度更新。**Actor-Critic** 引入价值网络评估状态价值,**优势 Actor-Critic** 用 `A = r + γV(s') - V(s)` 替代采样。RL 训练极耗时因为**同策略学习**每更新一次参数就要重新收集数据。

## Frameworks Introduced

- **RL 三步骤 (LeeDL)**
  - When to use: 理解 RL 与 ML 的对应
  - How:
    1. **未知函数 = 智能体**(策略网络 π_θ): 输入观测 s,输出动作 a 的概率分布
    2. **定义损失**: 负的回报 `L = -R(τ)`,τ 是轨迹 {s₁,a₁,s₂,a₂,...}
    3. **优化**: 梯度上升最大化回报(但环境/奖励是黑盒,不能用普通梯度下降)
  - 关键: 与监督学习结构相同,但优化更复杂

- **策略网络 (Policy Network) (LeeDL)**
  - When to use: RL 的"函数"
  - How:
    - 输入: 游戏画面(像素)
    - 输出: 每个动作的分数(经 softmax 概率)
    - 采样: 按概率随机选动作(不取最大,因随机性重要)
    - 架构: 图片用 CNN,需要记忆用 RNN/Transformer
  - 关键: 输出是分布,采样执行--石头剪刀布不能总出石头

- **回报 (Return) 定义 (LeeDL)**
  - When to use: 定义优化目标
  - How:
    - **奖励 r_t**: 单步反馈
    - **回报 R(τ) = Σ r_t**: 整场游戏累积奖励
    - **轨迹 τ = {s₁,a₁,...,s_T,a_T}**: 一场游戏序列
  - 关键: RL 最大化回报(不是最小化损失)

- **评价动作 A 的四个版本 (LeeDL)**
  - When to use: 评估每个动作好坏(策略梯度的核心)
  - How:
    - **版本 0 - 即时奖励**: `A_t = r_t` -- 短视,忽略延迟奖励(左右移动无奖但必要)
    - **版本 1 - 累积奖励**: `G_t = Σᵢ₌ₜᴺ r_i` -- 解决延迟,但游戏太长时 r_N 归功于 a_1 不合理
    - **版本 2 - 折扣累积**: `G'_t = Σγ^(i-t) r_i`,γ<1(如 0.9)-- 远期奖励权重小
    - **版本 3 - 减基线**: `A_t = G'_t - b` -- 解决全正值问题,让 A 有正有负
  - 关键: 好坏是相对的,减基线让"差动作"得负评价

- **策略梯度算法 (Policy Gradient) (LeeDL)**
  - When to use: 训练策略网络
  - How:
    ```
    初始化 θ
    for i = 1 to T:
        用 π_θ 与环境互动,收集 {(s_n, a_n)}
        计算 A_1, ..., A_N
        定义损失 L = Σ A_n · e_n (e_n 是交叉熵)
        θ ← θ - η ∇L
    ```
  - 关键: A_n 是动作好坏权重,正 A 鼓励该动作,负 A 抑制
  - **致命弱点**: **同策略 (on-policy)**--每次更新参数后旧数据失效,要重新收集

- **同策略 vs 异策略 (LeeDL)**
  - When to use: 决定数据收集策略
  - How:
    - **同策略 (on-policy)**: 训练的 agent = 互动的 agent(策略梯度)。每次更新都要新数据
    - **异策略 (off-policy)**: 训练的 agent ≠ 互动的 agent。互动 agent 收集一次数据,训练 agent 反复用
  - 关键: 同策略的概念--同一动作对不同 agent 好坏不同(进藤光 vs 佐为对小马步飞的判断)

- **探索 (Exploration) (LeeDL)**
  - When to use: 确保动作空间充分探索
  - How:
    - 增大输出分布熵,让低概率动作也有机会
    - 在参数上加噪声
  - 关键: 没执行过的动作好坏未知,要主动探索

- **Actor-Critic (LeeDL)**
  - When to use: 引入价值网络辅助训练
  - How:
    - **Actor (策略网络)**: 输入 s,输出动作概率
    - **Critic (价值网络 V_π(s))**: 输入 s,输出期望折扣累积奖励
    - 训练 Critic:
      - **蒙特卡洛 (MC)**: 玩完整场,`V(s_a) ≈ G'_a`
      - **时序差分 (TD)**: 用 `V(s_t) = r_t + γV(s_{t+1})` 关系,无需玩完
    - 用 `V_π(s)` 作为基线 b,`A = G' - V_π(s)`
  - 关键: MC 与 TD 假设不同--TD 假设 s_a 与 s_b 独立,MC 不假设

- **优势 Actor-Critic (Advantage Actor-Critic) (LeeDL)**
  - When to use: 改进 Actor-Critic,用期望替代采样
  - How:
    - 用 `r_t + γV(s_{t+1})` 替代采样 G'_t(用期望替代单次采样)
    - `A_t = r_t + γV(s_{t+1}) - V(s_t)`
    - 若 A_t > 0: a_t 比平均采样好
    - 若 A_t < 0: a_t 比平均采样差
  - 关键: 平均减平均更稳定;Actor 和 Critic 可共用前几层(输入相同)

- **DQN 与变体 (LeeDL)**
  - When to use: 直接用 Critic 决定动作
  - How: DQN 有 7 种改进,合称 Rainbow:
    - Double DQN、Dueling DQN、Prioritized Replay、Multi-step、Distributional、Noisy Nets、Rainbow
  - 关键: 不用 Actor,直接基于 Q 值选动作

## Key Concepts

- **强化学习 (RL)**: 通过互动学决策
- **智能体 (agent)**: 决策者
- **环境 (environment)**: 互动对象
- **观测 (observation) s**: 智能体的输入
- **动作 (action) a**: 智能体的输出
- **奖励 (reward) r**: 单步反馈
- **回报 (return) R**: 累积奖励
- **轨迹 (trajectory) τ**: 状态-动作序列
- **回合 (episode)**: 一场游戏
- **策略网络 (policy network) π_θ**: 输入 s 输出动作概率
- **价值函数 V_π(s)**: 状态 s 的期望回报
- **折扣因子 γ**: 远期奖励折扣
- **基线 (baseline) b**: 评价的参考值
- **优势 (advantage) A**: 动作相对好坏
- **策略梯度 (policy gradient)**: 基于 A 的梯度算法
- **同策略 (on-policy)**: 训练 = 互动 agent
- **异策略 (off-policy)**: 训练 ≠ 互动 agent
- **Actor-Critic**: 策略 + 价值双网络
- **蒙特卡洛 (MC)**: 玩完整场估计价值
- **时序差分 (TD)**: 单步差分估计价值
- **DQN**: 深度 Q 网络
- **Rainbow**: DQN 7 改进合集
- **稀疏奖励 (sparse reward)**: 奖励稀少(如围棋)
- **延迟奖励 (delayed reward)**: 牺牲短期换长期
- **探索 (exploration)**: 主动尝试新动作
- **模仿学习 (imitation learning)**: 从专家演示学

## Mental Models

- **RL = 没有 label 的学习**: 只有奖励信号,最佳动作未知
- **Agent = 函数**: 输入观测,输出动作
- **回报最大化 = 损失最小化的反面**: `L = -R`
- **版本 0 短视**: 左右移动无即时奖励但必要--要考虑累积
- **折扣 = 远期打折**: γ^N 让远期奖励几乎无影响
- **基线 = 让好坏相对**: 全正值时无法识别差动作,减基线让差动作得负
- **同策略 = 数据即用即弃**: 旧 agent 的经验对新 agent 不适用
- **进藤光 vs 佐为**: 同一动作对不同水平 agent 好坏不同
- **Actor-Critic = 演员 + 评论员**: Actor 表演,Critic 评分
- **MC vs TD = 完整 vs 单步**: MC 准但慢,TD 快但假设状态独立
- **优势 A = 比平均好多少**: 正 = 鼓励,负 = 抑制

## Anti-patterns

- **版本 0 用即时奖励**: 智能体只开火不移动。**用累积或折扣累积**
- **游戏太长用版本 1**: r_N 归功于 a_1 不合理。**用版本 2 折扣**
- **不减基线**: 全正值,差动作也被鼓励。**减 V(s) 或均值**
- **同策略用旧数据训新 agent**: 数据失效。**重新收集或用异策略**
- **不探索**: 某些动作好坏永不知。**加大熵或加噪声**
- **MC 用于超长游戏**: 玩不完。**用 TD**
- **Actor 与 Critic 不共用前层**: 浪费计算。**共享特征提取层**
- **围棋只用最终奖励训**: 稀疏奖励难训。**加启发式奖励或价值网络**
- **取最大动作而非采样**: 失去随机性,被对手预测。**按概率采样**

## Code Examples

**策略网络 + 策略梯度训练**:

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class PolicyNetwork(nn.Module):
    def __init__(self, obs_dim, n_actions, hidden=128):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(obs_dim, hidden), nn.ReLU(),
            nn.Linear(hidden, n_actions)
        )
    def forward(self, s):
        return F.softmax(self.net(s), dim=-1)  # 动作概率

policy = PolicyNetwork(obs_dim=4, n_actions=3)
optimizer = torch.optim.Adam(policy.parameters(), lr=1e-2)

def collect_episode(policy, env, max_steps=200):
    """收集一个 episode 的轨迹"""
    states, actions, rewards = [], [], []
    s = env.reset()
    for _ in range(max_steps):
        with torch.no_grad():
            probs = policy(torch.tensor(s, dtype=torch.float32))
        a = torch.multinomial(probs, 1).item()  # 按概率采样
        s_next, r, done, _ = env.step(a)
        states.append(s); actions.append(a); rewards.append(r)
        s = s_next
        if done: break
    return states, actions, rewards

def compute_discounted_returns(rewards, gamma=0.99):
    """计算 G'_t = Σ γ^(i-t) r_i"""
    G = []
    running = 0
    for r in reversed(rewards):
        running = r + gamma * running
        G.insert(0, running)
    return G

# 训练循环
for iteration in range(400):
    states, actions, rewards = collect_episode(policy, env)
    returns = compute_discounted_returns(rewards)
    
    # 减基线(均值)
    returns = torch.tensor(returns)
    returns = (returns - returns.mean()) / (returns.std() + 1e-8)
    
    # 计算损失 L = Σ A_n · e_n
    s_batch = torch.tensor(states, dtype=torch.float32)
    a_batch = torch.tensor(actions)
    probs = policy(s_batch)
    log_probs = torch.log(probs.gather(1, a_batch.unsqueeze(1)))
    loss = -(log_probs * returns).sum()  # 负号: 最大化回报
    
    optimizer.zero_grad(); loss.backward(); optimizer.step()
```
- **What it demonstrates**: 策略梯度 = 收集 episode -> 算 G' -> 减基线 -> 加权交叉熵 -> 梯度更新

**Actor-Critic 训练**:

```python
class ActorCritic(nn.Module):
    def __init__(self, obs_dim, n_actions, hidden=128):
        super().__init__()
        self.shared = nn.Sequential(  # 共享特征层
            nn.Linear(obs_dim, hidden), nn.ReLU()
        )
        self.actor = nn.Linear(hidden, n_actions)
        self.critic = nn.Linear(hidden, 1)
    
    def forward(self, s):
        feat = self.shared(s)
        return F.softmax(self.actor(feat), dim=-1), self.critic(feat)

ac = ActorCritic(obs_dim=4, n_actions=3)
optimizer = torch.optim.Adam(ac.parameters(), lr=1e-3)
gamma = 0.99

for iteration in range(1000):
    states, actions, rewards = collect_episode(ac, env)
    
    s_batch = torch.tensor(states, dtype=torch.float32)
    a_batch = torch.tensor(actions)
    r_batch = torch.tensor(rewards, dtype=torch.float32)
    
    probs, values = ac(s_batch)
    values = values.squeeze(-1)
    
    # 计算 TD 目标: r_t + γ V(s_{t+1})
    _, next_values = ac(s_batch[1:].detach()) if len(s_batch) > 1 else (None, torch.zeros_like(values[1:]))
    td_target = r_batch[:-1] + gamma * next_values
    td_target = torch.cat([td_target, r_batch[-1:]])  # 最后一步无 next
    
    # Critic 损失: TD 误差
    critic_loss = F.mse_loss(values[:-1], td_target.detach())
    
    # Actor 损失: 优势 A = r + γV(s') - V(s)
    advantage = (r_batch[:-1] + gamma * values[1:].detach() - values[:-1])
    log_probs = torch.log(probs[:-1].gather(1, a_batch[:-1].unsqueeze(1)))
    actor_loss = -(log_probs * advantage.detach()).mean()
    
    loss = actor_loss + critic_loss
    optimizer.zero_grad(); loss.backward(); optimizer.step()
```

**TD 训练 Critic(简化)**:

```python
def train_critic_td(critic, transitions, gamma=0.99):
    """transitions: [(s_t, a_t, r_t, s_{t+1}), ...]"""
    for s_t, a_t, r_t, s_next in transitions:
        v_t = critic(s_t)
        v_next = critic(s_next).detach()  # 不反传
        td_target = r_t + gamma * v_next
        td_error = v_t - td_target
        loss = td_error.pow(2)
        # 反传更新 critic
```

## Reference Tables

**评价动作 A 的版本对比**:

| 版本 | 公式 | 优点 | 缺点 |
|---|---|---|---|
| 0 即时 | A = r_t | 简单 | 短视,忽略延迟 |
| 1 累积 | G_t = Σr_i | 考虑延迟 | 长游戏归因不准 |
| 2 折扣 | G'_t = Σγ^(i-t) r_i | 远期权重小 | 全正值 |
| 3 减基线 | G' - b | 有正有负 | b 难选 |
| 4 优势 AC | r + γV(s') - V(s) | 期望替代采样 | 需训 Critic |

**MC vs TD 对比**:

| 维度 | 蒙特卡洛 (MC) | 时序差分 (TD) |
|---|---|---|
| 数据 | 玩完整场 | 单步 {(s_t, a_t, r_t, s_{t+1})} |
| 估计 | V(s_a) = G'_a | V(s_t) = r_t + γV(s_{t+1}) |
| 假设 | 直接观察 | s_t 与 s_{t+1} 独立 |
| 速度 | 慢(玩完整场) | 快(单步即可) |
| 适用 | 短游戏 | 长/无限游戏 |
| 方差 | 高(整场波动) | 低(单步) |
| 偏差 | 无偏 | 有偏(假设) |

**同策略 vs 异策略**:

| 维度 | 同策略 (on-policy) | 异策略 (off-policy) |
|---|---|---|
| 互动 agent | = 训练 agent | ≠ 训练 agent |
| 数据复用 | 否(更新后失效) | 是(反复用) |
| 数据收集次数 | 每次更新都收集 | 一次收集多次用 |
| 效率 | 低 | 高 |
| 代表 | 策略梯度、A2C | DQN、PPO(部分) |

**RL 关键算法**:

| 算法 | 类型 | 关键创新 |
|---|---|---|
| 策略梯度 | on-policy | 基于 A 加权交叉熵 |
| Actor-Critic | on-policy | 引入 Critic 估值 |
| 优势 Actor-Critic (A2C) | on-policy | A = r + γV(s') - V(s) |
| DQN | off-policy | Q 学习 + 经验回放 |
| Rainbow | off-policy | DQN 7 改进合集 |
| PPO | on-policy | 限制策略更新幅度 |

## Worked Example

**为什么左右移动要奖励?** -- 太空侵略者案例。

**情景**: 太空侵略者游戏,动作: 左移、右移、开火。
- 开火击中外星人: r = +5
- 左移/右移: r = 0(无即时奖励)

**用版本 0(即时奖励)训练**:
- 智能体学到: 开火得好分,左右移无分
- 结果: **一直开火**,从不瞄准
- 问题: 不瞄准的开火打不中外星人,但版本 0 看不到长期影响

**用版本 1(累积奖励)训练**:
- 一次轨迹: 左移(r=0) -> 左移(r=0) -> 开火(r=+5)
- A_1(左移) = G_1 = 0+0+5 = 5
- A_2(左移) = G_2 = 0+5 = 5
- A_3(开火) = G_3 = 5
- **左移也得正评价**--因为是开火前必要动作

**用版本 2(折扣累积)训练**:
- A_1 = 0 + 0.9·0 + 0.9²·5 = 4.05
- A_3 = 5
- 远期奖励(左移后的开火)对左移评价有贡献,但权重小于直接开火

**用版本 3(减基线)训练**:
- 假设平均回报 b = 3
- A_1 = 4.05 - 3 = 1.05(正,鼓励左移)
- 若某动作 G' = 2, A = -1(负,抑制)
- **好坏相对**: 即使全正值,差动作也得负评价

**MC vs TD 计算 V(s_a)**:
- 8 次游戏观察: s_a 后接 s_b,r=0; s_b 单独 6 次 r=1,2 次 r=0
- MC: V(s_a) = 0(只观察一次,s_a 后得 0)
- TD: V(s_a) = V(s_b) + 0 = 3/4 + 0 = 3/4
- **假设差异**: MC 看 s_a 后的实际结果;TD 假设 s_b 的奖励独立于 s_a

**关键洞察**:
1. **延迟奖励**: 左右移动的"价值"体现在后续开火,版本 0 看不到
2. **折扣解决长游戏**: γ 让远期奖励权重递减,避免 r_N 归因 a_1
3. **基线让好坏相对**: 全正值时减基线才能识别差动作
4. **同策略数据失效**: 旧 agent 不开火,它的经验对新 agent(已学会开火)无意义
5. **进藤光 vs 佐为**: 小马步飞对弱者是好的(不易错),对强者是差的(限制发展)
6. **TD 比 MC 快**: TD 单步可训,MC 要玩完整场
7. **Actor-Critic 共用前层**: 输入相同(游戏画面),特征提取可共享

## Key Takeaways

1. RL 适合"最佳输出未知"任务(下围棋、玩游戏),通过互动最大化累积奖励
2. RL 三步骤与 ML 相同: 函数(策略网络)、损失(负回报)、优化(策略梯度)
3. **策略网络**输出动作概率,**按概率采样**(不取最大,保留随机性)
4. 评价动作 A 的四个版本: 即时(短视)、累积、折扣累积、减基线
5. **折扣因子 γ**让远期奖励权重递减,适合长游戏
6. **减基线**让好坏相对,解决全正值问题
7. **策略梯度** = 加权交叉熵,A_n 是权重
8. **同策略学习**致命弱点: 每次更新参数后旧数据失效,需重新收集(极耗时)
9. **Actor-Critic** 引入 Critic(价值网络),用 V(s) 作基线
10. MC vs TD: MC 玩完整场准但慢,TD 单步快但假设状态独立
11. **优势 Actor-Critic** 用 `A = r + γV(s') - V(s)`,期望替代采样更稳定
12. 探索重要: 没执行过的动作好坏未知,要加大熵或加噪声
