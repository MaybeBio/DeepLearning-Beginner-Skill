# Chapter 16: 扩散模型

## Core Idea
**扩散模型 (DDPM)** 借鉴物理热力学: 把生成图片的过程拆成"加噪"(前向扩散) 和"逐步去噪"(逆向生成) 两阶段。**前向过程**给真实图逐步加高斯噪声直到变成纯噪声(无需训练);**逆向过程**训练一个**噪声预测器**,从纯噪声开始反复去噪,最终生成清晰图。**关键洞察**: 不直接预测"去噪后的图",而是预测"这幅图里加了什么噪声",再扣掉--预测噪声比预测图更易学。文生图加文字条件即可。Stable Diffusion/DALL-E/Midjourney 都基于此。

## Frameworks Introduced

- **前向扩散过程 (Forward Process) (LeeDL)**
  - When to use: 制造训练数据
  - How: 从真实图 x_0 开始,逐步加高斯噪声 ε_t:`x_t = √α_t · x_{t-1} + √(1-α_t) · ε_t`。重复 T 步(如 1000 步),最后 x_T 是纯高斯噪声
  - 关键: 不需要训练,纯加噪;每步噪声 ε_t 就是训练标签

- **逆向去噪过程 (Reverse Process) (LeeDL)**
  - When to use: 生成图片
  - How: 从 x_T ~ N(0, I) 开始,反复用噪声预测器:`x_{t-1} = denoise(x_t, t)`,T 步后得 x_0 是生成图
  - 关键: 每步都用**同一个**噪声预测器,但输入步数 ID t 告诉它当前噪声程度

- **噪声预测器 (Noise Predictor) (LeeDL)**
  - When to use: 扩散模型的核心组件
  - How:
    - 输入: (噪声图 x_t, 步数 ID t)
    - 输出: 预测的噪声 ε_θ(x_t, t)
    - 去噪: `x_{t-1} = x_t - ε_θ(x_t, t)` (简化版)
    - 损失: `||ε - ε_θ(x_t, t)||²`(MSE)
  - **为什么不直接预测去噪图**: 预测带噪声的猫 ≈ 已学会画猫;预测"噪声长啥样"更简单

- **训练数据生成 (LeeDL)**
  - When to use: 准备噪声预测器的训练数据
  - How:
    - 从数据集采一张图 x_0
    - 随机采高斯噪声 ε
    - 用前向公式算 x_t = √ᾱ_t · x_0 + √(1-ᾱ_t) · ε
    - 训练样本: 输入 (x_t, t),标签 ε
  - 关键: 自监督,无需人工标注;标签就是加的噪声

- **文生图扩散 (LeeDL)**
  - When to use: 文字描述生成图片(Stable Diffusion/DALL-E)
  - How:
    - 噪声预测器多输入文字条件 c: `ε_θ(x_t, t, c)`
    - 训练数据: (图片, 文字描述) 成对(LAION 58.5 亿对、ImageNet 100 万)
    - 去噪每步都条件于文字
  - 关键: 文字是额外输入,引导去噪方向

- **Stable Diffusion (潜在扩散) (LeeDL)**
  - When to use: 减少计算量(像素空间扩散太慢)
  - How: **不在像素空间扩散,而在 VAE 的潜在空间扩散**。先用 VAE 编码器把图压成低维潜在向量,在潜在空间做扩散,最后用 VAE 解码器还原成图。计算量降低数十倍

- **classifier-free guidance (CFG) (LeeDL 推断)**
  - When to use: 增强文字条件的控制力
  - How: 推理时混有条件和无条件预测:`ε = ε_uncond + w·(ε_cond - ε_uncond)`,w 是引导强度(典型 7.5)。w 大则更忠于文字但多样性下降

## Key Concepts

- **扩散模型 (Diffusion Model)**: 加噪 + 去噪的生成模型
- **DDPM (Denoising Diffusion Probabilistic Model)**: 主流扩散模型
- **前向过程 (forward process)**: 给图逐步加噪
- **逆向过程 (reverse process)**: 从噪声逐步生成图
- **噪声预测器 (noise predictor)**: 预测图中噪声的网络
- **步数 ID (step t)**: 告诉网络当前噪声程度
- **噪声调度 (noise schedule)**: α_t 的序列,决定每步加噪强度
- **潜在扩散 (latent diffusion)**: 在 VAE 潜在空间做扩散
- **VAE (Variational Autoencoder)**: 编码-解码器,用于潜在扩散
- **U-Net**: 扩散模型常用 backbone(图像去噪效果好)
- **文字条件 (text condition)**: 文生图的引导信号
- **classifier-free guidance (CFG)**: 条件 vs 无条件混合的引导技巧
- **LAION**: 大规模图文对数据集(58.5 亿对)
- **CLIP**: 文图对比学习模型,常用于评估扩散模型
- **DALL-E/Stable Diffusion/Midjourney/Imagen**: 商业扩散模型代表

## Mental Models

- **扩散 = 米开朗基罗雕塑**: "塑像就在石头里,我只是把不需要的部分去掉" -- 图就在噪声里,模型逐步去噪显形
- **预测噪声比预测图简单**: 学"猫加噪声后什么样" = 已学会画猫;直接学"猫长什么样"更难
- **步数 ID = 噪声严重度**: t=1000 几乎纯噪声,t=1 几乎是清晰图;同一网络用 ID 区分场景
- **训练 = 自监督**: 标签(噪声)由加噪过程自动生成,无需人工标注
- **文生图 = 加文字输入**: 噪声预测器多一个文字条件,引导去噪方向
- **潜在扩散 = 压缩空间扩散**: 256×256 图压成 32×32 潜在向量,计算量降低 64 倍
- **CFG = "听 vs 不听文字" 差异放大**: 大 w 时模型更"听话",小 w 时更自由

## Anti-patterns

- **像素空间直接做扩散**: 计算量爆炸。**用潜在扩散(Stable Diffusion)**
- **去噪步数太少**: 生成质量差。**典型 1000 步**(推理可减少到 20-50 步用 DDIM)
- **噪声预测器不输入步数 t**: 无法区分不同噪声程度。**必须输入 t**(常用正弦嵌入)
- **文生图不用条件**: 生成图与文字无关。**必须文字 + CFG**
- **CFG 强度过高**: 多样性丧失,生成图过拟合文字。**典型 7.5**
- **训练数据图文不匹配**: 模型学到错乱关联。**用高质量 LAION 等数据集**
- **加噪调度不合理**: 太快训不出,太慢推理慢。**用 cosine 或 linear 调度**
- **不用 U-Net**: 去噪效果差。**U-Net 是扩散 backbone 标准**

## Code Examples

**前向扩散(加噪)**:

```python
import torch

def linear_beta_schedule(timesteps, beta_start=1e-4, beta_end=0.02):
    return torch.linspace(beta_start, beta_end, timesteps)

T = 1000
betas = linear_beta_schedule(T)
alphas = 1. - betas
alphas_cumprod = torch.cumprod(alphas, axis=0)  # ᾱ_t

def q_sample(x_0, t, noise=None):
    """前向加噪: x_0 -> x_t
    用闭式解直接算 x_t, 不必逐步加噪
    """
    if noise is None:
        noise = torch.randn_like(x_0)
    sqrt_alpha = torch.sqrt(alphas_cumprod[t])[:, None, None, None]
    sqrt_one_minus = torch.sqrt(1. - alphas_cumprod[t])[:, None, None, None]
    return sqrt_alpha * x_0 + sqrt_one_minus * noise  # x_t
```

**噪声预测器 (U-Net 简化版)**:

```python
import torch.nn as nn
import math

class SinusoidalPositionEmbedding(nn.Module):
    """把步数 t 编码成向量(类似 Transformer 位置编码)"""
    def __init__(self, dim):
        super().__init__()
        self.dim = dim
    def forward(self, t):
        half = self.dim // 2
        emb = math.log(10000) / (half - 1)
        emb = torch.exp(torch.arange(half, device=t.device) * -emb)
        emb = t[:, None] * emb[None, :]
        return torch.cat([emb.sin(), emb.cos()], dim=-1)

class NoisePredictor(nn.Module):
    """简化版噪声预测器 (实际用 U-Net)"""
    def __init__(self, in_ch=3, hidden=128):
        super().__init__()
        self.time_emb = SinusoidalPositionEmbedding(hidden)
        self.time_mlp = nn.Sequential(
            nn.Linear(hidden, hidden), nn.SiLU(),
            nn.Linear(hidden, hidden)
        )
        # 简化: 实际用 U-Net (下采样-瓶颈-上采样 + skip)
        self.net = nn.Sequential(
            nn.Conv2d(in_ch + hidden, hidden, 3, padding=1), nn.SiLU(),
            nn.Conv2d(hidden, hidden, 3, padding=1), nn.SiLU(),
            nn.Conv2d(hidden, in_ch, 3, padding=1)
        )
    
    def forward(self, x_t, t):
        # x_t: (B, C, H, W), t: (B,)
        t_emb = self.time_mlp(self.time_emb(t))  # (B, hidden)
        t_emb = t_emb[:, :, None, None].expand(-1, -1, *x_t.shape[2:])
        return self.net(torch.cat([x_t, t_emb], dim=1))  # 预测噪声 ε_θ
```

**训练循环**:

```python
model = NoisePredictor()
optimizer = torch.optim.AdamW(model.parameters(), lr=1e-4)

for epoch in range(num_epochs):
    for x_0, _ in dataloader:  # x_0: 真实图 (B, C, H, W)
        batch = x_0.size(0)
        # 1. 随机采样步数 t
        t = torch.randint(0, T, (batch,))
        # 2. 随机采样噪声 ε
        noise = torch.randn_like(x_0)
        # 3. 前向加噪得 x_t
        x_t = q_sample(x_0, t, noise)
        # 4. 预测噪声
        noise_pred = model(x_t, t)
        # 5. MSE 损失
        loss = nn.functional.mse_loss(noise_pred, noise)
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
```
- **What it demonstrates**: 扩散训练 = 自监督(标签是加的噪声),MSE 损失极简

**采样(生成)**:

```python
@torch.no_grad()
def sample(model, shape, T=1000):
    """从纯噪声开始,逐步去噪生成图"""
    x = torch.randn(shape)  # x_T ~ N(0, I)
    for t in reversed(range(T)):
        t_batch = torch.full((shape[0],), t)
        # 预测噪声
        eps = model(x, t_batch)
        # 单步去噪 (简化版, 实际含随机噪声)
        alpha_t = alphas[t]
        alpha_bar_t = alphas_cumprod[t]
        x = (1 / torch.sqrt(alpha_t)) * (
            x - ((1 - alpha_t) / torch.sqrt(1 - alpha_bar_t)) * eps
        )
        if t > 0:
            x += torch.sqrt(betas[t]) * torch.randn_like(x)  # 加随机性
    return x  # x_0: 生成图
```

## Worked Example

**为什么扩散模型预测噪声而非预测图?** -- 难度差异。

设任务: 给定 x_t(带噪图),要还原 x_0(清晰图)。

**直接预测图 x_0**:
- 网络要学: `f(x_t, t) -> x_0`
- 难点: x_t 噪声严重时,几乎没线索;网络要"凭空画一只猫"
- 当网络能预测一只带噪的猫时,它已学会画猫--**任务太难**

**预测噪声 ε**:
- 网络要学: `f(x_t, t) -> ε`(就是加的噪声)
- 难点: ε 是高斯噪声,**结构相对统一**(无内容)
- 网络只需识别"哪些是信号、哪些是噪声",把噪声扣掉
- 去噪: `x_{t-1} = x_t - ε_θ(x_t, t)` (近似)

**关键洞察**:
1. **预测目标难度不同**: 画一只猫(预测 x_0)远比识别噪声(预测 ε)难
2. **梯度信号**: ε 与噪声真值一一对应,梯度清晰;预测 x_0 在 t 大时几乎无监督信号
3. **米开朗基罗比喻**: "塑像就在石头里" -- 不是"创造塑像",而是"去掉多余石头"
4. **训练稳定**: 预测 ε 是简单 MSE,无 GAN 的对抗不稳定
5. **生成质量**: 扩散模型质量超过 GAN,FID 更低,多样性也好

**文生图训练数据**:
- ImageNet: 100 万图,每图一个类别标签(简单)
- LAION: 58.5 亿图文对,自然语言描述(丰富)
- 商业模型(Midjourney/SD)用 LAION,所以能识别多种语言、名人、各种风格

## Key Takeaways

1. 扩散模型 = 前向加噪(造数据) + 逆向去噪(生成图)
2. **噪声预测器**预测"图里加了什么噪声",再扣掉;比直接预测图简单
3. 训练是自监督: 标签就是加的噪声 ε,损失是 MSE
4. 步数 ID t 告诉网络当前噪声程度,**同一网络处理所有步**
5. 文生图加文字条件: 噪声预测器多输入文字嵌入
6. **潜在扩散**(Stable Diffusion): 在 VAE 潜在空间扩散,计算量降低数十倍
7. **CFG**(classifier-free guidance): 混合有条件和无条件预测,增强文字控制(典型 w=7.5)
8. 扩散模型质量超过 GAN,但生成慢(T 步去噪);DDIM 可加速到 20-50 步
