# Chapter 03: 线性模型与从零实现

## Core Idea
线性回归和 softmax 回归是深度学习的"hello world"--它们用最简单的模型把**三步框架**(模型+损失+优化)完整跑通。从零实现一遍能看清每个零件;简洁实现展示了 PyTorch 的标准写法。**从线性回归到深度网络只差一个激活函数**。

## Frameworks Introduced

- **线性回归模型 (d2l)**
  - When to use: 输出是连续值的回归任务
  - How: `y = Xw + b`,其中 X∈ℝⁿˣᵈ,w∈ℝᵈˣ¹,b∈ℝ。MSE 损失 `ℓ = (1/2n)‖ŷ-y‖²`,SGD 更新

- **矢量化加速 (d2l)**
  - When to use: 永远用,避免 for 循环
  - How: 用矩阵运算代替逐元素循环;GPU 并行处理整个批量;for 循环比矩阵乘法慢 100 倍以上

- **小批量随机梯度下降 (mini-batch SGD) (d2l)**
  - When to use: 几乎所有 DL 训练的默认优化器
  - How: 每次抽一批样本(16-512),计算这批的平均梯度,更新参数 `w ← w - η·(1/B)Σ∇ℓᵢ`。批量小→噪声大但更新快;批量大→稳但慢

- **softmax 回归 (d2l)**
  - When to use: 多分类任务(输出是离散类别)
  - How: ① 全连接层算 logits `o = Xw + b` ② softmax 归一化 `ŷⱼ = exp(oⱼ)/Σₖexp(oₖ)` ③ 交叉熵损失 `-Σyⱼlog(ŷⱼ)`

- **从零实现 vs 简洁实现 (d2l)**
  - When to use: 学习时先从零(理解机制),工程时用简洁(简洁高效)
  - How: 从零=只用 tensor + autograd,手写模型/损失/优化器;简洁=`nn.Module` 定义模型、`nn.MSELoss`/`CrossEntropyLoss`、`optim.SGD`

- **从线性回归到深度网络 (d2l)**
  - When to use: 理解为什么线性模型是 DL 的基本单元
  - How: 把 `y = Xw + b` 看成一个带 1 层全连接 + 恒等激活的神经网络。**DL 只是把"层数加多 + 引入非线性激活"**

## Key Concepts

- **线性回归 (linear regression)**: `ŷ = Xw + b`,最简单的回归模型
- **权重 w / 偏置 b**: 线性模型的参数
- **均方误差 (MSE)**: `ℓ = (1/2n)Σ(ŷᵢ-yᵢ)²`,回归默认损失
- **小批量 (mini-batch)**: 一次训练用的小样本集,大小为 batch_size
- **epoch**: 完整扫过一次训练集
- **学习率 η (learning rate)**: 每步更新的步长,典型 0.001-0.1
- **one-hot 编码**: 类别 → 仅正确类为 1 其余为 0 的向量
- **softmax**: 把任意实数向量变成概率分布(非负+和为 1)
- **logits**: softmax 之前的原始输出(未归一化分数)
- **交叉熵 (cross-entropy)**: `H(y, ŷ) = -Σyⱼlog(ŷⱼ)`,分类默认损失
- **熵 (entropy)**: `H(p) = -Σpⱼlog(pⱼ)`,分布的不确定度
- **KL 散度**: `D(p‖q) = H(p,q) - H(p)`,两个分布的差异
- **数据迭代器 (data_iter / DataLoader)**: 按批量遍历数据集的工具

## Mental Models

- **线性回归 = 1 层无激活的神经网络**: DL 不是与线性模型对立,而是它的扩展
- **矢量化永远优先**: 任何能写成矩阵运算的就别写循环
- **softmax 把"最大值"变成"概率"**: 让分类输出可解释、可微、能与交叉熵配合
- **交叉熵 = 负对数似然**: 最小化交叉熵 = 最大化正确类的对数概率
- **MSE 假设噪声服从高斯**: `y = Xw + b + ε, ε~N(0,σ²)` 时,MLE 给出的就是 MSE
- **从零实现是体检工具**: 一旦简洁实现出 bug,回退到从零实现能定位是框架问题还是算法问题

## Anti-patterns

- **用 for 循环遍历批量做计算**: 慢 100 倍,永远矢量化
- **softmax 直接算 `exp/sum`**: 数值不稳定(大数溢出)。用 `softmax(x) = softmax(x - max(x))` 减最大值
- **分类用 MSE**: MSE 对分类的梯度在错误大时反而小(饱和),用交叉熵
- **学习率不调**: 太大发散,太小龟速。先用 0.01 试,看 loss 曲线调整
- **从零实现写完就直接用**: 工程上永远用 nn.Module,从零只为学习
- **混淆 logits 和 probabilities**: logits 是原始分数,probabilities 是 softmax 后的概率,损失函数对二者不同

## Code Examples

**线性回归从零实现 - 完整训练循环**:

```python
import torch
import random

# 1. 生成数据: y = Xw + b + noise
def synthetic_data(w, b, num_examples):
    X = torch.normal(0, 1, (num_examples, len(w)))
    y = torch.matmul(X, w) + b
    y += torch.normal(0, 0.01, y.shape)  # 噪声
    return X, y.reshape((-1, 1))

true_w = torch.tensor([2.0, -3.4])
true_b = 4.2
features, labels = synthetic_data(true_w, true_b, 1000)

# 2. 数据迭代器
def data_iter(batch_size, features, labels):
    num_examples = len(features)
    indices = list(range(num_examples))
    random.shuffle(indices)
    for i in range(0, num_examples, batch_size):
        batch_indices = torch.tensor(indices[i:min(i+batch_size, num_examples)])
        yield features[batch_indices], labels[batch_indices]

# 3. 初始化参数
w = torch.normal(0, 0.01, size=(2,1), requires_grad=True)
b = torch.zeros(1, requires_grad=True)

# 4. 模型、损失、优化
def linreg(X, w, b): return torch.matmul(X, w) + b
def squared_loss(y_hat, y): return ((y_hat - y.reshape(y_hat.shape))**2 / 2).mean()
def sgd(params, lr, batch_size):
    with torch.no_grad():
        for param in params:
            param -= lr * param.grad / batch_size
            param.grad.zero_()

# 5. 训练
lr, num_epochs, batch_size = 0.03, 3, 10
for epoch in range(num_epochs):
    for X, y in data_iter(batch_size, features, labels):
        l = squared_loss(linreg(X, w, b), y)
        l.backward()
        sgd([w, b], lr, batch_size)

print(f'w 误差: {true_w - w.reshape(true_w.shape)}')
print(f'b 误差: {true_b - b}')
```
- **What it demonstrates**: 三步框架在 PyTorch 中的最小实现。**注意**: ① `requires_grad=True` 开启追踪 ② `with torch.no_grad():` 包住参数更新 ③ `param.grad.zero_()` 必须清零

**线性回归简洁实现 - nn.Module 标准写法**:

```python
import torch.nn as nn

# 1. 数据
from torch.utils.data import DataLoader, TensorDataset
dataset = TensorDataset(features, labels)
data_iter = DataLoader(dataset, batch_size=10, shuffle=True)

# 2. 模型
net = nn.Sequential(nn.Linear(2, 1))
net[0].weight.data.normal_(0, 0.01)
net[0].bias.data.fill_(0)

# 3. 损失 + 优化器
loss = nn.MSELoss()
trainer = torch.optim.SGD(net.parameters(), lr=0.03)

# 4. 训练
for epoch in range(3):
    for X, y in data_iter:
        l = loss(net(X), y)
        trainer.zero_grad()
        l.backward()
        trainer.step()
```
- **What it demonstrates**: 工程标准写法--`nn.Sequential` 定义模型、内置 Loss + Optim、`zero_grad / backward / step` 三件套

## Reference Tables

**回归 vs 分类对照**:

| 维度 | 线性回归 | softmax 回归 |
|---|---|---|
| 任务 | 连续值预测 | 多分类 |
| 模型 | `ŷ = Xw + b` | `ŷ = softmax(Xw + b)` |
| 输出 | 标量 | 概率向量(和为 1) |
| 损失 | MSE `(ŷ-y)²/2` | 交叉熵 `-Σyⱼlog(ŷⱼ)` |
| 标签 | 实数 | one-hot 向量 |
| 输出激活 | 恒等 | softmax |

**softmax 数值稳定技巧**:

| 写法 | 问题 | 解决 |
|---|---|---|
| `exp(x)/sum(exp(x))` | 大 x 溢出 | 减最大值: `exp(x-max)/sum(...)` |
| `log(softmax(x))` | 小概率 log 负无穷 | 用 `log_softmax` |
| 交叉熵 `−log(ŷ_y)` | ŷ→0 时爆炸 | 用 `F.cross_entropy(logits, y)`(自动 log_softmax) |

## Worked Example

**为什么 MSE 假设高斯噪声?** -- 从最大似然推导线性回归损失。

假设 `yᵢ = wᵀxᵢ + b + εᵢ`,其中 `εᵢ ~ N(0, σ²)`。则 `P(yᵢ|xᵢ; w,b) = (1/√(2πσ²))·exp(−(yᵢ−ŷᵢ)²/(2σ²))`。

**最大似然**: 找参数使观测数据的概率最大:
$$\hat{w}, \hat{b} = \arg\max_{w,b} \prod_i P(y_i|x_i; w, b)$$

取对数:
$$\log L = \sum_i \left[ -\log(\sqrt{2\pi\sigma^2}) - \frac{(y_i - \hat{y}_i)^2}{2\sigma^2} \right]$$

由于 σ 是常数,**最大化 log L 等价于最小化 `Σ(yᵢ−ŷᵢ)²`** -- 这就是 MSE 损失。

**关键洞察**: 选 MSE 不是拍脑袋,而是高斯噪声假设下的 MLE。如果噪声是别的分布(如 Laplace),对应 L1 损失。

## Key Takeaways

1. 线性回归 = 单层无激活神经网络;理解它就理解了 DL 的最小骨架
2. 永远矢量化--矩阵运算比 for 循环快两个数量级
3. mini-batch SGD 是 DL 优化器的默认起点;批量大小是关键的训练超参
4. softmax + 交叉熵是分类问题的标配;注意 logits 与 probabilities 的区别
5. 数值稳定性靠 `cross_entropy(logits, y)` 而非手算 softmax + log
6. 从零实现理解机制,简洁实现用于工程;两者都要会
7. MSE 是高斯噪声下的 MLE;损失函数的选择暗含分布假设
