# Chapter 11: 循环神经网络

## Core Idea
序列数据(文本、音频、时序)有顺序,RNN 用**隐藏状态 h_t** 把过去信息压缩成向量,与当前输入一起决定输出:**h_t = φ(W_xh·x_t + W_hh·h_{t-1} + b)**。但原始 RNN 训练困难(梯度消失/爆炸),LSTM/GRU 用**门控**让信息选择性流过,让网络能学长期依赖。**LSTM 是"长短期记忆"--通过遗忘门、输入门、输出门协调记忆元的读写**。

## Frameworks Introduced

- **序列模型与自回归 (d2l)**
  - When to use: 时间序列、文本等有序数据
  - How: 用过去预测未来 `x_t ~ P(x_t | x_{t-1}, ..., x_{t-T})`。**自回归**: 用过去 T 步作为特征预测下一步

- **语言模型 (d2l)**
  - When to use: 文本生成、语音识别、机器翻译
  - How: 估计序列概率 `P(x_1, ..., x_T) = Π P(x_t | x_{<t})`。n-gram 用马尔可夫假设近似

- **基本 RNN (d2l + LeeDL)**
  - When to use: 处理变长序列
  - How: 维护隐藏状态 h_t,每步更新 `h_t = tanh(W_xh·x_t + W_hh·h_{t-1} + b_h)`,输出 `y_t = W_hy·h_t + b_y`。**h_t 是过去所有输入的压缩表示**

- **通过时间反向传播 BPTT (d2l)**
  - When to use: 训练 RNN
  - How: 沿时间步展开计算图,链式法则求梯度。**问题**: 长序列梯度连乘,消失或爆炸

- **RNN 梯度问题 (LeeDL + d2l)**
  - When to use: 理解为什么需要 LSTM/GRU
  - How: 梯度沿时间反向时连乘 W_hh。|W_hh|<1 -> 梯度消失(长依赖学不到);|W_hh|>1 -> 梯度爆炸(数值发散)。**梯度裁剪**缓解爆炸;**门控**缓解消失

- **LSTM (LeeDL + d2l)**
  - When to use: 需要长期依赖的序列任务(NLP 默认)
  - How: 三个 sigmoid 门(输入门 i、遗忘门 f、输出门 o)+ 记忆元 c:
    - `c_t = f_t ⊙ c_{t-1} + i_t ⊙ g(z)` (记忆更新)
    - `h_t = o_t ⊙ tanh(c_t)` (隐藏状态)
  - **关键**: c_t 的更新是**加性**的,梯度可直接流过,缓解消失

- **GRU (d2l)**
  - When to use: LSTM 的轻量替代,参数少 1/3
  - How: 两个门(重置门 r、更新门 z):
    - `h_t = (1-z_t) ⊙ h_{t-1} + z_t ⊙ h̃_t`
  - 把 LSTM 的输入门和遗忘门合并为更新门(互补),省一个门

- **双向 RNN (d2l + LeeDL)**
  - When to use: 序列两端都有上下文可用(如翻译、NER 标注)
  - How: 一个 RNN 从左到右,另一个从右到左,两个隐藏状态拼接。**不能用于实时预测**(未来信息泄漏)

- **深层 RNN (d2l)**
  - When to use: 需要更强表达力
  - How: 多层 RNN 堆叠,每层输出作为下层输入。**实践中很少超过 3 层**

- **RNN 的应用模式 (LeeDL)**
  - When to use: 选 RNN 架构
  - How:
    - 多对一: 情感分析(序列 -> 标签)
    - 多对多(同步): 词性标注(每步一输出)
    - 多对多(异步): 机器翻译(先编码再解码)
    - 一对多: 图像描述生成

## Key Concepts

- **序列模型**: 处理有序数据的模型
- **自回归 (autoregression)**: 用过去预测未来
- **语言模型**: 估计序列概率
- **n-gram**: 用最近 n-1 个词预测第 n 个词
- **词元 (token)**: 文本的基本单元(词/子词/字符)
- **词表 (vocabulary)**: 所有词元的集合
- **隐藏状态 h_t**: RNN 的"记忆",压缩了过去信息
- **通过时间反向传播 (BPTT)**: RNN 的反向传播
- **梯度裁剪 (gradient clipping)**: 限制梯度范数防止爆炸
- **门控 (gating)**: 用 sigmoid(0-1) 控制信息流
- **记忆元 (cell state) c_t**: LSTM 的长期记忆通道
- **输入门 / 遗忘门 / 输出门**: LSTM 的三个 sigmoid 门
- **peephole 连接**: LSTM 把 c_{t-1} 也加入门控输入
- **GRU**: LSTM 的简化版本,2 个门
- **双向 RNN (BiRNN)**: 同时跑正向反向 RNN
- **编码器-解码器 (encoder-decoder)**: 序列到序列架构

## Mental Models

- **h_t 是过去的压缩**: 每步把新输入"融合"进 h_t,h_t 携带了所有历史信息(理论上)
- **RNN = 共享参数的 MLP 沿时间展开**: 同一组权重在每步重复用
- **门 = 可学习的开关**: sigmoid 输出 0-1,0 关 1 开,网络学何时开关
- **LSTM c_t 是"高速公路"**: 加性更新让梯度能直接流过长序列
- **门控让网络学"何时记、何时忘、何时输出"**: 不是被动接收,是主动管理记忆
- **BiRNN 不能实时**: 看到未来才能反向跑,适合离线任务
- **RNN 训练慢**: 序列必须按顺序算,不能像 CNN 那样并行

## Anti-patterns

- **长序列用原始 RNN**: 梯度消失学不到长期依赖。**用 LSTM/GRU**
- **梯度不裁剪**: RNN 梯度爆炸很常见。**默认开 `clip_grad_norm_`**
- **BiRNN 用于实时预测**: 反向 RNN 用了未来信息,实时场景泄漏
- **LSTM 不调学习率**: RNN 比 CNN 对 lr 更敏感,通常要更小(1e-3 ~ 5e-4)
- **序列长度差异大不填充**: batch 训练需 pad 到等长,要 mask 掉 pad 位置
- **LSTM 用 sigmoid 当激活**: 门用 sigmoid(0-1),但候选值 g(z) 用 tanh(-1,1)
- **RNN 堆太深**: 超过 3-4 层梯度难传,不如换 Transformer
- **遗忘门偏置初始化为 0**: LSTM 遗忘门偏置应初始化为 1(默认打开),否则初期全忘

## Code Examples

**基本 RNN 单元**:

```python
import torch
from torch import nn

# PyTorch 内置 RNN 层
rnn = nn.RNN(input_size=10, hidden_size=20, num_layers=1, batch_first=True)
X = torch.randn(32, 5, 10)  # (batch, seq_len, input_size)
output, h_n = rnn(X)
print(output.shape)  # (32, 5, 20) -- 每步的隐藏状态
print(h_n.shape)     # (1, 32, 20) -- 最后一步的隐藏状态
```

**LSTM 单元**:

```python
lstm = nn.LSTM(input_size=10, hidden_size=20, num_layers=2, 
               batch_first=True, bidirectional=False)
output, (h_n, c_n) = lstm(X)
print(output.shape)  # (32, 5, 20)
print(h_n.shape)     # (2, 32, 20) -- 每层的最后隐藏状态
print(c_n.shape)     # (2, 32, 20) -- 每层的最后记忆元
```

**LSTM 单元的数学实现**:

```python
class LSTMCell(nn.Module):
    def __init__(self, input_size, hidden_size):
        super().__init__()
        # 4 个门: 输入门 i, 遗忘门 f, 候选值 g, 输出门 o
        self.W = nn.Linear(input_size + hidden_size, 4 * hidden_size)
        self.hidden_size = hidden_size
    
    def forward(self, x, h_prev, c_prev):
        z = self.W(torch.cat([x, h_prev], dim=-1))
        i, f, g, o = z.chunk(4, dim=-1)
        i = torch.sigmoid(i)  # 输入门
        f = torch.sigmoid(f)  # 遗忘门
        g = torch.tanh(g)     # 候选值
        o = torch.sigmoid(o)  # 输出门
        c = f * c_prev + i * g              # 记忆更新
        h = o * torch.tanh(c)               # 隐藏状态
        return h, c
```
- **What it demonstrates**: 4 个线性变换一次性算出 4 个门的输入,然后各自激活。**记忆更新是加性的**: `f * c_prev + i * g`

**梯度裁剪** -- RNN 训练必备:

```python
torch.nn.utils.clip_grad_norm_(lstm.parameters(), max_norm=5.0)
optimizer.step()
```

**双向 LSTM**:

```python
bilstm = nn.LSTM(input_size=10, hidden_size=20, batch_first=True, 
                 bidirectional=True, num_layers=1)
output, _ = bilstm(X)
print(output.shape)  # (32, 5, 40) -- 正反两个方向拼接
```

## Reference Tables

**门控 RNN 对照**:

| 模型 | 门数 | 参数 | 关键公式 | 适用 |
|---|---|---|---|---|
| RNN | 0 | 1× | `h_t = tanh(W_xh x_t + W_hh h_{t-1})` | 短序列、教学 |
| LSTM | 3 | 4× | `c_t = f·c_{t-1} + i·g; h_t = o·tanh(c_t)` | NLP 默认 |
| GRU | 2 | 3× | `h_t = (1-z)·h_{t-1} + z·h̃` | 资源受限 |

**RNN 应用模式**:

| 模式 | 输入 | 输出 | 例子 |
|---|---|---|---|
| 多对一 | 序列 | 单标签 | 情感分析 |
| 多对多(同步) | 序列 | 等长序列 | 词性标注 |
| 多对多(异步) | 序列 | 变长序列 | 机器翻译(编码-解码) |
| 一对多 | 单输入 | 序列 | 图像描述 |

## Worked Example

**LeeDL 的 LSTM 数值示例** -- 看 3 个门如何协调工作。

设 LSTM 单元的 4 个输入由三维向量 `[x_1, x_2, x_3]` 控制:
- `x_1`: 候选输入值(通过权重 1 直接当输入)
- `x_2`: 控制输入门(权重 100)和遗忘门(权重 100,偏置 10)
- `x_3`: 控制输出门(权重 100,偏置 -10)

**遗忘门偏置 = +10**(默认打开,记得);**输出门偏置 = -10**(默认关闭,不输出)。

输入序列与计算:

| t | x | z(候选) | 输入门 | 遗忘门 | 输出门 | c_t | 输出 y |
|---|---|---|---|---|---|---|---|
| 1 | [3,1,0] | 3 | 1 (x_2=1>0) | 1 (默认) | 0 (默认关) | 0+3=3 | 0 |
| 2 | [4,1,0] | 4 | 1 | 1 | 0 | 3+4=7 | 0 |
| 3 | [2,0,0] | 2 | 0 (x_2=0) | 1 | 0 | 0·2+1·7=7 | 0 |
| 4 | [1,0,1] | 1 | 0 | 1 | 1 (x_3=1>0) | 7 | tanh(7)·1 ≈ 7 |
| 5 | [3,-1,0] | 3 | 0 (x_2<0) | 0 (x_2=-1<0) | 0 | 0·7+0·3=0 | 0 |

**关键洞察**:
1. **x_2 = 1 时累加**: 输入门开,候选值写入记忆元 (t=1,2)
2. **x_2 = 0 时保持**: 输入门关,候选值被挡;遗忘门开,旧值保留 (t=3)
3. **x_3 = 1 时输出**: 输出门开,记忆元值被读出 (t=4 输出 7)
4. **x_2 = -1 时遗忘**: 遗忘门关,记忆元清零 (t=5)

这就是 LSTM "何时记、何时忘、何时输出" 的本质--**门是可学的开关,网络自己学何时控制**。

## Key Takeaways

1. RNN 用隐藏状态 h_t 压缩过去,处理变长序列
2. 原始 RNN 梯度消失/爆炸,长依赖学不到;**用 LSTM/GRU**
3. LSTM 三门(输入、遗忘、输出)+ 加性记忆元 = 长期依赖可学
4. GRU 是 LSTM 的简化(2 门),参数少 1/3,效果相近
5. RNN 训练必开梯度裁剪;遗忘门偏置初始化为 1
6. 双向 RNN 利用未来信息,不能用于实时
7. RNN 应用分多对一/多对多/一对多,序列到序列用编码器-解码器
