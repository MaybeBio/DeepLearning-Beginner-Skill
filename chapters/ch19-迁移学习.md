# Chapter 19: 迁移学习

## Core Idea
**迁移学习**解决"标注成本高、数据不足"问题: 把从源领域(数据多)学到的知识迁移到目标领域(数据少)。**领域偏移**(domain shift)指训练/测试分布不同--MNIST 黑白数字训的模型在 MNIST-M 彩色数字上仅 52.25%。**领域自适应**(domain adaptation)针对目标领域有所了解: ① 有少量标签 -> 微调;② 有大量无标注数据 -> **领域对抗训练**(特征提取器 + 标签预测器 + 领域分类器三方博弈)。**领域泛化**(domain generalization)对目标领域一无所知: 训练时见多领域,测试新领域;或单领域训练 + 数据增强造多领域。

## Frameworks Introduced

- **领域偏移 (Domain Shift) (LeeDL)**
  - When to use: 训练/测试数据分布不同
  - How: 三类偏移:
    - **输入分布变化**: 训练黑白、测试彩色(MNIST -> MNIST-M, 52.25%)
    - **输出分布变化**: 训练各类等概率、测试某类概率高
    - **输入-输出关系变化**: 同图训练标 0、测试标 1
  - 关键: 实际应用普遍存在,基准数据集假设同分布掩盖了问题

- **迁移学习分类 (LeeDL)**
  - When to use: 任务数据不足
  - How:
    - **领域自适应**: 知道目标领域(有标签/无标签/少量标签)
    - **领域泛化**: 对目标领域一无所知
  - 源领域 = 训练数据;目标领域 = 测试数据

- **目标领域有少量标签: 微调 (LeeDL)**
  - When to use: 目标领域有少量标注数据
  - How:
    - 源领域预训练模型
    - 用目标领域数据跑 2-3 个 epoch 微调
    - **调小学习率**: 防止参数变化过大
    - **限制迭代次数**: 防止过拟合少量数据
  - 关键: 与 BERT 微调同理;模型参数/输入输出关系不要差太多

- **目标领域有大量无标注数据: 领域对抗训练 (LeeDL)**
  - When to use: 系统上线收集大量无标注数据(典型场景)
  - How:
    - 分类器拆为: **特征提取器**(前 N 层) + **标签预测器**(后 N 层)
    - **三方博弈**:
      - 标签预测器 θ_p: 最小化分类损失 L(在源领域上)
      - 领域分类器 θ_d: 最小化领域分类损失 L_d(区分源/目标)
      - 特征提取器 θ_f: 最小化 `L - L_d`(帮标签预测 + 骗领域分类器)
    - 特征提取器学"领域无关"表示,过滤颜色等无关信息
  - 关键: 类 GAN,但特征提取器优势大,要靠标签预测器约束不能输出零向量

- **决策边界对齐 (LeeDL)**
  - When to use: 领域对抗训练的改进
  - How:
    - 仅让特征分布对齐不够,还要让目标领域样本**远离决策边界**
    - 简单做法: 把无标注图过模型,若输出集中在某类(低熵)则离边界远;若各类接近(高熵)则近边界
    - 改进方法: DIRT-T、最大分类器差异(MCD)
  - 关键: 避免目标样本落在错误一侧的边界附近

- **类别不匹配的领域自适应 (LeeDL)**
  - When to use: 源/目标领域类别不同
  - How:
    - 三种情况: 源多目标少、源少目标多、各有独特类别
    - 强制完全对齐会失败: 老虎被拉到狗/狮子附近就分不开
    - 方法: Universal Domain Adaptation
  - 关键: 不能盲目对齐,要识别"共有类"和"私有类"

- **测试时训练 TTT (LeeDL)**
  - When to use: 目标领域只有一张图,无法对齐
  - How: 测试时用自监督任务在该样本上微调模型,再预测
  - 关键: 极端少样本场景

- **领域泛化 (Domain Generalization) (LeeDL)**
  - When to use: 对目标领域一无所知
  - How:
    - **情况 A - 多领域训练**: 训练数据含真实/素描/水彩等多种风格,测试时新风格(卡通)也能分类。方法: Domain Generalization with Adversarial Feature Learning
    - **情况 B - 单领域 + 数据增强**: 训练只有一领域,用数据增强造多领域。方法: Learning to Learn Single Domain Generalization
  - 关键: 学"弥平领域差异"的能力,而非针对某目标领域

## Key Concepts

- **迁移学习 (transfer learning)**: 跨任务/领域转移知识
- **源领域 (source domain)**: 训练数据来源
- **目标领域 (target domain)**: 测试数据来源
- **领域偏移 (domain shift)**: 训练/测试分布不同
- **领域自适应 (domain adaptation)**: 已知目标领域的迁移
- **领域泛化 (domain generalization)**: 未知目标领域的迁移
- **特征提取器 (feature extractor)**: 前层,提取领域无关特征
- **标签预测器 (label predictor)**: 后层,分类
- **领域分类器 (domain classifier)**: 区分源/目标领域
- **领域对抗训练 (domain-adversarial training)**: 三方博弈
- **领域无关表示 (domain-invariant representation)**: 两领域特征分布相同
- **决策边界 (decision boundary)**: 分类分界线
- **测试时训练 (TTT)**: 测试时用自监督微调
- **Universal Domain Adaptation**: 类别不匹配的领域自适应
- **DIRT-T**: 决策边界对齐方法
- **最大分类器差异 (MCD)**: 领域自适应改进

## Mental Models

- **领域偏移 = 跨域失效**: 训练黑白 99.91%,测试彩色 52.25%--分布差异击穿泛化
- **领域对抗 = GAN 变体**: 特征提取器(生成器)骗领域分类器(判别器),但还要服务标签预测器
- **特征提取器优势**: 输出零向量就骗过领域分类器--但标签预测器约束它必须有意义
- **决策边界 = 安全区**: 目标样本离边界越远越安全,近边界易跨到错误侧
- **微调 = 轻触即止**: 少量数据 + 大学习率 = 灾难性遗忘;小学习率 + 少迭代
- **领域泛化 = 见多识广**: 训练见多领域,测试新领域也能泛化
- **单领域泛化 = 数据增强造多样性**: 没多领域就用增强造

## Anti-patterns

- **假设训练/测试同分布**: 实际应用普遍领域偏移。**用领域自适应或泛化**
- **少量目标数据大学习率微调**: 过拟合到少量数据。**小学习率,少迭代**
- **强制完全对齐不同类别的领域**: 老虎被拉到狗附近。**识别共有/私有类**
- **特征提取器太强**: 输出零向量骗过分类器。**靠标签预测器约束**
- **只对齐特征分布**: 目标样本可能落在错误边界侧。**让样本远离决策边界**
- **目标领域一张图也用领域对抗**: 无法对齐。**用 TTT**
- **领域分类器权重过大**: 特征提取器只顾骗分类器,忽略标签预测。**平衡三项损失**
- **领域泛化只训一领域**: 无领域多样性。**多领域训练或数据增强**

## Code Examples

**微调(少量目标标签)**:

```python
import torch
from torch import nn, optim

# 1. 加载源领域预训练模型
model = load_pretrained_model()
# 2. 用目标领域少量数据微调
optimizer = optim.SGD(model.parameters(), lr=1e-4)  # 小学习率!
criterion = nn.CrossEntropyLoss()

for epoch in range(3):  # 少迭代!
    for x, y in target_labeled_loader:
        loss = criterion(model(x), y)
        optimizer.zero_grad(); loss.backward(); optimizer.step()
```

**领域对抗训练**:

```python
class DomainAdversarialModel(nn.Module):
    def __init__(self, feature_dim=256, num_classes=10):
        super().__init__()
        self.feature_extractor = nn.Sequential(
            nn.Conv2d(3, 64, 3, padding=1), nn.ReLU(),
            nn.AdaptiveAvgPool2d((1, 1)), nn.Flatten(),
            nn.Linear(64, feature_dim)
        )
        self.label_predictor = nn.Linear(feature_dim, num_classes)
        self.domain_classifier = nn.Sequential(
            nn.Linear(feature_dim, 64), nn.ReLU(),
            nn.Linear(64, 2)  # 源 vs 目标
        )
    
    def forward(self, x):
        feat = self.feature_extractor(x)
        logits = self.label_predictor(feat)
        domain = self.domain_classifier(feat)
        return feat, logits, domain

model = DomainAdversarialModel()
opt_f = optim.SGD(model.feature_extractor.parameters(), lr=0.01)
opt_p = optim.SGD(model.label_predictor.parameters(), lr=0.01)
opt_d = optim.SGD(model.domain_classifier.parameters(), lr=0.01)

for src_x, src_y in source_loader:
    tgt_x, _ = next(iter(target_unlabeled_loader))
    
    feat_s, logits_s, domain_s = model(src_x)
    feat_t, _, domain_t = model(tgt_x)
    
    # 1. 标签预测器 + 特征提取器: 最小化 L
    loss_cls = nn.functional.cross_entropy(logits_s, src_y)
    # 2. 领域分类器: 最小化 L_d
    domain_labels = torch.cat([torch.zeros(len(src_x)),
                                torch.ones(len(tgt_x))]).long()
    loss_domain = nn.functional.cross_entropy(
        torch.cat([domain_s, domain_t]), domain_labels)
    # 3. 特征提取器: L - L_d (骗领域分类器)
    
    # 更新领域分类器
    opt_d.zero_grad(); loss_domain.backward(retain_graph=True); opt_d.step()
    # 更新标签预测器
    opt_p.zero_grad(); loss_cls.backward(retain_graph=True); opt_p.step()
    # 更新特征提取器 (分类损失 - 领域损失)
    loss_fe = loss_cls - loss_domain
    opt_f.zero_grad(); loss_fe.backward(); opt_f.step()
```
- **What it demonstrates**: 三方博弈--特征提取器要骗领域分类器(-L_d),又要服务标签预测器(+L)

**决策边界对齐(简化版)**:

```python
def boundary_alignment_loss(model, tgt_x):
    """让目标领域预测熵最小化(远离边界)"""
    _, logits, _ = model(tgt_x)
    probs = nn.functional.softmax(logits, dim=1)
    entropy = -(probs * torch.log(probs + 1e-8)).sum(1).mean()
    return entropy  # 最小化熵 = 远离边界

# 加到总损失
total_loss = loss_cls + loss_domain + 0.1 * boundary_alignment_loss(model, tgt_x)
```

## Worked Example

**为什么领域对抗训练能让模型在彩色数字上工作?** -- MNIST -> MNIST-M 案例。

**问题**: MNIST(黑白)训的模型在 MNIST-M(彩色)只有 52.25%。颜色信息干扰了形状识别。

**直接思路**: 把彩色图变回黑白? -- 但目标领域图片无法预先处理(系统上线后无法改输入)。

**领域对抗训练思路**: 让模型自己学会"忽略颜色"。

**机制**:
1. **特征提取器**: 把图压成特征向量
2. **标签预测器**: 从特征预测数字类别
3. **领域分类器**: 从特征预测"来自 MNIST 还是 MNIST-M"

**三方博弈**:
- 领域分类器想区分源/目标(看特征就知道颜色)
- 特征提取器想骗领域分类器(让源/目标特征分布相同)
- 骗成功 = 特征里**没有颜色信息** = 只剩形状信息
- 标签预测器用这种"颜色无关"特征预测,自然在彩色图上也工作

**为什么特征提取器不会作弊(输出零向量)?**
- 输出零向量骗过领域分类器,但标签预测器无法从零向量预测类别
- 标签预测器的损失 L 约束特征提取器必须输出有意义的向量
- **平衡**: 特征要"领域无关"但"类别可分"

**LeeDL 实验数据**:
- MNIST -> MNIST-M:
  - 只用源领域: 57.49%
  - 领域对抗训练: 81.49% (大幅提升)
  - 上限(目标领域训练): 98.91%

**关键洞察**:
1. **领域无关表示**: 两领域特征分布相同 = 特征不含领域专属信息(颜色)
2. **GAN 思想迁移**: 骗分类器 = 让特征无领域信息
3. **三类损失平衡**: L(分类)、L_d(领域)、-L_d(反领域)共同优化
4. **决策边界对齐**: 仅对齐特征分布不够,目标样本还要远离边界
5. **类别不匹配陷阱**: 强制对齐会让"老虎"被拉到"狗"附近--需识别共有类
6. **现实场景契合**: 系统上线后收集无标注数据,正好用于领域对抗训练
7. **领域泛化是升级版**: 训练时见多领域,测试新领域无需适应

## Key Takeaways

1. 领域偏移: 训练/测试分布不同导致泛化失效,MNIST->MNIST-M 仅 52.25%
2. 迁移学习分两类: 领域自适应(知目标领域) + 领域泛化(不知目标领域)
3. 目标领域有少量标签: **微调**(小学习率、少迭代防过拟合)
4. 目标领域有大量无标注: **领域对抗训练**,三方博弈让特征领域无关
5. 领域对抗 = 特征提取器(θ_f) + 标签预测器(θ_p) + 领域分类器(θ_d),优化 `L - L_d`
6. 特征提取器靠标签预测器约束,不能输出零向量作弊
7. 决策边界对齐: 目标样本要远离边界,方法有 DIRT-T、MCD
8. 类别不匹配时不能强制完全对齐,用 Universal Domain Adaptation
9. 领域泛化: 多领域训练学弥平差异,或单领域 + 数据增强造多样性
10. 极端少样本(一张图)用 TTT 测试时训练
