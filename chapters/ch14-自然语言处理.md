# Chapter 14: 自然语言处理

## Core Idea
NLP 的核心是**把词变成向量**。从独热(无相似度) -> word2vec(上下文无关静态向量) -> ELMo(上下文敏感但任务特定) -> GPT(任务无关但单向) -> **BERT**(双向 + 任务无关)。**BERT** 用 Transformer 编码器 + 两个自监督预训练任务(**掩蔽语言模型 MLM** + **下一句预测 NSP**)学到通用文本表示,微调即可适配大多数 NLP 任务(分类、推断、问答、标注),只需加一个输出层。这彻底改变了 NLP 范式: 不再为每个任务设计专门架构,而是预训练 + 微调。

## Frameworks Introduced

- **词嵌入动机 (d2l)**
  - When to use: 把词映射为稠密向量供神经网络用
  - How: 独热向量两两余弦相似度为 0,无法表达词间相似度。**词嵌入**学得稠密向量,使语义相近的词向量也相近

- **跳元模型 Skip-Gram (d2l)**
  - When to use: 大规模无标注语料上预训练词向量
  - How: 用中心词预测上下文词。每个词有 2 个向量(中心词 v、上下文词 u)。给定中心词 w_c,生成上下文词 w_o 的概率 `P(w_o|w_c) = softmax(u_oᵀv_c)`。最大化所有 (中心, 上下文) 对的似然
  - 关键: 自监督(无需人工标签),窗口 m 内的词都算上下文

- **连续词袋 CBOW (d2l)**
  - When to use: 小数据集上训词向量(比 Skip-Gram 更稳)
  - How: 用上下文词预测中心词。上下文词向量平均后做 softmax。与 Skip-Gram 对偶

- **近似训练 (d2l)**
  - When to use: 词表大时 softmax 太慢(|V| 几十万)
  - How:
    - **负采样**: 把多类问题转成二分类。正例 = (中心, 真上下文),负例 = (中心, 随机噪声词 K 个)。损失 = -log σ(u_oᵀv_c) - Σ log σ(-u_kᵀv_c)。**计算量与词表无关,只与 K 有关**
    - **层序 softmax**: 用二叉树,叶节点是词,路径长度 log|V|。每个内部节点做二分类。计算量降到 log|V|

- **GloVe (d2l)**
  - When to use: 想用全局词共现统计(而非局部窗口)
  - How: 基于词共现矩阵 X(全局统计)。损失 `Σ (w_iᵀw_j + b_i + b_j - log X_ij)² · f(X_ij)`,f 是权重函数(高频词降权)。**从条件概率比值理解 GloVe**: 比值能编码词间关系

- **子词嵌入 (d2l)**
  - When to use: 罕见词、OOV(out-of-vocabulary)词
  - How:
    - **fastText**: 把词拆成字符 n-gram,词向量 = n-gram 向量之和。罕见词也能拼出来
    - **BPE (字节对编码)**: 从字符开始,迭代合并最高频字符对,得到子词词表。能处理任何词,跨语言通用

- **BERT (d2l)**
  - When to use: 任何 NLP 任务的预训练 backbone
  - How:
    - **输入表示**: 词元嵌入 + 片段嵌入(区分句子 A/B) + 可学习位置嵌入。`<cls>` 句首,`<sep>` 句分隔
    - **架构**: Transformer 编码器堆 N 层
    - **预训练任务 1 - 掩蔽语言模型 MLM**: 随机选 15% 词元掩蔽(80% 替换为 `<mask>`,10% 随机替换,10% 不变),用双向上下文预测被掩蔽的词
    - **预训练任务 2 - 下一句预测 NSP**: 50% 句子对是真实连续(B=IsNext),50% 随机配对(B=NotNext)。用 `<cls>` 表示做二分类
    - **微调**: 加一个输出层,所有 BERT 参数 + 新层一起训

- **从 ELMo/GPT 到 BERT (d2l)**
  - When to use: 理解预训练模型演进
  - How:
    - **ELMo**: 双向 LSTM,上下文敏感,但**任务特定架构**(每个任务设计专门模型)
    - **GPT**: Transformer 解码器,任务无关,但**单向**(只能看左不能看右)
    - **BERT**: Transformer 编码器,**双向 + 任务无关**,集大成

- **NLP 应用分类 (d2l)**
  - When to use: 选 NLP 任务范式
  - How:
    - **序列级**: 单文本分类(情感分析)、文本对分类/回归(NLI、语义相似度)
    - **词元级**: 文本标注(词性标注)、问答(预测片段起止)
  - 微调 BERT 时,不同任务只需不同的输出层,BERT 主体共享

- **微调 BERT 的四种任务 (d2l)**
  - When to use: 把 BERT 适配到下游任务
  - How:
    - **单文本分类**: `<cls>` 表示 -> MLP -> 类别
    - **文本对分类**: 拼成 `<cls> A <sep> B <sep>` -> `<cls>` 表示 -> MLP -> 类别
    - **文本标注**: 每个词元表示 -> 共享 MLP -> 每词元标签
    - **问答**: 拼问题和段落 -> 每词元预测 start/end 分数 -> softmax 找最佳片段

## Key Concepts

- **词嵌入 (word embedding)**: 词到稠密向量的映射
- **独热向量 (one-hot)**: 长度 |V|、单个 1 的稀疏向量
- **余弦相似度**: `xᵀy / (||x||·||y||)`,衡量向量相似度
- **词表 (vocabulary)**: 所有词元的集合
- **词元 (token)**: 文本基本单元(词/子词/字符)
- **跳元模型 (Skip-Gram)**: 中心词预测上下文
- **连续词袋 (CBOW)**: 上下文预测中心词
- **自监督 (self-supervised)**: 从数据自身构造标签
- **负采样 (negative sampling)**: 把多类转二分类的近似训练
- **层序 softmax (hierarchical softmax)**: 二叉树路径近似 softmax
- **GloVe**: 全局向量词嵌入,基于共现统计
- **子词嵌入 (subword embedding)**: 拆词为子词,处理 OOV
- **fastText**: 字符 n-gram 子词嵌入
- **BPE (字节对编码)**: 迭代合并最高频字符对
- **上下文无关表示 (context-free)**: 同词总同向量(word2vec)
- **上下文敏感表示 (context-sensitive)**: 同词不同上下文不同向量(ELMo/BERT)
- **ELMo**: 双向 LSTM 的上下文表示
- **GPT**: Transformer 解码器,单向任务无关
- **BERT**: Transformer 编码器,双向任务无关
- **掩蔽语言模型 (MLM)**: BERT 预训练任务之一
- **下一句预测 (NSP)**: BERT 预训练任务之二
- **片段嵌入 (segment embedding)**: 区分句子 A/B
- **微调 (fine-tuning)**: 预训练模型适配下游任务
- **自然语言推断 (NLI)**: 判断前提和假设的关系(蕴涵/矛盾/中性)

## Mental Models

- **词向量 = 词义坐标**: 词嵌入把词映射到语义空间,相似词聚集;`vec(king) - vec(man) + vec(woman) ≈ vec(queen)` -- 类比关系也在空间中
- **word2vec = 窗口内预测**: 不需要标签,从语料本身的窗口结构自监督学
- **负采样 = "是真上下文 vs 随机噪声"**: 不用算 |V| 类 softmax,只需 K+1 个二分类
- **静态 vs 动态**: word2vec 是静态的("bank"无论上下文都是同一向量);BERT 是动态的(根据上下文调整)
- **BERT = "完形填空 + 句子关系"**: MLM 像完形填空(挖空猜词),NSP 像判断两句话是否相连;两个任务都用自监督标签
- **BERT 输入 = 三种嵌入相加**: 词元 + 片段 + 位置,告诉模型"是什么词、属于哪句、在哪个位置"
- **微调 = 复用 backbone + 加小头**: 不同任务共享 BERT,只换输出层;预训练知识迁移到所有任务

## Anti-patterns

- **小语料用 word2vec**: 学不到好向量,数据少时直接用预训练 GloVe/BERT
- **Skip-Gram 不用负采样**: 全 softmax 计算爆炸,必须用负采样或层序 softmax
- **掩蔽比例过高/过低**: BERT 用 15% 是经验最优;过高破坏语法,过低学不到双向
- **掩蔽全用 `<mask>`**: 微调时没 `<mask>` 会失配;**80% `<mask>` + 10% 随机 + 10% 不变**
- **GPT 用于需要双向的任务**: GPT 单向,做 NER/NLI 不如 BERT
- **BERT 用于生成**: BERT 是编码器,不能自回归生成;用 GPT/T5
- **微调时学习率过大**: BERT 参数多,大 lr 会破坏预训练知识。**用 1e-4 ~ 5e-5**
- **下游任务不截断长序列**: BERT 输入长度有限(原版 512),长文本要滑窗或截断
- **NSP 任务被认为无效**: RoBERTa 发现 NSP 作用有限,可去掉;但原始 BERT 仍保留

## Code Examples

**Skip-Gram 模型前向**:

```python
import torch
from torch import nn

def skip_gram(center, contexts_and_negatives, embed_v, embed_u):
    """Skip-Gram 前向
    center: (batch, 1) 中心词索引
    contexts_and_negatives: (batch, max_len) 上下文+负样本索引
    """
    v = embed_v(center)                  # (batch, 1, dim)
    u = embed_u(contexts_and_negatives)  # (batch, max_len, dim)
    pred = torch.bmm(v, u.permute(0, 2, 1))  # (batch, 1, max_len)
    return pred
```

**BERT 输入表示**:

```python
class BERTEncoder(nn.Module):
    def __init__(self, vocab_size, num_hiddens, num_heads, num_layers,
                 ffn_num_hiddens, dropout, max_len=1000):
        super().__init__()
        self.token_embedding = nn.Embedding(vocab_size, num_hiddens)
        self.segment_embedding = nn.Embedding(2, num_hiddens)  # 句子 A/B
        # 可学习的位置嵌入 (与原始 Transformer 不同)
        self.pos_embedding = nn.Parameter(torch.randn(1, max_len, num_hiddens))
        self.blks = nn.Sequential(*[
            EncoderBlock(num_hiddens, num_heads, ffn_num_hiddens, dropout)
            for _ in range(num_layers)
        ])
    
    def forward(self, tokens, segments, valid_lens):
        # 三种嵌入相加
        X = self.token_embedding(tokens) + self.segment_embedding(segments)
        X = X + self.pos_embedding.data[:, :X.shape[1], :]
        for blk in self.blks:
            X = blk(X, valid_lens)
        return X
```

**BERT 掩蔽语言模型头**:

```python
class MaskLM(nn.Module):
    """预测被掩蔽的词元"""
    def __init__(self, vocab_size, num_hiddens, num_inputs):
        super().__init__()
        self.mlp = nn.Sequential(
            nn.Linear(num_inputs, num_hiddens), nn.ReLU(),
            nn.LayerNorm(num_hiddens),
            nn.Linear(num_hiddens, vocab_size)
        )
    
    def forward(self, X, pred_positions):
        # X: (batch, seq_len, dim), pred_positions: (batch, n_pred)
        num_pred = pred_positions.shape[1]
        pred_positions = pred_positions.reshape(-1)
        batch_size = X.shape[0]
        batch_idx = torch.arange(batch_size).repeat_interleave(num_pred)
        masked_X = X[batch_idx, pred_positions].reshape(batch_size, num_pred, -1)
        return self.mlp(masked_X)  # (batch, n_pred, vocab_size)
```

**微调 BERT 做单文本分类**:

```python
class BERTClassifier(nn.Module):
    def __init__(self, bert):
        super().__init__()
        self.encoder = bert.encoder           # 预训练 BERT 编码器
        self.hidden = bert.hidden             # 预训练 MLP 隐藏层
        self.output = nn.Linear(256, num_classes)  # 新输出层 (从头训)
    
    def forward(self, inputs):
        tokens_X, segments_X, valid_lens = inputs
        encoded_X = self.encoder(tokens_X, segments_X, valid_lens)
        # 用 <cls> (位置 0) 的表示做分类
        return self.output(self.hidden(encoded_X[:, 0, :]))
```

## Worked Example

**为什么 BERT 用 MLM 而不是标准语言模型?** -- 看双向 vs 单向的差别。

设句子 "the man went to the bank to deposit cash"。要表示 "bank" 一词:

**标准语言模型 (GPT, 单向)**:
- 只看左侧: "the man went to the"
- "bank" 的表示 = f("the man went to the")
- 问题: "bank" 是河岸还是银行? 看左侧分不清

**掩蔽语言模型 (BERT, 双向)**:
- 掩蔽 "bank" 为 `<mask>`: "the man went to the `<mask>` to deposit cash"
- "bank" 的表示 = f("the man went to the ... to deposit cash")
- 关键: 看到了 "deposit cash",**确定是银行**

**关键洞察**:
1. **多义词消歧**: 同一个 "bank" 在不同上下文有不同含义,静态向量(word2vec)无法区分;BERT 根据上下文动态调整
2. **双向看完整句**: MLM 让每个位置都能看到左右两侧,完整理解语境
3. **GPT 的局限**: 自回归 LM 只能左到右,右侧信息丢失;适合生成,不适合理解
4. **BERT 的代价**: 不能用于生成(掩蔽位置在微调时不存在),但理解任务(NLI/QA/分类)上完胜
5. **掩蔽策略 (80/10/10)**:
   - 80% `<mask>`: 主任务,预测原词
   - 10% 随机词: 强迫模型不单纯依赖 `<mask>` 标记
   - 10% 不变: 让模型学到"即使没掩蔽也要预测",与微调一致

**实验数据**(SNLI 自然语言推断):
- 用注意力机制专门设计: ~78% 测试精度
- 微调 bert.small: ~78% (相当)
- 微调 bert.base: **>86%** (大幅提升)

预训练 + 微调范式胜过为每个任务设计专门架构。

## Key Takeaways

1. 词嵌入把词映射为稠密向量,word2vec(Skip-Gram/CBOW)是经典方法,**自监督**学
2. 大词表 softmax 太慢,用**负采样**或**层序 softmax**近似训练
3. GloVe 用全局共现统计,fastText 用子词处理 OOV,BPE 是现代分词基础
4. 静态嵌入无法消歧;**上下文敏感表示**(ELMo/GPT/BERT)根据上下文调整
5. **BERT = Transformer 编码器 + MLM + NSP**,双向 + 任务无关,只需微调即可适配大多数 NLP 任务
6. BERT 输入 = 词元嵌入 + 片段嵌入 + 可学习位置嵌入;`<cls>` 表示整句,`<sep>` 分隔
7. 微调 BERT: 加输出层,小学习率(1e-4),所有参数一起训
8. 不同 NLP 任务(分类/标注/问答)只需不同输出层,BERT 主体共享
