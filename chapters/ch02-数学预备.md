# Chapter 02: 数学预备

## Core Idea
深度学习的全部数学骨架是:**张量**存数据、**线性代数**做变换、**微积分**算梯度、**自动微分**让框架代你求导、**概率**刻画不确定性。这五块合起来,就是把"找函数"这件事落到可执行代码的语言。

## Frameworks Introduced

- **张量 = n 维数组 + GPU + 自动微分 (d2l)**
  - When to use: 任何 DL 代码的起点
  - How: 用 `torch.tensor` 创建;支持按元素运算、广播、索引切片;比 NumPy 多了 GPU 加速和 `requires_grad`

- **降维操作的三种语义 (d2l)**
  - When to use: 理解 sum/mean 等操作如何改变形状
  - How: `axis=` 指定沿哪个轴求和;`keepdims=True` 保留形状便于广播;不指定 axis 则对所有元素求和得到标量

- **范数 = 衡量向量"大小" (d2l)**
  - When to use: 衡量权重、梯度、损失的距离
  - How: L2 范数 `‖x‖₂ = √Σxᵢ²`;L1 范数 `‖x‖₁ = Σ|xᵢ|`;范数满足:非负、齐次、三角不等式

- **梯度 = 多元函数的最速上升方向 (d2l)**
  - When to use: 优化算法的更新方向
  - How: 函数 f 对向量 x 的梯度 ∇ₓf 是各偏导数组成的向量;**梯度指向上升方向,所以梯度下降要减去梯度**

- **链式法则 = 反向传播的数学基础 (d2l)**
  - When to use: 复合函数求导
  - How: 若 z = f(g(x)),则 `dz/dx = (df/dg)·(dg/dx)`。深度网络的损失对参数的梯度就是一长串链式法则的乘积

- **自动微分 = 框架替你跑链式法则 (d2l)**
  - When to use: 永远用,不要手算梯度
  - How: ① 在叶子张量上设 `requires_grad_(True)` ② 计算前向得到标量 loss ③ `loss.backward()` 触发反向 ④ 在 `x.grad` 读取梯度

## Key Concepts

- **张量 (tensor)**: n 维数组,DL 的基本数据结构
- **广播机制 (broadcasting)**: 形状不同的张量按规则扩展到相同形状再按元素运算(沿长度为 1 的轴复制)
- **点积 (dot product)**: `xᵀy = Σxᵢyᵢ`,衡量两个向量的相似度
- **矩阵-向量积**: `Ax` 把 x 变换到 A 的列空间
- **范数 (norm)**: 衡量向量长度;L2 平滑可导(默认选择),L1 促进稀疏
- **导数 (derivative)**: 一元函数的瞬时变化率
- **偏导数 (partial derivative)**: 多元函数沿某一坐标轴的导数(其他变量固定)
- **梯度 (gradient)**: 所有偏导数组成的向量
- **链式法则 (chain rule)**: 复合函数求导法则
- **计算图 (computational graph)**: 框架记录前向运算的有向无环图,反向传播沿图回传梯度
- **反向传播 (backpropagation)**: 沿计算图从输出到输入填充每个参数的偏导数
- **随机变量 (random variable)**: 可能取多个值的变量,用分布刻画
- **联合概率 P(A,B)**: 两事件同时发生
- **条件概率 P(A|B)**: 已知 B 发生时 A 的概率
- **贝叶斯定理**: `P(A|B) = P(B|A)·P(A) / P(B)`--由先验和似然推后验
- **期望 (expectation)**: `E[X] = Σx·P(x)`,随机变量的平均取值
- **方差 (variance)**: `Var(X) = E[(X-E[X])²]`,取值的离散程度
- **大数定律**: 样本均值随样本数增加趋近真实期望

## Mental Models

- **把张量当 NumPy 数组 + 梯度追踪**: 任何 NumPy 直觉都可直接迁移,只是多了 `requires_grad` 这个开关
- **降维=沿轴求和**: `axis=0` 压扁行,`axis=1` 压扁列;`keepdims=True` 保留维度便于广播
- **梯度下降的物理图像**: 站在山上,梯度指向最陡上升方向;往梯度反方向走一步,就下降最快
- **自动微分是"录音回放"**: 前向是把操作录下来,backward 是倒带计算每个参数对损失的贡献
- **概率刻画不确定性**: 不知道是猫是狗时,用 P(猫)=0.7 比"我觉得是猫"更精确

## Anti-patterns

- **忘记 `x.grad.zero_()`**: PyTorch 的梯度默认累积,不清零就会把上一轮的梯度加到这一轮,训练直接崩
- **对非标量直接 `backward()`**: loss 必须是标量;若是向量,要么 `.sum().backward()` 要么传 `gradient` 参数
- **混用 NumPy 和 tensor 不转换**: 数据进入 PyTorch 前要 `torch.tensor(np_array)`,否则类型不匹配
- **手算复杂梯度**: 任何超过两层的网络手算必错,永远用 autograd
- **忽略 `requires_grad`**: 默认张量不追踪梯度,模型参数必须显式开启

## Code Examples

**自动微分的最小例子** -- 求 `y = 2xᵀx` 关于 x 的梯度:

```python
import torch

x = torch.arange(4.0)
x.requires_grad_(True)  # 等价于 x = torch.arange(4.0, requires_grad=True)
print(x.grad)  # 默认 None

y = 2 * torch.dot(x, x)  # y = 2 * (x·x) = 2 * 14 = 28
y.backward()
print(x.grad)        # tensor([0., 4., 8., 12.])
print(x.grad == 4*x) # tensor([True, True, True, True])
```
- **What it demonstrates**: `requires_grad_(True)` 开启追踪;前向算 y;`y.backward()` 触发反向;`x.grad` 读取结果。梯度 `4x` 与手算一致

**清零 + 重新求导** -- 必须清零否则梯度累积:

```python
x.grad.zero_()  # 清除上一轮的梯度
y = x.sum()
y.backward()
print(x.grad)  # tensor([1., 1., 1., 1.])  -- 而不是 [1,5,9,13]
```

**分离计算 (detach)** -- 把张量从计算图里摘出来当常数:

```python
x.grad.zero_()
y = x * x
u = y.detach()  # u 与 y 同值,但不再追踪对 x 的依赖
z = u * x
z.sum().backward()
print(x.grad)  # u (常数) -- 而不是 3x² (若不 detach)
```
- **What it demonstrates**: detach 用于"把某段计算冻结成常数",常用于 GAN 的判别器训练、target 网络更新等场景

## Worked Example

**从零开始用 autograd 实现梯度下降** -- 求 f(x) = x² 的最小值(应在 x=0):

```python
import torch

x = torch.tensor(5.0, requires_grad=True)  # 起点 x=5
lr = 0.1                                   # 学习率

for i in range(50):
    y = x * x           # f(x) = x²
    y.backward()        # 反向传播: dy/dx = 2x
    with torch.no_grad():
        x -= lr * x.grad  # 梯度下降: x ← x - η·∇f
    x.grad.zero_()      # 必须清零!
    
print(x)  # tensor(~0., requires_grad=True)
```

**关键洞察**:
1. `requires_grad=True` 让 x 追踪运算
2. `y.backward()` 自动算 dy/dx
3. 更新参数时要用 `with torch.no_grad():` 包住,否则会被记录到计算图
4. **梯度必须清零**,否则下一轮 backward 会累加
5. 这个 4 步循环就是所有深度学习训练的雏形

## Key Takeaways

1. 张量是 DL 的统一数据结构,兼具 NumPy 的接口 + GPU 加速 + 自动微分
2. 线性代数三件套: 点积(相似度)、矩阵乘(变换)、范数(大小)
3. 链式法则是反向传播的数学基础,自动微分让框架替你算
4. 永远手写 `x.grad.zero_()` 清零梯度;永远用 `with torch.no_grad():` 包住参数更新
5. detach 把张量冻结成常数,常用于 GAN、target net 等场景
6. 概率是刻画不确定性的语言;贝叶斯定理是从先验+似然推后验的核心工具
