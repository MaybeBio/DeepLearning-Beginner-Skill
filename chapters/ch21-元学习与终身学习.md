# Chapter 21: 元学习与终身学习

## Core Idea
**元学习 (Meta Learning)** = "学习如何学习"--用一批**训练任务**学一个**学习算法 F_φ**(而非单个模型),让它能快速适应**新任务**。**终身学习 (LifeLong Learning, LLL)** = 让模型持续学习多任务,但面临**灾难性遗忘**(学新忘旧)。元学习三步骤与 ML 对应: ① 学算法中的可学参数 φ(初始化/学习率/网络架构);② 在 N 个任务上定义损失 `L(φ) = Σ l^n`(每个任务用**查询集**评估);③ 优化 φ。**MAML** 学好的初始化参数,让少数几步梯度下降就能适应新任务(小样本学习)。终身学习解法: **选择性突触可塑性**给重要参数加"守卫" `L' = L + λΣb_i(θ_i - θ_i^b)²`,重要参数不动,次要参数自由学。**多任务学习是终身学习上界**但需重训所有任务数据。

## Frameworks Introduced

- **元学习本质 (LeeDL)**
  - When to use: 学习算法本身需要优化(超参/初始化/架构)
  - How:
    - 学习算法 F_φ 是函数,输入训练数据,输出模型 f_θ
    - φ 是算法中可学的部分(初始化参数、学习率、网络架构等)
    - 目标: 找 φ* 让 F_φ 在新任务上表现最好
  - 关键: ML 找模型 f,元学习找学习算法 F

- **元学习三步骤 (LeeDL)**
  - When to use: 设计元学习算法
  - How:
    1. **定义可学参数 φ**: 初始化参数、学习率、网络架构
    2. **定义损失 L(φ)**: 在 N 个训练任务上,每个任务用训练数据学出 f_{θ^n*},用测试数据(查询)算 l^n,`L(φ) = Σ l^n`
    3. **优化**: 梯度下降/RL/进化算法找 φ* 最小化 L(φ)
  - 关键: 训练单位是任务,每个任务有训练(支持)+ 测试(查询)

- **元学习 vs 机器学习 (LeeDL)**
  - When to use: 区分两者
  - How:
    - **ML**: 找 f,用训练数据(支持)训,单一任务训练 + 测试
    - **元学习**: 找 F_φ,用训练任务训,跨任务训练 + 测试
    - 损失: ML 用 `L(θ) = Σ e_k`(训练数据损失);元学习用 `L(φ) = Σ l^n`(查询损失)
    - **外循环**: 跨任务训练(更新 φ);**内循环**: 单一任务训练(更新 θ)
  - 关键: 元学习用查询训练,ML 用支持训练

- **MAML 模型不可知元学习 (LeeDL)**
  - When to use: 学好的初始化参数(小样本学习)
  - How:
    - 初始化参数 θ⁰ 可学(不随机)
    - **最大化损失对超参数的敏感性**: 让小步更新就能降低损失
    - 两次求梯度:
      1. 内层: 每个任务算 ∇L,梯度下降得 θ^n*
      2. 外层: 对 θ^n* 求梯度,更新 θ⁰
    - **假设**: 好的初始化让少次梯度下降即可适应新任务
  - 变体: FOMAML(一阶)、Reptile、ANIL
  - 关键: MAML = 学初始化;自监督预训练 = 用大数据学初始化,多任务学习 = 用多任务数据学初始化--三目的同,方法异

- **可学的优化器 (LeeDL)**
  - When to use: 自动调学习率/动量
  - How: 学习率等超参不用人工设,用网络学出来。代表: "Learning to learn by gradient descent by gradient descent"(NIPS 2016)
  - 关键: 优化器本身是网络,根据训练任务学

- **神经网络架构搜索 NAS (LeeDL)**
  - When to use: 自动设计网络架构
  - How:
    - φ = 网络架构(滤波器大小、步长、层数)
    - 架构不可微,用 **RL**(智能体输出架构参数,最大化准确率回报)
    - 或**进化算法**,或**可微分架构搜索**(DARTS,让架构可微)
  - 关键: NAS 是元学习的特例,φ = 架构

- **小样本学习与 N-way K-shot (LeeDL)**
  - When to use: 每类只有 K 个样例
  - How:
    - **N-way K-shot**: N 类每类 K 样例(如 20-way 1-shot)
    - 用 Omniglot(1623 字符,每字 20 样例)造任务
    - 训练任务和测试任务用不同字符集
  - 关键: 小样本学习靠元学习算法实现

- **灾难性遗忘 (LeeDL)**
  - When to use: 终身学习的核心难题
  - How:
    - 依次学任务 1 -> 任务 2 -> ... -> 任务 T
    - 学新任务时,参数移到新任务最优点,旧任务表现暴跌
    - 模型有潜力同时学好(多任务学习证明),但依次学就忘
  - 关键: 模型像"上下都接水龙头的池子"--新任务进,旧任务出

- **终身学习评估 (LeeDL)**
  - When to use: 评估终身学习算法
  - How:
    - 准确率矩阵 R_{i,j}: 学完任务 i 后在任务 j 上的准确率
    - **最终准确率**: `(1/T) Σ R_{T,i}`(学完所有任务后平均)
    - **反向迁移 BWT**: `(1/(T-1)) Σ (R_{T,i} - R_{i,i})`(学后续任务对前任务的影响,负值=遗忘)
  - 关键: 不仅看最终,还要看遗忘程度

- **选择性突触可塑性 (LeeDL)**
  - When to use: 解决灾难性遗忘(正则化法)
  - How:
    - 给每个参数 θ_i^b 赋"守卫" b_i(对旧任务的重要性)
    - 新损失: `L'(θ) = L(θ) + λ Σ b_i (θ_i - θ_i^b)²`
    - b_i = 0: 自由学(易遗忘);b_i → ∞: 完全固化(学不进新)
    - **b_i 设定**: 改变 θ_i^b 看损失变化,变化大=重要=b_i 大
  - 关键: 重要参数不动,次要参数自由适应新任务

- **梯度回合记忆 GEM (LeeDL)**
  - When to use: 基于梯度的终身学习
  - How:
    - 不仅算当前任务梯度,还算历史任务梯度
    - 向量求和得实际梯度方向
    - 约束: 新梯度方向不与历史梯度冲突(点积 ≥ 0)
  - 关键: 存历史梯度(少量信息,可接受)

## Key Concepts

- **元学习 (meta learning)**: 学习如何学习
- **学习算法 F_φ**: 输入训练数据,输出模型
- **可学参数 φ**: 初始化/学习率/架构
- **跨任务训练**: 元学习训练单位
- **内循环**: 单任务训练
- **外循环**: 跨任务训练
- **支持 (support)**: 任务内的训练数据
- **查询 (query)**: 任务内的测试数据
- **回合 (episode)**: 一次训练 + 一次测试
- **MAML**: 模型不可知元学习,学初始化
- **Reptile**: MAML 变体
- **FOMAML**: 一阶 MAML
- **NAS**: 神经网络架构搜索
- **DARTS**: 可微分架构搜索
- **多任务学习**: 所有任务数据一起训
- **N-way K-shot**: N 类每类 K 样例
- **小样本学习 (few-shot)**: 少样例学新任务
- **Omniglot**: 手写字符元学习基准
- **终身学习 (LLL)**: 持续学多任务
- **灾难性遗忘**: 学新忘旧
- **选择性突触可塑性**: 重要参数固化
- **守卫 b_i**: 参数重要性系数
- **GEM**: 梯度回合记忆
- **反向迁移 (BWT)**: 学新任务对旧任务的影响
- **learning to compare**: 基于度量的元学习

## Mental Models

- **元学习 = 学学习算法**: ML 学分类器,元学习学"如何学分类器"
- **训练任务 = 训练数据**: 元学习的数据是任务,不是样本
- **查询训练 = 测试当训练**: 训练任务的查询集可用于训练(因训练单位是任务)
- **MAML = 找好起点**: 好初始化让少步梯度下降就能适应新任务
- **MAML vs 自监督**: MAML 用任务学,自监督用数据学,多任务用混合数据学--都为好初始化
- **外循环 = 元更新**: 更新 φ(算法);**内循环 = 任务更新**: 更新 θ(模型)
- **灾难性遗忘 = 池子漏水**: 新任务进,旧任务出
- **多任务学习 = 上界**: 一起学不会忘,但需重训所有
- **守卫 = 重要参数锁**: 重要参数加锁,次要参数自由
- **b_i = 参数重要性**: 改变参数对旧任务损失的影响

## Anti-patterns

- **MAML 内循环太多步**: 计算量大,易过拟合任务。**1-5 步即可**
- **训练任务太少**: 元学习过拟合。**收集更多训练任务或数据增强**
- **训练任务和测试任务字符重叠**: 评估不公平。**严格划分字符集**
- **NAS 用 RL 训不稳定**: 架构搜索空间大。**用 DARTS 让可微**
- **b_i 全设大**: 学不进新任务。**按重要性差异化**
- **b_i 全设 0**: 灾难性遗忘。**重要参数必须大**
- **GEM 存所有历史数据**: 违背终身学习初衷。**只存梯度信息**
- **期望终身学习超多任务学习**: 多任务是上界。**先跑多任务做基准**
- **小样本只用 MAML**: 可能学不到通用表示。**结合预训练**

## Code Examples

**MAML 简化实现**:

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class MAML:
    def __init__(self, model, lr_inner=0.01, lr_outer=0.001, n_inner_steps=1):
        self.model = model  # 基模型(参数 θ⁰)
        self.lr_inner = lr_inner
        self.lr_outer = lr_outer
        self.n_inner_steps = n_inner_steps
        self.optimizer = torch.optim.Adam(model.parameters(), lr=lr_outer)
    
    def inner_update(self, support_x, support_y):
        """内循环: 在支持集上做几步梯度下降"""
        fast_weights = list(self.model.parameters())
        for _ in range(self.n_inner_steps):
            loss = F.cross_entropy(self.model.forward_with(support_x, fast_weights), support_y)
            grads = torch.autograd.grad(loss, fast_weights, create_graph=True)
            fast_weights = [w - self.lr_inner * g for w, g in zip(fast_weights, grads)]
        return fast_weights
    
    def train_step(self, tasks):
        """外循环: 在一批任务上更新 θ⁰"""
        meta_loss = 0
        for support_x, support_y, query_x, query_y in tasks:
            # 内循环: 在支持集上得到任务专属参数
            fast_weights = self.inner_update(support_x, support_y)
            # 用查询集算损失(二阶梯度)
            query_pred = self.model.forward_with(query_x, fast_weights)
            meta_loss += F.cross_entropy(query_pred, query_y)
        meta_loss = meta_loss / len(tasks)
        
        self.optimizer.zero_grad()
        meta_loss.backward()
        self.optimizer.step()
        return meta_loss.item()

# 训练循环
maml = MAML(model, lr_inner=0.01, lr_outer=0.001)
for epoch in range(num_epochs):
    tasks = sample_tasks(task_distribution, batch_size=8)  # 8 个任务
    loss = maml.train_step(tasks)
```
- **What it demonstrates**: MAML = 内循环(支持集梯度下降) + 外循环(查询集损失反传到 θ⁰)

**选择性突触可塑性(EWC 简化)**:

```python
class EWC:
    """Elastic Weight Consolidation: 选择性突触可塑性"""
    def __init__(self, model, lam=1000):
        self.model = model
        self.lam = lam  # 正则强度
        self.fisher = {}  # 每个参数的重要性(Fisher 信息)
        self.old_params = {}
    
    def compute_fisher(self, dataset):
        """任务结束后计算 Fisher 信息矩阵(参数重要性)"""
        fisher = {n: 0 for n, p in self.model.named_parameters()}
        for x, y in dataset:
            self.model.zero_grad()
            loss = F.cross_entropy(self.model(x), y)
            loss.backward()
            for n, p in self.model.named_parameters():
                fisher[n] += p.grad.data ** 2 / len(dataset)
        return fisher
    
    def update_after_task(self, dataset):
        """任务结束后调用"""
        self.fisher = self.compute_fisher(dataset)
        self.old_params = {n: p.clone().detach() 
                          for n, p in self.model.named_parameters()}
    
    def penalty(self):
        """EWC 正则项: Σ b_i (θ_i - θ_i^b)²"""
        loss = 0
        for n, p in self.model.named_parameters():
            if n in self.fisher:
                loss += (self.fisher[n] * (p - self.old_params[n]) ** 2).sum()
        return self.lam * loss

# 终身学习训练
ewc = EWC(model, lam=1000)
for task_id, task_data in enumerate(tasks_sequence):
    for epoch in range(epochs_per_task):
        for x, y in task_data:
            loss = F.cross_entropy(model(x), y)
            if task_id > 0:  # 第二个任务起加正则
                loss += ewc.penalty()
            optimizer.zero_grad(); loss.backward(); optimizer.step()
    ewc.update_after_task(task_data)  # 任务结束更新 Fisher 和旧参数
```

**终身学习评估**:

```python
def evaluate_lifelong(model, tasks):
    """返回 R_{i,j} 矩阵"""
    T = len(tasks)
    R = torch.zeros(T, T)
    for i, task_i in enumerate(tasks):
        # 在任务 i 上训练
        train(model, task_i)
        # 在所有任务上测试
        for j, task_j in enumerate(tasks):
            R[i, j] = test(model, task_j)
    
    # 最终准确率
    final_acc = R[T-1, :].mean().item()
    # 反向迁移
    bwt = (R[T-1, :-1] - torch.diag(R)[:-1]).mean().item()
    return R, final_acc, bwt
```

## Worked Example

**为什么 MAML 能少样本学习?** -- 20-way 1-shot Omniglot 案例。

**情景**: 20 个字符类,每类 1 张图(共 20 张支持),要分类新查询图。

**普通 ML 训**:
- 20 张图训分类器: 严重过拟合(数据太少)
- 测试准确率低

**MAML 训练阶段**:
1. 准备大量训练任务(不同字符集,每任务 20-way 1-shot)
2. 对每个任务:
   - 用支持集(20 张)做 1 步梯度下降: θ⁰ -> θ'
   - 用查询集算 loss(θ')
3. 反传 query loss 到 θ⁰(外循环)

**MAML 测试阶段**:
- 新字符集的 20-way 1-shot 任务
- 用支持集做 1-5 步梯度下降: θ⁰ -> θ'
- 在查询集上准确率高

**为什么有效?**
- **假设 1**: θ⁰ 是"敏感"的初始化--小步更新就能降损失
- **假设 2**: θ⁰ 接近所有任务的最优点--少步即达
- **类比**: 自监督预训练找好初始化(用大数据),MAML 找好初始化(用大任务)

**MAML vs 多任务学习**:
- 多任务: 所有任务数据混合训,得 θ^MT
- MAML: 任务分开,找对**少步适应**最敏感的 θ⁰
- 多任务基线常用作 MAML 上界

**灾难性遗忘案例**:
- 任务 1(数字识别加噪)训完: 任务 1 = 90%, 任务 2(数字无噪) = 96%
- 再训任务 2: 任务 1 = 80%(忘), 任务 2 = 97%
- **多任务同训**: 任务 1 = 90%, 任务 2 = 97%(无遗忘)
- **池子比喻**: 新任务从上进,旧任务从下出--学不进多任务

**EWC 解决方案**:
- 任务 1 训完后,算 Fisher 信息: 哪些参数对任务 1 重要
- 训任务 2 时,损失加 `λΣ b_i (θ_i - θ_i^b)²`
- 重要参数(b_i 大)几乎不动,自由参数(b_i 小)适应任务 2
- 结果: 任务 1 准确率保持,任务 2 也能学

**关键洞察**:
1. **元学习训练单位是任务**: 用任务训练学习算法,而非样本训模型
2. **MAML 内外循环**: 内=任务适应,外=算法优化
3. **好初始化 = 少步适应**: MAML/自监督/多任务都为找好起点
4. **灾难性遗忘是参数冲突**: 任务 2 把参数从任务 1 最优点拉走
5. **守卫保护重要参数**: b_i 大的参数不被新任务改变
6. **多任务是上界**: 终身学习永远比多任务差(数据不全)
7. **反向迁移为负 = 遗忘**: BWT < 0 表示学新忘旧

## Key Takeaways

1. 元学习 = "学习如何学习",找学习算法 F_φ 而非模型 f
2. 元学习三步骤: 可学参数 φ(初始化/学习率/架构)、损失 L(φ) = Σ l^n、优化
3. **训练单位是任务**: 训练任务有支持+查询,查询用于训练(与 ML 不同)
4. **MAML** 学好的初始化,让少步梯度下降适应新任务--小样本学习核心
5. MAML 假设: 好初始化敏感(小步降损)且接近所有任务最优
6. NAS 是元学习特例,φ = 网络架构,用 RL/进化/DARTS 优化
7. N-way K-shot 是小样本基准,Omniglot 是经典数据集
8. 终身学习面临**灾难性遗忘**: 学新任务参数移走,旧任务表现暴跌
9. 多任务学习是终身学习上界(无遗忘),但需重训所有数据
10. **选择性突触可塑性(EWC)**: 守卫 b_i 保护重要参数,`L' = L + λΣb_i(θ_i - θ_i^b)²`
11. **GEM** 用历史梯度约束新梯度方向,避免冲突
12. 终身学习评估: 最终准确率 + 反向迁移(BWT < 0 = 遗忘)
