# Chapter 17: 自监督学习与表示学习

## Core Idea
**自监督学习 (SSL)** 用数据自身构造标签(无需人工标注)预训练模型。**BERT** 做"填空题"(掩蔽语言模型),**GPT** 做"完形续写"(预测下一词),都从海量无标注文本学到通用语言表示。预训练后,只需少量标注数据微调即可适配下游任务,**比从头训少用 100× 数据**。更老的**自编码器**把图/文压成低维嵌入再还原,是 SSL 的早期形态。**核心动机**: 嵌入(向量)承载语义--意思相近的词/句嵌入相近;上下文不同,同词嵌入不同(BERT 解决了 word2vec 静态嵌入的局限)。

## Frameworks Introduced

- **自监督学习范式 (LeeDL)**
  - When to use: 大量无标注数据,想训通用表示
  - How: 把数据 x 拆成 x' (输入) 和 x'' (标签),让模型从 x' 预测 x''。例: 文本填空、下一词预测、图像掩蔽
  - 关键: 无需标注;预训练后微调到下游任务

- **BERT: 掩蔽语言模型 (LeeDL)**
  - When to use: NLP 理解任务(分类、标注、问答)
  - How:
    - 输入文本,随机掩蔽 15% 词元(80% `<MASK>` + 10% 随机词 + 10% 不变)
    - BERT (Transformer 编码器) 输出每个位置的嵌入
    - 掩蔽位置的嵌入经线性层 + softmax 预测原词
    - 损失: 交叉熵(分类问题,类数=词表大小)
  - 附: 下一句预测 NSP 被发现效果有限(RoBERTa),可省略

- **BERT 微调四情况 (LeeDL)**
  - When to use: 把 BERT 适配下游任务
  - How:
    - **情况 1 - 单序列分类**: `[CLS] 序列 [SEP]` -> 用 `[CLS]` 输出 -> 线性层 -> 类别。例: 情感分析
    - **情况 2 - 序列标注**: 每位置输出 -> 共享线性层 -> 每位置标签。例: 词性标注
    - **情况 3 - 序列对分类**: `[CLS] 句1 [SEP] 句2 [SEP]` -> `[CLS]` 输出 -> 类别。例: NLI
    - **情况 4 - 抽取式问答**: `[CLS] 问题 [SEP] 文章 [SEP]`,训两个向量(起始/结束),与文章每位置嵌入做内积 + softmax 找答案片段

- **GPT: 自回归语言模型 (LeeDL)**
  - When to use: 文本生成任务(GPT 系列颠覆了 NLP)
  - How:
    - 给前缀词元,预测下一词元 `<BOS> -> "深" -> "度" -> "学" -> "习"`
    - GPT = Transformer 解码器(掩蔽自注意力,只看左侧)
    - 损失: 交叉熵(下一词预测)
    - **生成能力**: 自回归逐词生成,可写完整文章

- **GPT 的 in-context learning (LeeDL)**
  - When to use: GPT 太大,微调都困难时
  - How:
    - **Zero-shot**: 只给任务描述,直接让 GPT 输出
    - **One-shot**: 给 1 个范例 + 新问题
    - **Few-shot**: 给 K 个范例 + 新问题
    - 完全无梯度下降,通过 prompt 上下文"学习"
  - 关键: GPT-3 (1750 亿参数)展示 few-shot 能力,开启 prompt 工程时代

- **BERT vs GPT (LeeDL)**
  - When to use: 选预训练模型
  - How:
    - **BERT**: Transformer 编码器,双向,适合**理解**任务(分类、QA、NLI)
    - **GPT**: Transformer 解码器,单向,适合**生成**任务(对话、写作、翻译)
    - **T5**: 编码器+解码器,统一为 text-to-text 格式

- **自编码器 (LeeDL)**
  - When to use: 无监督学特征表示
  - How:
    - **编码器**: 把图/文压成低维嵌入(瓶颈)
    - **解码器**: 从嵌入还原原数据
    - 损失: 重构误差 `||x - D(E(x))||`
    - **无监督**: 不需标注,只是重构
  - 关键: 嵌入是高维数据的低维压缩表示

- **去噪自编码器 (LeeDL)**
  - When to use: 学鲁棒表示
  - How: 给输入加噪声,要求解码器还原**原始**(未加噪)输入。强迫编码器学去噪
  - **BERT 本质是去噪自编码器**: 掩蔽 = 加噪,预测原词 = 还原

- **特征解耦 (LeeDL)**
  - When to use: 想分离嵌入中的不同信息(如内容 vs 说话人)
  - How: 强制嵌入某些维度编码某种信息。例: 语音转换--前 N 维内容、后 N 维语者,拼接 A 内容 + B 语者 -> 解码器生成 B 说 A 的话

- **VQ-VAE 离散隐表征 (LeeDL)**
  - When to use: 想要离散的嵌入(可解释、可分类)
  - How: 嵌入不是连续向量,而是从**码本**中选最相似的离散向量。码本学出来后,每个向量对应一个"基本单元"(如音素、笔画)

- **多语言 BERT (LeeDL)**
  - When to use: 跨语言任务
  - How: 用 104 种语言联合训 BERT,学到语言无关的嵌入。**神奇现象**: 英文问答微调后,可直接做中文问答(零样本跨语言迁移)
  - 关键: 不同语言同义词汇的嵌入被拉到接近

## Key Concepts

- **自监督学习 (self-supervised learning)**: 数据自身造标签
- **预训练 (pre-training)**: 自监督学通用表示
- **微调 (fine-tuning)**: 预训练模型适配下游任务
- **下游任务 (downstream task)**: 实际关心的任务
- **掩蔽语言模型 (MLM)**: BERT 的填空任务
- **下一句预测 (NSP)**: BERT 的句子关系任务(已被 RoBERTa 弃用)
- **自回归语言模型**: GPT 的下一词预测
- **语境化嵌入 (contextualized embedding)**: 同词不同上下文不同向量
- **嵌入 (embedding)**: 数据的低维稠密表示
- **瓶颈 (bottleneck)**: 自编码器的低维中间层
- **重构 (reconstruction)**: 还原原始输入
- **去噪自编码器 (denoising AE)**: 加噪后还原原始
- **特征解耦 (feature disentanglement)**: 嵌入不同维度编码不同信息
- **VQ-VAE**: 离散码本嵌入
- **in-context learning**: GPT 通过 prompt 上下文学任务
- **zero/one/few-shot learning**: 0/1/K 个范例学习
- **GLUE**: NLP 通用语言理解评估基准(9 任务)
- **SUPERB**: 语音版 GLUE(10 任务)
- **多语言 BERT**: 跨语言预训练模型

## Mental Models

- **预训练 = 胚胎干细胞**: BERT 学填空后能分化到任何任务(情感分析、QA、NLI...),像干细胞分化为各种细胞
- **嵌入 = 语义坐标**: 意思相近的词嵌入相近;BERT 让同词在不同上下文有不同嵌入(消歧)
- **上下文 = 词义**: Firth 1957: "词义由其上下文决定"。BERT 通过填空学到上下文,即学到词义
- **GPT = 完形续写**: 学会预测下一词 = 学会语言规律 = 会写文章
- **自编码器 = 压缩 + 解压**: 编码器压缩,解码器还原;瓶颈学到的低维向量是数据本质特征
- **去噪 AE = 学鲁棒特征**: 强迫模型忽略噪声,学到本质
- **特征解耦 = 拆解纠缠**: 把"内容+语者"拆开,可重组(A 说 B 的话)
- **GPT prompt = 任务说明书**: GPT 太大无法微调,用 prompt 文字告诉它做什么

## Anti-patterns

- **小数据训 BERT/GPT**: 30 亿词才够,小数据用预训练模型微调即可
- **NSP 任务**: 已被 RoBERTa 证明无效。**省略 NSP,只用 MLM**
- **BERT 输入超 512**: 自注意力 O(n²),实际限制 512。长文要滑窗或 Longformer
- **BERT 用于生成**: 编码器不能自回归。**用 GPT/BART/T5**
- **GPT 用于双向理解**: 单向看不到右侧。**用 BERT**
- **静态嵌入处理多义**: word2vec 一个词一个向量。**用 BERT 语境化嵌入**
- **自编码器瓶颈太宽**: 编码器输出维度过高 = 没压缩 = 直接复制。**瓶颈要远小于输入**
- **自编码器不约束嵌入**: 信息全纠缠,无法解耦。**加解耦损失或 VAE**
- **GPT 微调**: 大模型微调成本高。**优先用 prompt/in-context learning**
- **跨语言零迁移期望过高**: 多语言 BERT 中文问答 F1=78%(英文微调),仍不如中文专训(89%)

## Code Examples

**BERT 掩蔽语言模型训练**:

```python
import torch
from torch import nn

class BERTMLM(nn.Module):
    def __init__(self, vocab_size, hidden=768):
        super().__init__()
        self.encoder = BERTEncoder(vocab_size, hidden)  # Transformer 编码器
        self.mlm_head = nn.Sequential(
            nn.Linear(hidden, hidden), nn.GELU(),
            nn.LayerNorm(hidden),
            nn.Linear(hidden, vocab_size)
        )
    
    def forward(self, tokens, segments, mask_positions):
        # 1. BERT 编码
        encoded = self.encoder(tokens, segments, None)  # (B, L, H)
        # 2. 取掩蔽位置的输出
        # mask_positions: (B, n_mask)
        batch_size = encoded.size(0)
        batch_idx = torch.arange(batch_size).repeat_interleave(
            mask_positions.size(1))
        masked = encoded[batch_idx, mask_positions.reshape(-1)]
        masked = masked.reshape(batch_size, -1, encoded.size(-1))
        # 3. 预测原词
        logits = self.mlm_head(masked)  # (B, n_mask, vocab_size)
        return logits

# 训练
model = BERTMLM(vocab_size=30000)
loss = nn.CrossEntropyLoss()
# 假设已构造: tokens(掩蔽后), mask_positions, mask_labels(原词)
logits = model(tokens, segments, mask_positions)
loss_value = loss(logits.reshape(-1, vocab_size), mask_labels.reshape(-1))
```

**GPT 自回归训练**:

```python
class GPT(nn.Module):
    def __init__(self, vocab_size, hidden=768, n_layers=12, n_heads=12):
        super().__init__()
        self.token_emb = nn.Embedding(vocab_size, hidden)
        self.pos_emb = nn.Embedding(1024, hidden)
        # Transformer 解码器 (掩蔽自注意力, 无交叉注意力)
        self.blocks = nn.Sequential(*[
            DecoderBlock(hidden, n_heads) for _ in range(n_layers)
        ])
        self.ln_f = nn.LayerNorm(hidden)
        self.head = nn.Linear(hidden, vocab_size, bias=False)
    
    def forward(self, tokens):
        B, L = tokens.shape
        pos = torch.arange(L).unsqueeze(0).expand(B, L)
        x = self.token_emb(tokens) + self.pos_emb(pos)
        x = self.blocks(x)
        x = self.ln_f(x)
        return self.head(x)  # (B, L, vocab_size)

# 训练: 输入 [BOS, w1, w2, ..., w_{n-1}], 目标 [w1, w2, ..., w_n]
logits = gpt(input_tokens)
loss = nn.functional.cross_entropy(
    logits.reshape(-1, vocab_size), 
    target_tokens.reshape(-1)
)
```
- **What it demonstrates**: GPT 自回归训练 = 输入左移一位,目标右移一位,逐词预测

**自编码器**:

```python
class Autoencoder(nn.Module):
    def __init__(self, input_dim=784, latent_dim=32):
        super().__init__()
        # 编码器: 784 -> 32
        self.encoder = nn.Sequential(
            nn.Linear(input_dim, 256), nn.ReLU(),
            nn.Linear(256, 128), nn.ReLU(),
            nn.Linear(128, latent_dim)
        )
        # 解码器: 32 -> 784
        self.decoder = nn.Sequential(
            nn.Linear(latent_dim, 128), nn.ReLU(),
            nn.Linear(128, 256), nn.ReLU(),
            nn.Linear(256, input_dim), nn.Sigmoid()
        )
    
    def forward(self, x):
        z = self.encoder(x)        # 嵌入 (压缩表示)
        x_recon = self.decoder(z)  # 重构
        return x_recon, z

# 训练
ae = Autoencoder()
loss = nn.MSELoss()
x_recon, z = ae(x)
loss_value = loss(x_recon, x)  # 重构误差
```

**去噪自编码器**:

```python
def train_denoising_ae(ae, x, noise_factor=0.3):
    # 加噪声
    noise = torch.randn_like(x) * noise_factor
    x_noisy = (x + noise).clamp(0, 1)
    # 重构目标是无噪声原图
    x_recon, _ = ae(x_noisy)
    return nn.functional.mse_loss(x_recon, x)
```

**BERT 微调做分类**:

```python
class BERTClassifier(nn.Module):
    def __init__(self, bert, num_classes):
        super().__init__()
        self.bert = bert
        self.classifier = nn.Linear(bert.hidden_size, num_classes)
    
    def forward(self, tokens, segments):
        encoded = self.bert(tokens, segments, None)
        cls_repr = encoded[:, 0, :]  # [CLS] 位置输出代表整句
        return self.classifier(cls_repr)
```

## Worked Example

**为什么 BERT 嵌入能消歧?** -- "苹果"案例。

考虑两个句子:
- 句 1: "今天买了苹果吃" (水果)
- 句 2: "苹果即将在下个月发布新 iPhone" (公司)

**静态嵌入(word2vec)**:
- 两个"苹果"映射到**同一向量**(因为词相同)
- 无法区分水果 vs 公司

**BERT 语境化嵌入**:
- 句 1 的"苹果": 经过自注意力,看到"买"、"吃"等词,生成嵌入接近"水果"、"食物"
- 句 2 的"苹果": 看到"发布"、"iPhone"等词,生成嵌入接近"公司"、"科技"
- **同词不同嵌入**,因为上下文不同

**LeeDL 实验数据**:
- 收集 10 个含"苹果"的句子(5 水果 + 5 公司)
- 算 10 个"苹果"嵌入两两余弦相似度
- 结果: 同类(水果-水果、公司-公司)相似度高;异类(水果-公司)相似度低
- BERT 真的"理解"了"苹果"在不同上下文的不同含义

**关键洞察**:
1. **Firth 假设(1957)**: "词义由上下文决定" -- 经典语言学
2. **BERT 学填空 = 学提取上下文**: 掩蔽"苹果"后预测它,模型必须从上下文推断词义
3. **嵌入 = 上下文的精华**: BERT 输出的嵌入融合了上下文信息,自然消歧
4. **跨语言迁移**: 多语言 BERT 把"rabbit"和"兔子"的嵌入拉到接近(同义),所以英文问答可迁移到中文
5. **不只是语言**: BERT 训练在文字上,可用于 DNA/蛋白质/音乐分类--说明它学到的不只是"语义",更是"序列结构"

**BERT vs 从头训对比**(LeeDL 实验):
- 微调(预训练初始化): loss 快速下降,训练几轮达高精度
- 从头训(随机初始化): loss 下降慢,最终精度低
- **预训练 = 好的初始化**,即使下游任务与填空不同也有效

## Key Takeaways

1. 自监督学习用数据自身造标签,无需人工标注,可利用海量无标注数据
2. **BERT = Transformer 编码器 + MLM(填空)**,双向,适合理解任务
3. **GPT = Transformer 解码器 + 自回归**,单向,适合生成任务
4. 预训练 + 微调范式: 用大量无标注数据预训练,小标注数据微调,效果远超从头训
5. BERT 微调四情况: 单序列分类、序列标注、序列对分类、抽取式问答--只需不同输出层
6. **语境化嵌入**: BERT 让同词在不同上下文有不同嵌入,解决了 word2vec 静态嵌入的多义问题
7. GPT-3 大到无法微调,通过 **in-context learning**(prompt + 范例)适配任务,开启 prompt 工程时代
8. 自编码器 = 编码器 + 解码器 + 重构损失,无监督学特征;**BERT 本质是去噪自编码器**
9. 特征解耦可分离嵌入中的不同信息(内容/语者),实现语音转换等
10. VQ-VAE 用离散码本,可学出基本单元(音素、笔画)
