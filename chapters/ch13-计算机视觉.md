# Chapter 13: 计算机视觉

## Core Idea
图像分类只是 CV 入门。真实 CV 任务主要有三类: ① **目标检测**--找出图中所有物体的位置和类别(用边界框) ② **语义分割**--对每个像素分类 ③ **风格迁移**--把一张图的内容用另一张图的风格重绘。这些任务建立在 CNN backbone 之上,加上**图像增广**(扩展数据)、**微调**(迁移学习)、**锚框/IoU/NMS**(检测三件套)、**转置卷积**(上采样)等关键技术。SSD/R-CNN 系列是检测的代表,FCN 是分割的代表。

## Frameworks Introduced

- **图像增广 (Image Augmentation) (d2l)**
  - When to use: 训练数据不足、模型过拟合时
  - How: 对训练图像随机变换(翻转、裁剪、改颜色)生成"新"样本。**测试时不增广**(或只做标准化中心裁剪)。`torchvision.transforms.Compose` 串联多种增广

- **微调 / 迁移学习 (Fine-tuning) (d2l)**
  - When to use: 目标数据集远小于源数据集(如 ImageNet 预训练 -> 自定义小数据集)
  - How:
    1. 在源数据集(ImageNet)预训练模型
    2. 复制除输出层外的所有参数到目标模型
    3. 新建输出层(类数=目标数据集类数),随机初始化
    4. 在目标数据集上训练,**输出层用更大学习率**(10×),其他层用小学习率
  - 直觉: 浅层学的是通用特征(边缘、纹理),深层学的是任务相关特征;通用特征可迁移

- **边界框 (Bounding Box) (d2l)**
  - When to use: 目标检测的标注与预测
  - How: 用 `(左上 x, 左上 y, 右下 x, 右下 y)` 四个数表示一个矩形框。`box_corner_to_center` 与 `box_center_to_corner` 在两种格式间转换(中心+宽高 vs 两角)

- **锚框 (Anchor Box) (d2l)**
  - When to use: 目标检测的标准做法--以每个像素为中心预设大量框,模型预测每个框的类别+偏移
  - How: 对特征图每个像素,以 `sizes`(尺度) × `ratios`(宽高比)生成 a 个锚框。`multibox_prior` 一次生成所有锚框

- **交并比 IoU (d2l)**
  - When to use: 衡量两个边界框的重合度
  - How: `IoU(A, B) = A∩B 面积 / A∪B 面积`。0 表示不重叠,1 表示完全重合。**IoU ≥ 0.5 通常算正确检测**

- **非极大值抑制 NMS (d2l)**
  - When to use: 检测后大量锚框重叠,需合并属于同一目标的框
  - How:
    1. 按置信度(预测类别的最大概率)降序排所有非背景框
    2. 取置信度最高的框 B₁为基准,移除与 B₁ 的 IoU > 阈值(常用 0.5)的其他框
    3. 取次高的 B₂ 重复,直到所有框都做过基准
    4. 输出剩余框

- **多尺度目标检测 (d2l)**
  - When to use: 图中物体大小差异大
  - How: 在不同层级的特征图上生成锚框--浅层特征图大、感受野小,适合小物体;深层特征图小、感受野大,适合大物体。SSD 用 5 个尺度的特征图

- **SSD (Single Shot MultiBox Detector) (d2l)**
  - When to use: 端到端快速目标检测
  - How:
    - 基础网络(ResNet/VGG 截断)+ 多个高宽减半块,得 5 个尺度特征图
    - 每尺度: 类别预测层(`a(q+1)` 通道的 3×3 卷积) + 边界框预测层(`4a` 通道的 3×3 卷积)
    - 拼接所有尺度的预测,**用卷积通道做预测避免全连接参数爆炸**
    - 训练: 损失 = 分类损失(交叉熵) + 边界框损失(平滑 L1)

- **R-CNN 系列 (d2l)**
  - When to use: 理解检测网络演进
  - How:
    - **R-CNN**: 选择性搜索出 2000 个提议区域 -> 每个独立过 CNN -> SVM 分类 + 回归边界框。慢
    - **Fast R-CNN**: 整张图过 CNN 一次 -> 用 **RoI Pooling** 从特征图抽每个提议区域的定长特征 -> 分类+回归。快 100×
    - **Faster R-CNN**: 用 **区域提议网络 (RPN)** 替代选择性搜索,端到端训练。快 10×
    - **Mask R-CNN**: 在 Faster R-CNN 上加全卷积分支,做像素级掩码。RoI Align 替代 RoI Pooling(双线性插值保留空间精度)

- **语义分割 (Semantic Segmentation) (d2l)**
  - When to use: 像素级分类(如自动驾驶场景理解)
  - How: 对每个像素预测类别。**不同于目标检测用框,分割要逐像素**

- **转置卷积 (Transposed Convolution) (d2l)**
  - When to use: 上采样--放大特征图空间尺寸
  - How: 普通卷积把"输入"乘核加到"输出"一格;转置卷积把"输入的每个元素"乘核广播到"输出"的 k×k 区域,所有广播结果相加。**输入 n_h×n_w,输出 (n_h+k-1) × (n_w+k-1)**(步幅 1 无填充)。命名来自矩阵视角: 普通卷积 y = Wx,转置卷积 y = Wᵀx

- **全卷积网络 FCN (d2l)**
  - When to use: 端到端语义分割
  - How:
    1. 用 CNN backbone(如 ResNet 截断到全局池化前)提取特征,空间下采样 32×
    2. 1×1 卷积把通道数变成类别数
    3. 转置卷积(步幅 32,核 64,填充 16)上采样回原图尺寸
    4. 用双线性插值初始化转置卷积核
    5. 损失: 每像素交叉熵的平均

- **风格迁移 (Style Transfer) (d2l)**
  - When to use: 把内容图用风格图的画风重绘
  - How: 训练一个合成图(初始为内容图或噪声),损失 = 内容损失(合成图与内容图在 VGG 中层特征上的差)+ 风格损失(合成图与风格图在 Gram 矩阵上的差)+ 全变分损失(平滑约束)。**不训练网络权重,训练像素值**

## Key Concepts

- **图像增广 (augmentation)**: 随机变换扩充训练数据
- **微调 (fine-tuning)**: 预训练模型参数迁移到新任务
- **迁移学习 (transfer learning)**: 跨数据集复用知识
- **边界框 (bounding box)**: 矩形定位框
- **锚框 (anchor/prior box)**: 预设的候选框
- **交并比 (IoU)**: 两框重合度
- **非极大值抑制 (NMS)**: 合并重复检测
- **多尺度检测**: 不同层级特征图检测不同大小物体
- **SSD**: 单发多框检测,一次前向出所有框
- **RoI Pooling / Align**: 从特征图抠出定长特征
- **RPN (区域提议网络)**: 学习生成提议区域
- **语义分割**: 像素级分类
- **实例分割**: 区分不同实例的像素级分割
- **转置卷积**: 上采样卷积
- **全卷积网络 (FCN)**: 全卷积,输出与输入同尺寸
- **双线性插值 (bilinear interpolation)**: 上采样方法,也用于初始化转置卷积
- **Gram 矩阵**: 风格迁移中度量特征相关性的矩阵

## Mental Models

- **增广是"免费数据"**: 翻转/裁剪/改色等价于告诉模型"物体长这样、转一下也长这样"--注入不变性先验
- **微调 = 浅层通用 + 深层专用**: 浅层学的边缘/纹理跨任务通用,直接复用;深层学的语义需在新数据上微调;输出层一定从头训
- **锚框 = 穷举候选位置**: 模型不再"想"物体在哪,而是给每个位置一堆预设框,只预测"这框里是不是物体、是的话偏移多少"
- **NMS = 一物一框**: 同一目标被多个锚框检测到时,只留置信度最高的,其他抑制
- **多尺度 = 金字塔**: 浅层特征图大但感受野小,适合小物体;深层相反--不同层负责不同大小
- **转置卷积是"反卷积"**: 普通卷积压缩空间,转置卷积放大空间;命名来自矩阵的转置
- **FCN = 编码器-解码器**: CNN 把空间信息压缩成通道语义,转置卷积把语义"展开"回像素级

## Anti-patterns

- **测试时用随机增广**: 测试必须确定性的(中心裁剪 + 标准化)。**训练增广、测试不增广**
- **微调时所有层同一学习率**: 输出层随机初始化,需要 10× 学习率;预训练层用小 lr(1e-4 ~ 5e-5)
- **IoU 阈值设过低**: 0.3 算正确,会高估精度。**常用 0.5**
- **NMS 阈值设过低**: 阈值 0.3 会把同物的多框都干掉。**常用 0.5**
- **检测数据集不平衡**: 背景框远多于物体框,训练时要负样本采样或加 focal loss
- **分割任务用普通卷积上采样**: 普通卷积降尺寸,要做分割必须用转置卷积或反池化
- **转置卷积核学不好**: 用双线性插值初始化,否则初期可能学到棋盘伪影
- **小数据集从头训 CNN**: 必然过拟合。**用预训练 + 微调**
- **风格迁移不约束全变分**: 输出会噪点满满。**加 TV loss 平滑**

**微调预训练模型**:

```python
import torchvision
from torch import nn
import torch.nn.init as init

# 1. 加载预训练 ResNet-18
finetune_net = torchvision.models.resnet18(pretrained=True)
# 2. 替换输出层 (类数=2, 热狗识别)
finetune_net.fc = nn.Linear(finetune_net.fc.in_features, 2)
init.xavier_uniform_(finetune_net.fc.weight)

# 3. 训练时: 预训练层小 lr, 新输出层大 lr (10x)
def train_fine_tuning(net, lr, num_epochs=5):
    params_1x = [p for name, p in net.named_parameters() 
                 if name not in ["fc.weight", "fc.bias"]]
    trainer = torch.optim.SGD([
        {'params': params_1x},                              # 预训练层小 lr
        {'params': net.fc.parameters(), 'lr': lr * 10}      # 新层大 lr
    ], lr=lr, weight_decay=0.001)
    # ... 标准 train loop
```
- **What it demonstrates**: 微调的关键是分层学习率--预训练层用 5e-5,新输出层用 5e-4

**IoU 计算**:

```python
def box_iou(boxes1, boxes2):
    """计算两组框两两之间的 IoU
    boxes1, boxes2: (N, 4), (M, 4) -- 左上右下格式
    返回: (N, M)
    """
    # 计算面积
    area1 = (boxes1[:, 2] - boxes1[:, 0]) * (boxes1[:, 3] - boxes1[:, 1])
    area2 = (boxes2[:, 2] - boxes2[:, 0]) * (boxes2[:, 3] - boxes2[:, 1])
    
    # 交集左上 = max(boxes1.lt, boxes2.lt); 右下 = min(boxes1.rb, boxes2.rb)
    lt = torch.max(boxes1[:, None, :2], boxes2[:, :2])  # (N, M, 2)
    rb = torch.min(boxes1[:, None, 2:], boxes2[:, 2:])  # (N, M, 2)
    
    inter = (rb - lt).clamp(min=0).prod(dim=-1)  # (N, M)
    union = area1[:, None] + area2 - inter
    return inter / union
```

**NMS**:

```python
def nms(boxes, scores, iou_threshold):
    """非极大值抑制
    boxes: (N, 4), scores: (N,)
    返回: 保留的索引
    """
    B = torch.argsort(scores, descending=True)
    keep = []
    while B.numel() > 0:
        i = B[0]
        keep.append(i)
        if B.numel() == 1: break
        iou = box_iou(boxes[i].reshape(-1, 4), boxes[B[1:]].reshape(-1, 4)).reshape(-1)
        inds = torch.nonzero(iou <= iou_threshold).reshape(-1)
        B = B[inds + 1]
    return torch.tensor(keep)
```

**FCN 分割模型**:

```python
import torchvision
from torch import nn

# 1. 用 ResNet-18 截断到全局池化前作为编码器
pretrained = torchvision.models.resnet18(pretrained=True)
net = nn.Sequential(*list(pretrained.children())[:-2])  # 去掉 GAP 和 FC

# 2. 1x1 卷积降到类别数
num_classes = 21
net.add_module('final_conv', nn.Conv2d(512, num_classes, kernel_size=1))

# 3. 转置卷积上采样 32 倍回原图尺寸
net.add_module('transpose_conv', nn.ConvTranspose2d(
    num_classes, num_classes, kernel_size=64, padding=16, stride=32))

# 4. 用双线性插值初始化转置卷积核 (关键!)
def bilinear_kernel(in_c, out_c, k):
    f = (k + 1) // 2
    c = f - 1 if k % 2 == 1 else f - 0.5
    og = (torch.arange(k).reshape(-1, 1), torch.arange(k).reshape(1, -1))
    filt = (1 - torch.abs(og[0] - c) / f) * (1 - torch.abs(og[1] - c) / f)
    w = torch.zeros((in_c, out_c, k, k))
    w[range(in_c), range(out_c), :, :] = filt
    return w

net.transpose_conv.weight.data.copy_(
    bilinear_kernel(num_classes, num_classes, 64))

# 损失: 每像素交叉熵
def loss(inputs, targets):
    # inputs: (B, C, H, W), targets: (B, H, W) 类索引
    return nn.functional.cross_entropy(inputs, targets, reduction='none').mean(1).mean(1)
```

## Reference Tables

**CV 任务对照**:

| 任务 | 输出 | 典型模型 | 损失 |
|---|---|---|---|
| 图像分类 | (B, C) 类概率 | ResNet, VGG | 交叉熵 |
| 目标检测 | N×(类+置信度+4 坐标) | SSD, R-CNN, YOLO | 分类+回归 |
| 语义分割 | (B, C, H, W) 像素类概率 | FCN, U-Net, DeepLab | 像素交叉熵 |
| 实例分割 | 每实例掩码 + 类 | Mask R-CNN | 分类+回归+掩码 |
| 风格迁移 | 合成图 (H, W, 3) | Gatys | 内容+风格+TV |

**目标检测算法对照**:

| 模型 | 提议区域 | 特征抽取 | 速度 | 精度 |
|---|---|---|---|---|
| R-CNN | 选择性搜索 (2000) | 每区域独立 CNN | 极慢 | 中 |
| Fast R-CNN | 选择性搜索 | 整图 CNN + RoI Pool | 中 | 高 |
| Faster R-CNN | RPN 学习 | 整图 CNN + RoI Pool | 快 | 高 |
| Mask R-CNN | RPN | + RoI Align + 掩码分支 | 中 | 高 |
| SSD | 锚框 (无提议) | 多尺度特征图直接预测 | 极快 | 中高 |
| YOLO | 网格预测 | 单次前向 | 极快 | 中 |

**微调 vs 从头训对照**:

| 维度 | 从头训 | 微调 |
|---|---|---|
| 数据需求 | 大 (≥10万) | 小 (几百-几千可用) |
| 训练时间 | 长 | 短 |
| 初始精度 | 低 | 高 (复用预训练) |
| 学习率 | 大 (1e-3) | 小 (1e-4 ~ 5e-5) + 新层 10× |
| 过拟合风险 | 高 | 低 |

**输出尺寸对照 (转置卷积)**:

| 输入 | 核 | 步幅 | 填充 | 输出 |
|---|---|---|---|---|
| 2×2 | 2 | 1 | 0 | 3×3 |
| 2×2 | 2 | 2 | 0 | 4×4 |
| 10×15 | 64 | 32 | 16 | 320×480 (32× 上采样) |

## Worked Example

**为什么微调比从头训好?** -- 看一笔账。

设目标任务: 识别 100 种椅子,每把 100 张图,共 1 万张。ImageNet 有 100 万张图、1000 类。

**从头训 ResNet-18**:
- 参数 ~11M,数据 1 万张
- 数据/参数比 = 1/1000,**严重过拟合**
- 测试准确率可能 60-70%

**微调 ResNet-18**:
- 复用 ImageNet 预训练参数(浅层: 边缘/纹理/形状检测器)
- 仅新输出层(2 个输出)从头训
- 数据/参数比 = 10000/2 = 5000,**输出层充分训练**
- 浅层参数只需小幅调整(小 lr)
- 测试准确率 90%+

**关键洞察**:
1. **CNN 浅层是通用特征检测器**: 边缘、角点、纹理在 ImageNet 上学到的,对椅子也有用
2. **预训练 = 免费的"视觉系统"**: 不用从零学边缘检测,直接复用
3. **分层学习率**: 预训练层已接近最优,只需小步微调(5e-5);新输出层从随机开始,要大步学(5e-4)
4. **数据少时微调几乎必胜**: ImageNet 学过的"形状组合"知识迁移过来,1 万张图够用了
5. **数据多时差距缩小**: 有 100 万张目标数据,从头训也能赶上,但微调仍更快收敛

**d2l 实验数据**(热狗识别):
- 微调: loss 0.22, train acc 91.5%, test acc 93.9% (lr=5e-5)
- 从头训: loss 0.37, train acc 83.9%, test acc 84.3% (lr=5e-4)

差距 10 个百分点,微调完胜。

## Key Takeaways

1. 图像增广用随机翻转/裁剪/改色扩展训练数据,**测试不增广**
2. 微调 = 复用预训练参数 + 新输出层;**新层用 10× 学习率**
3. 目标检测三件套: 锚框(穷举候选) + IoU(衡量重合) + NMS(去重)
4. SSD 用多尺度特征图,每尺度预测 a(q+1) 类别 + 4a 偏移,**端到端一次前向**
5. R-CNN 系列演进: R-CNN(慢) -> Fast(共享特征) -> Faster(学习提议) -> Mask(加分割)
6. 语义分割要逐像素分类,**转置卷积上采样**是关键工具
7. FCN = CNN 编码器 + 1×1 卷积 + 转置卷积解码器,**双线性插值初始化**转置卷积核
8. 风格迁移: 损失 = 内容(中层特征差) + 风格(Gram 矩阵差) + 全变分(平滑);**训像素不训网络**
