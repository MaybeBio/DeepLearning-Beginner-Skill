# Chapter 15: 生成模型

## Core Idea
普通网络学"输入->确定输出",但有些任务需要"创造性输出"(画图、聊天、续写)。**生成模型**给网络加一个随机噪声 z,使输出变成一个**分布**而非单一值,从而能产生多种合理结果。**GAN (生成对抗网络)**是其中最出名的: **生成器 G** 把噪声变成假图,**判别器 D** 区分真假图,两者对抗训练--G 越来越会骗,D 越来越会抓,最终 G 能画出逼真图像。后续 WGAN 解决训练不稳定,Cycle GAN 实现无成对数据的风格转换。GAN 评估用 IS/FID 而非损失。

## Frameworks Introduced

- **生成器 (Generator) (LeeDL)**
  - When to use: 任务需要"创造性"/多种可能输出(画图、对话)
  - How: 网络输入 = 原输入 x + 随机噪声 z(从简单分布如高斯采样)。`y = G(x, z)`,z 不同则 y 不同,输出变成一个分布
  - 直觉: 监督学习对"同一输入有多个合理输出"会取平均(模糊),生成模型用 z 索引不同可能

- **GAN 双网络对抗 (LeeDL)**
  - When to use: 训练无监督生成器(尤其图像)
  - How:
    - **步骤 1 - 训练 D**: 固定 G,真实图标 1,生成图标 0,训练二分类器
    - **步骤 2 - 训练 G**: 固定 D,让 G 生成图使 D 打高分(梯度上升)
    - 交替训练,G 和 D 互相促进
  - 关键: G 和 D 都是神经网络,都可微;G 通过 D 的梯度学

- **GAN 的理论 (LeeDL)**
  - When to use: 理解 GAN 训练目标
  - How: 目标是让生成分布 P_G 与真实分布 P_data 接近。判别器目标函数 `V(D, G) = E_{y~P_data}[log D(y)] + E_{y~P_G}[log(1-D(y))]` 的最大值与 JS 散度相关。MinMax 博弈: `min_G max_D V(D, G)`
  - 关键: 不需要知道 P_G/P_data 的公式,只要能采样即可

- **WGAN (LeeDL)**
  - When to use: GAN 训练不稳定(损失不收敛、模式崩塌)
  - How:
    - 问题: JS 散度在 P_G 和 P_data 不重叠时恒为 log 2,无法反映距离
    - **Wasserstein 距离(推土机距离 EMD)**: 把 P_G "推"到 P_data 的最小平均距离,即使分布不重叠也能反映差距
    - 判别器需 **1-Lipschitz**(梯度范数 ≤ 1,足够平滑),否则目标函数可无限大
    - 实现: 梯度惩罚(Improved WGAN)或谱归一化
  - 关键: WGAN 损失随训练下降,可监控训练进度

- **条件型 GAN (Conditional GAN) (LeeDL)**
  - When to use: 想控制生成内容(文字->图、图->图)
  - How:
    - 生成器输入 = 条件 x + 噪声 z,输出 y = G(x, z)
    - 判别器输入 = (x, y),输出"y 与 x 是否匹配"的分数
    - 训练数据: 成对的 (x, y) 真实样本标 1,生成样本标 0;还需"真实 y 配错 x"的负样本标 0
  - 关键: 判别器必须看条件,否则 G 学会无视条件只产生清晰图

- **Cycle GAN (LeeDL)**
  - When to use: 风格转换但**无成对数据**(如真人脸 <-> 动漫脸)
  - How:
    - 两个生成器: G_xy 把 X 域转 Y 域,G_yx 把 Y 域转回 X 域
    - 两个判别器: D_x 区分真假 X,D_y 区分真假 Y
    - **循环一致性损失**: `||G_yx(G_xy(x)) - x||` 要小,保证 G_xy 输出与输入有关
    - 可双向训练
  - 关键: 循环一致性防 G_xy 无视输入

- **GAN 评估 (LeeDL)**
  - When to use: 评估生成器质量(损失不能反映)
  - How:
    - **Inception Score (IS)**: 用 Inception 网络算生成图的类别分布。质量高 = 单图分布尖锐;多样性高 = 平均分布平坦。`IS = exp(E[KL(p(y|x) || p(y))])`
    - **FID (Fréchet Inception Distance)**: 把生成图和真实图过 Inception 取中间层特征,假设服从高斯,算两个高斯的 Fréchet 距离。**FID 越小越好**
  - 关键: FID 比 IS 更鲁棒,是当前主流

- **GAN 训练难点 (LeeDL)**
  - When to use: 训练 GAN 时诊断问题
  - How:
    - **G 和 D 必须均衡**: D 太强 -> G 学不动;G 太强 -> D 无信号
    - **模式崩塌 (mode collapse)**: G 只会生成几种图,失去多样性
    - **模式丢失 (mode dropping)**: G 生成的图覆盖不了真实数据全部模式
    - **序列生成难**: 文字 GAN 因 argmax 不可微,需强化学习(极难训)

## Key Concepts

- **生成模型 (generative model)**: 输出分布而非单一值
- **生成器 (generator G)**: 把噪声变成数据的网络
- **判别器 (discriminator D)**: 区分真假数据的网络
- **随机噪声 z**: 从简单分布(高斯/均匀)采样的输入
- **对抗训练 (adversarial training)**: G 和 D 互相博弈
- **JS 散度**: 衡量两分布相似度,不重叠时为 log 2(问题)
- **Wasserstein 距离 (推土机距离 EMD)**: 把一分布"推"到另一分布的最小平均距离
- **1-Lipschitz**: 函数梯度范数 ≤ 1,保证 WGAN 收敛
- **梯度惩罚 (gradient penalty)**: 强制判别器平滑的 WGAN 技巧
- **谱归一化 (spectral normalization)**: 另一种让判别器 1-Lipschitz 的方法
- **模式崩塌 (mode collapse)**: 生成器只会几种输出
- **模式丢失 (mode dropping)**: 多样性不足
- **条件型生成 (conditional generation)**: 给条件控制输出
- **图像翻译 (image-to-image)**: Pix2pix,图转图
- **循环一致性 (cycle consistency)**: G_yx(G_xy(x)) ≈ x
- **Inception Score (IS)**: 评估生成质量+多样性
- **FID**: Fréchet Inception Distance,主流评估指标

## Mental Models

- **生成模型 = 输出分布而非值**: z 索引不同输出,模型有"创造力"
- **GAN = 假币制造者 vs 警察**: G 是造假币的,D 是警察;两者互相升级,最终假币以假乱真
- **MinMax 博弈**: G 最小化 D 的最大识别能力;D 最大化识别能力
- **JS 散度失效**: 高维空间两低维流形几乎不重叠,JS 恒为 log 2--训练损失看不出进步
- **Wasserstein = 推土机距离**: 即使分布不重叠,推土机走的距离仍能反映远近
- **条件 GAN = G 看 x 输出,D 看 (x, y) 评分**: 防止 G 无视条件
- **Cycle GAN = 双向 + 循环一致**: 没有成对数据,靠"能转回来"约束
- **GAN 评估不能看损失**: 必须用 IS/FID 等基于分类器的指标

## Anti-patterns

- **D 太强**: G 梯度消失学不动。**降低 D 学习率或减少 D 训练次数**
- **G 太强**: D 完全失效,无信号。**增强 D 或加正则**
- **JS 散度训练**: 用原 GAN 损失无法监控训练。**改用 WGAN**
- **判别器不平滑**: WGAN 不收敛。**必须梯度惩罚或谱归一化**
- **条件 GAN 判别器只看 y**: G 无视条件。**D 必须 (x, y) 一起看**
- **Cycle GAN 无循环一致性**: G_xy 无视输入。**必须有循环一致性损失**
- **用损失评估 GAN**: GAN 损失不反映质量。**用 FID/IS**
- **FID 过小但生成立即崩**: 检查是否过拟合训练集(直接复制)。**用训练集 vs 生成集分类器检测**
- **GAN 生成文字**: argmax 不可微。**用 VAE/扩散模型,或强化学习**
- **不存档中间模型**: 模式崩塌后无法回退。**训练时定期保存,选最佳**

## Code Examples

**GAN 基本结构**:

```python
import torch
from torch import nn

class Generator(nn.Module):
    def __init__(self, z_dim=100, img_dim=784):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(z_dim, 256), nn.LeakyReLU(0.2),
            nn.Linear(256, 512), nn.LeakyReLU(0.2),
            nn.Linear(512, img_dim), nn.Tanh()  # 输出 [-1, 1]
        )
    def forward(self, z):
        return self.net(z)

class Discriminator(nn.Module):
    def __init__(self, img_dim=784):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(img_dim, 512), nn.LeakyReLU(0.2),
            nn.Linear(512, 256), nn.LeakyReLU(0.2),
            nn.Linear(256, 1), nn.Sigmoid()  # 输出 [0, 1] 真假概率
        )
    def forward(self, x):
        return self.net(x)

G, D = Generator(), Discriminator()
opt_G = torch.optim.Adam(G.parameters(), lr=2e-4, betas=(0.5, 0.999))
opt_D = torch.optim.Adam(D.parameters(), lr=2e-4, betas=(0.5, 0.999))
criterion = nn.BCELoss()
```

**GAN 训练循环**:

```python
for epoch in range(num_epochs):
    for real_imgs, _ in dataloader:
        batch = real_imgs.size(0)
        real_imgs = real_imgs.view(batch, -1)
        
        # === 步骤 1: 训练 D (固定 G) ===
        opt_D.zero_grad()
        # 真实图标签 1
        d_real = D(real_imgs)
        loss_real = criterion(d_real, torch.ones_like(d_real))
        # 生成图标签 0
        z = torch.randn(batch, 100)
        fake_imgs = G(z).detach()  # detach! 不训 G
        d_fake = D(fake_imgs)
        loss_fake = criterion(d_fake, torch.zeros_like(d_fake))
        loss_D = loss_real + loss_fake
        loss_D.backward()
        opt_D.step()
        
        # === 步骤 2: 训练 G (固定 D) ===
        opt_G.zero_grad()
        z = torch.randn(batch, 100)
        fake_imgs = G(z)
        d_fake = D(fake_imgs)
        # G 希望 D 给假图高分 (标签 1)
        loss_G = criterion(d_fake, torch.ones_like(d_fake))
        loss_G.backward()
        opt_G.step()
```
- **What it demonstrates**: GAN 训练 = D 与 G 交替,关键点 `detach()` 防止训 G 时影响 D

**WGAN-GP 判别器(梯度惩罚)**:

```python
def gradient_penalty(D, real, fake):
    """WGAN-GP 梯度惩罚: 让 D 在插值点上 1-Lipschitz"""
    alpha = torch.rand(real.size(0), 1)
    alpha = alpha.expand_as(real)
    interpolated = alpha * real + (1 - alpha) * fake
    interpolated.requires_grad_(True)
    d_interp = D(interpolated)
    # 计算梯度
    grad = torch.autograd.grad(
        outputs=d_interp, inputs=interpolated,
        grad_outputs=torch.ones_like(d_interp),
        create_graph=True
    )[0]
    grad_norm = grad.view(grad.size(0), -1).norm(2, dim=1)
    return ((grad_norm - 1) ** 2).mean()  # 惩罚偏离 1

# WGAN-GP D 损失
loss_D = D(fake).mean() - D(real).mean() + 10 * gradient_penalty(D, real, fake)
# WGAN-GP G 损失
loss_G = -D(G(z)).mean()  # G 希望 D 给假图高分
```

**Cycle GAN 循环一致性**:

```python
# 两个生成器: G_xy (X->Y), G_yx (Y->X)
# 两个判别器: D_x, D_y

def cycle_loss(x, G_xy, G_yx):
    """x -> G_xy -> y' -> G_yx -> x' 应与 x 接近"""
    y_fake = G_xy(x)
    x_recon = G_yx(y_fake)
    return torch.mean(torch.abs(x - x_recon))  # L1

# 总损失
loss_G_xy = adversarial_loss(D_y(G_xy(x)), True)  # 骗 D_y
        + lambda_cycle * cycle_loss(x, G_xy, G_yx)
        + lambda_cycle * cycle_loss(y, G_yx, G_xy)  # 反向
```

**GAN 评估指标对照**:

| 指标 | 衡量 | 计算 | 优劣 |
|---|---|---|---|
| Inception Score (IS) | 质量+多样性 | Inception 网络输出分布 | 越大越好 |
| FID | 真实 vs 生成分布距离 | Inception 特征空间 Fréchet 距离 | 越小越好 |

## Worked Example

**为什么需要生成模型而非监督学习?** -- 视频预测的"两面讨好"问题。

设任务: 给小精灵游戏前几帧,预测下一帧。角色在转角可能左转或右转,数据集中两种情况都有。

**监督学习**:
- 输入: 转角前几帧
- 真实标签 1: 角色左转下一帧
- 真实标签 2: 角色右转下一帧
- 损失: MSE(y_pred, y_left) + MSE(y_pred, y_right)
- 最优解: `y_pred = (y_left + y_right) / 2` -- **平均,模糊**
- 现象: 角色变成残影、消失

**生成模型**:
- 输入: 转角前几帧 x + 噪声 z
- z ~ 二项分布(0/1 各 50%)
- z=1: G 学会输出左转下一帧
- z=0: G 学会输出右转下一帧
- 结果: **清晰的左转或右转**(具体看 z)

**关键洞察**:
1. **多模态输出**: 同一输入有多种合理输出时,监督学习取平均(模糊),生成模型用 z 索引不同模态
2. **"创造力"来自 z**: z 是随机性来源,让模型不固定输出
3. **不需要标签**: GAN 用真实数据本身做对比,自监督学习
4. **训练目标不是"答对"**: 而是"让分布接近"--P_G ≈ P_data
5. **GAN 评估难**: 损失不反映质量,需用 FID 等

**WGAN 改进**:
- 原 GAN: JS 散度在 P_G/P_data 不重叠时恒为 log 2,损失看不出进步
- WGAN: Wasserstein 距离 = "推土距离",即使分布不重叠也能反映远近
- 训练曲线: WGAN 损失随训练下降,可监控;原 GAN 损失乱跳

## Key Takeaways

1. 生成模型输出分布而非单一值,用噪声 z 索引不同可能--适合"创造性"任务
2. GAN = 生成器 G + 判别器 D 对抗训练;G 骗 D,D 抓 G
3. GAN 目标是 min_G max_D V(D,G),V 的最大值与 JS 散度相关
4. JS 散度在不重叠分布时失效;**WGAN 用 Wasserstein 距离** + 1-Lipschitz 判别器解决
5. 条件型 GAN: G 看 x 输出,D 看 (x, y) 评分,才能让 G 不无视条件
6. Cycle GAN: 无成对数据时用循环一致性 `G_yx(G_xy(x)) ≈ x` 约束
7. GAN 评估用 IS/FID,不用损失;FID 是当前主流
8. GAN 难训: G/D 必须均衡、模式崩塌、模式丢失;序列生成尤难(argmax 不可微)
9. 生成模型家族: GAN(图像强)、VAE(简单但模糊)、流模型(精确但受限)、扩散模型(质量高但慢)
