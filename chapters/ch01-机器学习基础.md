# Chapter 01: 机器学习基础

## Core Idea
机器学习不是手工编写规则,而是**让机器找一个函数**——输入数据,输出预测。这个"找"的过程由数据驱动:定义一个带未知参数的函数(模型),用一个目标函数(损失)衡量参数好坏,通过优化算法在数据上反复更新参数。

## Frameworks Introduced

- **"机器学习 = 找函数" (LeeDL)**
  - When to use: 解释 ML 在做什么的最简心智模型
  - How: 把任务描述成 `y = f(x)`,f 由机器从数据中学出来。语音识别→声音到文字;图像识别→图片到类别;AlphaGo→棋盘到落子位置

- **三步框架 (LeeDL)**
  - When to use: 拆解任何一个 ML 任务的最小骨架
  - How: ① 写出带未知参数的函数 `y = f_θ(x)`(模型) ② 定义损失 `L(θ)`(评估参数好坏) ③ 优化:用梯度下降等算法找 `θ* = argmin L(θ)`

- **四组件模型 (d2l)**
  - When to use: 系统化检查 ML 问题时(漏了哪一块?)
  - How: 数据 → 模型 → 目标函数 → 优化算法。四者缺一不可,任一环节薄弱都会拖垮整体

- **训练循环 (d2l)**
  - When to use: 实际写训练代码时
  - How: ① 随机初始化参数 ② 取一批数据 ③ 根据损失梯度更新参数 ④ 重复②③直到满意。**这是所有深度学习训练的物理骨架**

## Key Concepts

- **模型 (model)**: 带未知参数的函数。如 `y = b + w·x₁`,x 是特征,y 是预测
- **参数 (parameter)**: 模型里要学的量,如权重 w 和偏置 b。可以想象成"旋钮"
- **特征 (feature)**: 输入 x 中已知的、来自数据的信息
- **标签 (label)**: 监督学习中的真实值 y,用于计算损失
- **损失 (loss)**: `L(θ)` 衡量一组参数的好坏,值越小越好
- **回归 (regression)**: 输出标量数值的任务(如预测明日 PM2.5)
- **分类 (classification)**: 输出从预设类别中选一个的任务(如垃圾邮件检测)
- **结构化学习 (structured learning)**: 输出有结构物体(图片、文章)的任务
- **监督学习 (supervised)**: 数据带标签
- **无监督学习 (unsupervised)**: 数据无标签,自己找结构(聚类、降维)
- **强化学习 (RL)**: 通过与环境互动获得奖励信号来学习
- **领域知识 (domain knowledge)**: 对问题本质的理解,用来初始化模型形式(如"今日观看次数与昨日相关")
- **用数据编程 (programming with data)**: 不写规则,而是定义带参数的程序并用数据确定参数

## Mental Models

- **把参数当旋钮**: 模型是一族程序,每个参数组合对应一个具体程序。学习=找最好旋钮位置
- **用三步框架体检任何 ML 项目**: 如果效果差,先问哪一步出了问题——模型假设错?损失设计错?优化没收敛?
- **任务类型决定输出形态**: 标量→回归;离散选项→分类;复杂结构→结构化学习。先确定任务类型再选模型

## Anti-patterns

- **跳过损失设计直接调模型**: 损失函数定义了"好"的标准,模型只是逼近这个标准的工具。标准错了,模型再好也跑偏
- **用领域知识过度约束模型**: 初始猜测(如 `y=b+w·x₁`)只是起点,要让数据说话——如果数据表明需要更复杂函数,就放宽模型形式
- **混淆参数与超参数**: 参数是学出来的(w,b);超参数是人为设定的(学习率、批量大小)。调参调的是超参数

## Worked Example

**LeeDL 的 YouTube 观看次数预测**——完整走一遍三步框架。

**步骤 1: 定义模型**(基于领域知识"今日与昨日观看次数相关"):
$$y = b + w \cdot x_1$$
- `x₁` = 前一天观看次数(已知特征)
- `y` = 今天观看次数预测值
- `w` = 权重,`b` = 偏置(未知参数,要从数据学)
- 初始猜测: `b=500, w=1` → `y = 500 + x₁`

**步骤 2: 定义损失**(用 MAE):
$$e_n = |\hat{y}_n - y_n|, \quad L = \frac{1}{N}\sum_n e_n$$
- 把 2017/1/1 的真实观看次数 4800 代入: 预测 `ŷ=5300`,真实 `y=4900`,误差 `e₁=400`
- 2017/1/2 真实值 5400 预测 1/3 的值,真实 7500,误差 `e₂=2100`
- 三年共 N=1095 天,求平均得 `L`。L 越小,这组参数越好

**步骤 3: 优化**(梯度下降,后续章节详解):
- 计算损失对参数的梯度 ∂L/∂w、∂L/∂b
- 沿梯度反方向小步更新参数: `w ← w - η·∂L/∂w`
- 重复直到损失不再下降

**关键洞察**: 这个一元线性模型太弱(只能拟合直线),后续会看到如何用分段线性曲线、神经网络扩展模型族,但**三步骨架始终不变**。

## Key Takeaways

1. ML 的本质是"用数据驱动地找一个函数",而非手工编码规则
2. 任何 ML 任务都可以拆成**模型 + 损失 + 优化**三步,或更细的**数据 + 模型 + 目标 + 优化**四组件
3. 训练就是"随机初始化 → 取数据 → 按梯度更新参数 → 重复"的物理循环
4. 任务类型按输出形态分类: 回归(标量)、分类(离散选项)、结构化(复杂结构)
5. 领域知识用于初始化模型形式,但要让数据来精修参数
6. 损失函数定义"好坏"的标准,模型只是逼近标准的工具——别本末倒置
