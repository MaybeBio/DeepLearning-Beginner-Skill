# Chapter 04: 多层感知机

## Core Idea
线性模型只能拟合仿射变换。**多层感知机 (MLP)** = 全连接层 + 非线性激活,通过堆叠隐藏层突破线性限制。**激活函数是不可省的非线性**--没有它,再多层等价于一层。通用近似定理保证单隐层足够表达任意函数,但实际中更深更易学。

## Frameworks Introduced

- **隐藏层 + 激活函数 (d2l)**
  - When to use: 数据中存在非线性交互(如图像像素间的关系)
  - How: `H = σ(XW⁽¹⁾ + b⁽¹⁾)`,然后 `O = HW⁽²⁾ + b⁽²⁾。σ` 是按元素非线性的关键。**没有 σ,多层塌缩成单层**

- **通用近似定理 (d2l)**
  - When to use: 理解神经网络表达能力的天花板
  - How: 单隐层 MLP(足够神经元 + 正确权重)可近似任意连续函数到任意精度。**但"能表达"≠"能学会"**--更深的网络往往更容易学

- **激活函数选择 (d2l)**
  - When to use: 选 ReLU/sigmoid/tanh 等
  - How: **默认 ReLU**(简单、不饱和、计算快);sigmoid 用于二分类输出层或门控;tanh 用于需要 [-1,1] 范围的场景

- **前向传播 vs 反向传播 (d2l)**
  - When to use: 理解训练时网络在做什么
  - How: 前向=按计算图算输出并缓存中间值;反向=从 loss 倒着用链式法则算每个参数的梯度。**反向需要前向缓存的中间值**

- **参数初始化 (d2l)**
  - When to use: 永远要初始化,不能全 0
  - How: Xavier (适用于 tanh):方差 `1/fan_in`;He (适用于 ReLU):方差 `2/fan_in`。**全 0 初始化会让所有神经元学同样的东西**--对称性破坏失败

- **激活函数的导数与梯度流 (LeeDL)**
  - When to use: 调试梯度消失/爆炸
  - How: sigmoid 导数最大 0.25(易消失);ReLU 导数 0 或 1(不饱和但可能有 dead ReLU);tanh 导数 0-1

## Key Concepts

- **多层感知机 (MLP)**: 由多个全连接层 + 激活函数堆叠的网络
- **隐藏层 (hidden layer)**: 输入与输出之间的层,产生"隐藏表示"
- **隐藏变量 H**: 隐藏层的输出,数据的中间表示
- **激活函数 (activation function) σ**: 按元素的非线性函数,打破线性
- **活性值 (activation)**: 激活函数的输出
- **ReLU**: `max(x, 0)`,默认激活函数
- **sigmoid**: `1/(1+e⁻ˣ)`,值域 (0,1),用于二分类输出
- **tanh**: `(eˣ-e⁻ˣ)/(eˣ+e⁻ˣ)`,值域 (-1,1)
- **前向传播 (forward)**: 输入 -> 逐层计算 -> 输出 + 缓存中间值
- **反向传播 (backward)**: loss -> 链式法则倒着算 -> 每个参数的梯度
- **梯度消失 (vanishing gradient)**: 深层网络梯度反向时连乘小数 -> 趋近 0
- **梯度爆炸 (exploding gradient)**: 连乘大数 -> 趋近无穷
- **Xavier 初始化**: 方差 `1/fan_in`,适合 tanh/sigmoid
- **He 初始化**: 方差 `2/fan_in`,适合 ReLU
- **dead ReLU**: 大幅更新后某神经元永远输出 0,不再激活

## Mental Models

- **激活函数是"开关"**: ReLU 是按元素开关--正过负停。多层 ReLU = 多段折线组合,可拟合任意函数
- **深度 > 宽度**: 同样参数量,深而窄比宽而浅更易学复杂函数(每层做一次"复合变换")
- **网络是计算图**: 前向是建图+缓存,反向是倒着求导。框架(autograd)替你管理这些缓存
- **初始化打破对称性**: 全 0 初始化时同层神经元梯度相同,永远学同样的特征--网络退化成一个神经元
- **梯度消失 = 长链连乘小数**: sigmoid 网络超过 5 层基本训不动,这就是 ReLU 兴起的原因

## Anti-patterns

- **多层线性不带激活**: `O = XW⁽¹⁾W⁽²⁾ + b` 等价于 `O = XW + b`,白加一层参数
- **全 0 初始化**: 所有神经元对称,梯度相同,只学到 1 个特征。**永远用随机初始化**
- **深层 + sigmoid**: 梯度消失几乎必然。深网默认 ReLU
- **学习率太大 + ReLU**: 大梯度把权重推到让某些神经元永远输出 0(dead ReLU)
- **混用激活函数层和输出层**: 输出层激活由任务决定(回归恒等、二分类 sigmoid、多分类 softmax);隐藏层默认 ReLU
- **算 sigmoid 导数时忘 `(1-σ)`**: sigmoid 导数是 `σ(1-σ)`,这是它容易饱和的根源

## Code Examples

**MLP 从零实现 (Fashion-MNIST 分类)**:

```python
import torch
from torch import nn

# 1. 模型: 784 -> 256 (ReLU) -> 10
num_inputs, num_outputs, num_hiddens = 784, 10, 256
W1 = nn.Parameter(torch.randn(num_inputs, num_hiddens) * 0.01)
b1 = nn.Parameter(torch.zeros(num_hiddens))
W2 = nn.Parameter(torch.randn(num_hiddens, num_outputs) * 0.01)
b2 = nn.Parameter(torch.zeros(num_outputs))

def relu(X): return torch.max(X, 0)

def net(X):
    X = X.reshape((-1, num_inputs))
    H = relu(X @ W1 + b1)
    return H @ W2 + b2

# 2. 损失 + 优化
loss = nn.CrossEntropyLoss()
trainer = torch.optim.SGD([W1, b1, W2, b2], lr=0.5)

# 3. 训练循环同 Ch03
```

**MLP 简洁实现**:

```python
net = nn.Sequential(
    nn.Flatten(),
    nn.Linear(784, 256),
    nn.ReLU(),
    nn.Linear(256, 10)
)
# 默认初始化已是 He-like,直接用
trainer = torch.optim.SGD(net.parameters(), lr=0.5)
loss = nn.CrossEntropyLoss()
```
- **What it demonstrates**: `nn.Sequential` 把层堆起来,ReLU 作为单独"层"插入;`nn.Linear` 包含参数 + 仿射变换

**自定义 He 初始化**:

```python
def init_weights(m):
    if type(m) == nn.Linear:
        nn.init.kaiming_normal_(m.weight, mode='fan_in', nonlinearity='relu')

net.apply(init_weights)  # 递归应用到所有 Linear 层
```

## Reference Tables

**激活函数对照**:

| 激活 | 公式 | 导数 | 范围 | 适用场景 |
|---|---|---|---|---|
| ReLU | `max(x, 0)` | 0 或 1 | [0, ∞) | 隐藏层默认 |
| sigmoid | `1/(1+e⁻ˣ)` | `σ(1-σ)` | (0, 1) | 二分类输出、门控 |
| tanh | `(eˣ-e⁻ˣ)/(eˣ+e⁻ˣ)` | `1-tanh²` | (-1, 1) | RNN 默认 |
| LeakyReLU | `max(0.01x, x)` | 0.01 或 1 | (-∞, ∞) | 防 dead ReLU |

**初始化方法对照**:

| 方法 | 方差 | 适用 | 公式 |
|---|---|---|---|
| Xavier | `1/fan_avg` | tanh/sigmoid | `U(-√(6/(fan_in+fan_out)), √(6/(fan_in+fan_out)))` |
| He | `2/fan_in` | ReLU | `N(0, √(2/fan_in))` |
| 全 0 | 0 | **永不可用** | - |

## Worked Example

**为什么"无激活函数的多层 = 单层"?** -- 数学推导。

设两层无激活的网络:
$$O = (XW^{(1)} + b^{(1)}) W^{(2)} + b^{(2)}$$

展开:
$$O = XW^{(1)}W^{(2)} + b^{(1)}W^{(2)} + b^{(2)}$$

令 `W = W⁽¹⁾W⁽²⁾`, `b = b⁽¹⁾W⁽²⁾ + b⁽²⁾`,则:
$$O = XW + b$$

**关键洞察**: 这是仿射变换的复合,仍是仿射变换。再多层也没用--参数空间反而变大但表达能力不变。**激活函数打破这个塌缩**:`σ(XW⁽¹⁾ + b⁽¹⁾)W⁽²⁾ + b⁽²⁾` 无法重写成 `XW + b` 的形式。

**这就是为什么激活函数是必须的**--不是"可选优化",而是"必须的非线性"。

## Key Takeaways

1. MLP = 全连接层 + 非线性激活;激活函数不可省,否则多层塌缩成单层
2. 隐藏层默认 ReLU;输出层激活由任务决定(回归恒等、二分类 sigmoid、多分类 softmax)
3. 通用近似定理保证表达能力,但深度让函数更易学
4. 前向建图缓存,反向链式求导;autograd 管理缓存
5. 全 0 初始化是死罪;Xavier 用于 tanh,He 用于 ReLU
6. sigmoid 深层会梯度消失;ReLU 不饱和但可能 dead
7. 工程上永远用 `nn.Sequential + nn.Linear + nn.ReLU` 的简洁写法
