# Chapter 05: 深度学习计算

## Core Idea
真实的神经网络不是单个层,而是**层、块、模型**的递归组合。PyTorch 用 `nn.Module` 统一表达这三者:任何子类只要实现 `__init__`(声明组件)和 `forward`(定义计算),就能像函数一样被调用、自动追踪参数、自动求导。这是构建任意复杂网络的工程基石。

## Frameworks Introduced

- **块 = Module = 可调用 + 有参数 + 可求导 (d2l)**
  - When to use: 构建任何超过单层的网络
  - How: 继承 `nn.Module`,在 `__init__` 里声明子层,在 `forward(X)` 里定义前向。`net(X)` 是 `net.__call__(X)` 的语法糖,会触发 forward + autograd 记录

- **Sequential 块 (d2l)**
  - When to use: 层按顺序串接的简单网络
  - How: `nn.Sequential(layer1, layer2, ...)`,自动按顺序把上一层的输出作为下一层的输入

- **自定义块 (d2l)**
  - When to use: 网络有分支、跳连、控制流等非顺序结构(如 ResNet)
  - How: 继承 `nn.Module`,实现 `__init__` 和 `forward`。**前向函数里可以写任意 Python 控制流(if、for、print、debug)**

- **参数访问与管理 (d2l)**
  - When to use: 调参、保存、迁移学习、参数共享
  - How: `net[i].weight` / `net.state_dict()` 访问;`nn.init.normal_(m.weight, ...)` 初始化;`load_state_dict` 加载

- **延后初始化 (d2l)**
  - When to use: 不知道输入维度时(如从配置文件构造网络)
  - How: PyTorch 的 `nn.LazyLinear` 等延迟到第一次前向时推断形状并初始化

- **保存与加载 (d2l)**
  - When to use: 训练断点续训、模型部署、迁移学习
  - How: 张量用 `torch.save/load`;模型用 `net.state_dict()` + `load_state_dict()`。**永远保存 state_dict,不保存整个模型对象**

- **GPU 计算 (d2l)**
  - When to use: 任何真实训练
  - How: `tensor.to('cuda')` 或 `tensor.cuda()`;模型 `net.to('cuda')`。**数据和模型必须在同一设备**--否则报错

## Key Concepts

- **块 (block)**: 抽象层级,可以是单层、层组合,或整个模型
- **层 (layer)**: 最小的可复用计算单元,如 `nn.Linear`、`nn.Conv2d`
- **nn.Module**: PyTorch 所有块/层的基类,提供参数管理、autograd 集成
- **forward 函数**: 定义前向计算,被 `__call__` 触发
- **state_dict**: 模型所有参数的有序字典,保存/加载的标准格式
- **参数初始化 (init)**: 通过 `nn.init.*` 函数设置参数值
- **参数共享 (parameter sharing)**: 多个层引用同一个 Module 实例,共享参数
- **延后初始化 (lazy init)**: 第一次前向时才确定形状并分配参数
- **自定义层 (custom layer)**: 继承 `nn.Module` 实现非标准计算
- **nn.Parameter**: 标记"这是可训练参数"的 Tensor 子类,自动注册到 Module
- **device**: 张量/模型所在的设备(`cpu` / `cuda:0`)

## Mental Models

- **块就是函数**: `net(X)` 看成函数调用,只不过这个函数有可学习的参数
- **递归组合**: 大模型 = 块的列表;块 = 层的列表;层 = 算子+参数。所有层级共用 `nn.Module` 接口
- **state_dict 是模型快照**: 序列化模型 = 序列化 state_dict。模型类本身由代码定义,不存盘
- **设备一致性**: tensor 和 net 必须在同一 GPU,否则跨设备运算报错
- **forward 是计算图定义**: 里面写的控制流(if/for)会被 autograd 追踪,影响反向传播路径

## Anti-patterns

- **在 forward 外做大计算**: forward 里只写"前向",反向需要的中间值由 autograd 自动缓存
- **直接保存整个模型 `torch.save(net)`**: pickle 绑定类定义路径,迁移时易坏。**保存 `net.state_dict()`**
- **数据和模型不同设备**: `X.cuda(); net` 仍在 CPU -> 报错。**统一用 `.to(device)`**
- **在 `__init__` 里创建非参数张量不包 `nn.Parameter`**: 不会被注册为可训练参数。**用 `nn.Parameter(torch.tensor(...))`**
- **忘记 `super().__init__()`**: 子类初始化失败,Module 内部状态没建好
- **在 forward 里修改参数**: 破坏计算图,导致内存泄漏或梯度错误
- **共享 Module 实例时不理解梯度累积**: 同一 Module 被多次调用,梯度会累加到同一组参数

## Code Examples

**自定义 MLP 块** -- `nn.Module` 的最小骨架:

```python
import torch
from torch import nn

class MLP(nn.Module):
    def __init__(self):
        super().__init__()
        self.hidden = nn.Linear(20, 256)
        self.out = nn.Linear(256, 10)
    
    def forward(self, X):
        return self.out(torch.relu(self.hidden(X)))

net = MLP()
print(net(X))          # 调用 = net.__call__(X) = net.forward(X)
print(net.state_dict())  # 查看所有参数
```
- **What it demonstrates**: `__init__` 声明子层(自动注册参数);`forward` 定义计算;`net(X)` 触发调用

**带控制流的自定义块** -- forward 里可以写任意 Python:

```python
class FixedHiddenMLP(nn.Module):
    def __init__(self):
        super().__init__()
        self.rand_weight = torch.rand((20, 20), requires_grad=False)  # 常量
        self.linear = nn.Linear(20, 20)
    
    def forward(self, X):
        X = self.linear(X)
        X = torch.mm(X, self.rand_weight) + 1
        X = torch.relu(X)
        while X.abs().sum() > 1:    # while 循环!
            X = X / 2
        return X.sum()              # 标量输出
```
- **What it demonstrates**: `requires_grad=False` 的张量当常量;`while` 控制流被 autograd 正确处理

**参数初始化** -- 递归 apply:

```python
def init_normal(m):
    if type(m) == nn.Linear:
        nn.init.normal_(m.weight, mean=0, std=0.01)
        nn.init.zeros_(m.bias)

net = nn.Sequential(nn.Linear(20, 256), nn.ReLU(), nn.Linear(256, 10))
net.apply(init_normal)  # 递归应用到所有 Linear
```

**保存与加载模型**:

```python
# 保存
torch.save(net.state_dict(), 'mlp.params')

# 加载
clone = nn.Sequential(nn.Linear(20, 256), nn.ReLU(), nn.Linear(256, 10))
clone.load_state_dict(torch.load('mlp.params'))
clone.eval()  # 推理模式,关闭 dropout/BN 更新
```

**GPU 使用**:

```python
device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
X = X.to(device)
net = net.to(device)
y = net(X)  # 全在 GPU 上
```

## Reference Tables

**nn.Module 关键方法**:

| 方法 | 用途 |
|---|---|
| `__call__(X)` | 触发 forward + autograd |
| `parameters()` | 迭代所有可训练参数 |
| `state_dict()` | 返回参数字典 |
| `load_state_dict(d)` | 从字典加载参数 |
| `apply(fn)` | 递归对子模块应用 fn |
| `to(device)` | 移动到设备 |
| `eval()` / `train()` | 切换推理/训练模式 |
| `add_module(name, m)` | 注册子模块 |

**保存/加载对照**:

| 对象 | 保存 | 加载 |
|---|---|---|
| 单个张量 | `torch.save(t, 't.pt')` | `t = torch.load('t.pt')` |
| 张量字典 | `torch.save(d, 'd.pt')` | `d = torch.load('d.pt')` |
| 模型参数 | `torch.save(net.state_dict(), 'm.params')` | `net.load_state_dict(torch.load(...))` |
| 整个模型 | **避免**: `torch.save(net)` | `net = torch.load(...)`(易坏) |

## Worked Example

**实现一个嵌套块** -- 展示块的递归组合:

```python
class NestMLP(nn.Module):
    def __init__(self):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(20, 64), nn.ReLU(),
            nn.Linear(64, 32), nn.ReLU()
        )
        self.linear = nn.Linear(32, 16)
    
    def forward(self, X):
        return self.linear(self.net(X))

# 组合两个块
chimera = nn.Sequential(NestMLP(), nn.Linear(16, 20), FixedHiddenMLP())
chimera(X)
```

**关键洞察**:
1. `NestMLP` 内部用 `nn.Sequential` 组合更小的层
2. `chimera` 把 `NestMLP` 当成一个块,与 `nn.Linear`、`FixedHiddenMLP` 串接
3. 所有块共用 `nn.Module` 接口,可以无限嵌套
4. **这就是 ResNet-152 的构造方式**:几十个残差块组合,每个块内部又含若干卷积层

## Key Takeaways

1. `nn.Module` 是 PyTorch 表达层/块/模型的统一抽象
2. 自定义块只需 `__init__` 声明组件 + `forward` 定义计算
3. `forward` 里可以写任意 Python 控制流,autograd 会正确追踪
4. 永远保存 `state_dict()`,不要保存整个模型对象
5. 数据和模型必须在同一设备;用 `.to(device)` 统一管理
6. `nn.init.*` + `apply` 是参数初始化的标准方式
7. 复杂架构(ResNet、Transformer)的本质就是块的递归组合
