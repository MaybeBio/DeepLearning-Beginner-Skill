# Chapter 07: 优化算法

## Core Idea
深度学习优化的真实难点不是"卡在局部最小值",而是**梯度震荡、学习率难调、参数间尺度不一**。三个核心改进: ① **小批量**引入噪声帮助逃离鞍点+泛化更好 ② **动量**累加历史梯度,惯性越过平缓区 ③ **自适应学习率**(AdaGrad/RMSProp/Adam)给每个参数不同学习率,大梯度小步、小梯度大步。

## Frameworks Introduced

- **优化的真实挑战 (LeeDL + d2l)**
  - When to use: 理解为什么训练常"卡住"
  - How: 不是只有局部最小值/鞍点。更常见的是**梯度在山谷两壁来回震荡,损失不降但梯度仍大**。这需要学习率调整,而非"逃出临界点"

- **批量大小选择 (LeeDL)**
  - When to use: 训练任何模型时
  - How:
    - **大批量**:一个回合快、梯度稳;但易陷"峡谷最小值"(sharp minima),泛化差
    - **小批量**:一个回合慢、梯度噪;但噪声帮逃离峡谷、陷"平原最小值"(flat minima),泛化好
    - 实践:32-256 是常见甜区;超大批量(>32K)需配合 LARS/LAMB 等特殊优化器

- **平原 vs 峡谷最小值 (LeeDL)**
  - When to use: 理解为什么小批量泛化好
  - How: 训练损失和测试损失有微小偏移。峡谷最小值训练好但偏移后损失暴涨;平原最小值两边都稳。**小批量噪声大,会从峡谷跳出,停在平原**

- **动量法 (LeeDL + d2l)**
  - When to use: 默认开启,几乎所有现代优化器都含
  - How: `m_t = λ·m_{t-1} - η·g_t`,然后 `θ_{t+1} = θ_t + m_t`。**动量 = 过去所有梯度的指数加权和**。物理直觉:球带惯性,能越过局部小丘

- **自适应学习率 (LeeDL + d2l)**
  - When to use: 不同参数梯度尺度差异大时(几乎所有深网)
  - How: 给每个参数独立学习率 `η/σ_t^i`,σ_i 是该参数历史梯度的均方根
    - **AdaGrad**: `σ_t = √(Σ₀ᵗ g²)/(t+1)`--累积所有历史,单调下降,后期学习率过小
    - **RMSProp**: `σ_t = √(α·σ_{t-1}² + (1-α)·g_t²)`--指数移动平均,能跟上梯度变化
    - **Adam**: RMSProp + 动量,默认选择

- **Adam 优化器 (d2l)**
  - When to use: 大多数深网的默认起点
  - How: `m_t = β₁m_{t-1} + (1-β₁)g_t`(动量) + `v_t = β₂v_{t-1} + (1-β₂)g_t²`(RMSProp) + 偏差修正 + `θ -= η·m̂/(√v̂ + ε)`。默认 β₁=0.9, β₂=0.999, ε=1e-8

- **学习率调度 (LeeDL + d2l)**
  - When to use: 训练中后期,接近收敛时
  - How:
    - **warmup**: 初期小学习率预热(防止初期不稳)
    - **cosine decay**: 学习率按余弦曲线降到 0
    - **step decay**: 每 N epoch 乘 0.1
    - **reduce-on-plateau**: 验证损失停滞时降学习率

## Key Concepts

- **临界点 (critical point)**: 梯度为 0 的点(局部最小值、局部最大值、鞍点)
- **鞍点 (saddle point)**: 某方向极小、另方向极大的临界点;高维下比局部最小值常见得多
- **海森矩阵 (Hessian)**: 二阶导数矩阵;特征值全正->局部最小;全负->局部最大;有正有负->鞍点
- **批量 (batch)**: 一次前向反向用的样本集
- **批量大小 (batch size)**: 一个批量的样本数
- **epoch (回合)**: 完整扫过一次训练集
- **BGD / SGD / mini-batch SGD**: 全批量 / 单样本 / 小批量梯度下降
- **动量 (momentum) m_t**: 历史梯度的指数加权和
- **自适应学习率**: 每个参数独立学习率
- **AdaGrad**: 累积所有历史梯度平方的开方做分母
- **RMSProp**: 指数移动平均梯度平方
- **Adam**: 动量 + RMSProp + 偏差修正
- **学习率预热 (warmup)**: 初期用小学习率逐步增加
- **学习率衰减 (decay)**: 后期降学习率精细收敛
- **sharp minima / flat minima**: 峡谷/平原最小值;平原更鲁棒

## Mental Models

- **"卡住"≠临界点**: 训练损失不降时,梯度往往仍很大--是震荡而非卡住
- **梯度是方向,学习率是步长**: 方向不对步长再大也白搭;步长不对方向再准也跑偏
- **小批量噪声是免费正则**: 噪声帮你逃出 sharp minima,跑到 flat minima,泛化更好
- **动量是低通滤波器**: 累加多步梯度,平均掉震荡分量,留下稳定方向
- **Adam = 方向滤波(动量) + 步长自适应(RMSProp)**: 大多数场景的默认选择
- **学习率是唯一最重要的超参**: 调参 80% 时间应花在学习率(初始值、调度、warmup)

## Anti-patterns

- **永远用默认学习率**: 不同模型/数据需要不同 lr;从 0.001 开始看 loss 曲线调
- **大批量 + 不调学习率**: 大批量需要线性增大 lr;否则一个回合慢、效果差
- **训练 loss 不降就加层**: 先检查是不是学习率/优化问题(参见 Ch 06)
- **Adam 不调 β₂**: 0.999 适合大多数,但 RNN 等梯度稀疏场景可能要 0.98 或更小
- **不warmup 直接大 lr 训 Transformer**: 初期梯度大,易发散。**Transformer 必须 warmup**
- **只看 loss 曲线判断收敛**: 看 **梯度范数**--loss 不降但梯度大 ≠ 卡住,可能是震荡
- **Adam 用了就忘 L2**: AdamW 是带正确权重衰减的 Adam,比 Adam + L2 更稳

## Code Examples

**SGD / Momentum / Adam 对比**:

```python
import torch
import torch.optim as optim

# 1. 普通 SGD (很少用)
optimizer = optim.SGD(net.parameters(), lr=0.01)

# 2. SGD + Momentum
optimizer = optim.SGD(net.parameters(), lr=0.01, momentum=0.9)

# 3. Adam (默认选择)
optimizer = optim.Adam(net.parameters(), lr=1e-3, betas=(0.9, 0.999))

# 4. AdamW (Transformer 推荐)
optimizer = optim.AdamW(net.parameters(), lr=1e-3, weight_decay=0.01)

# 训练循环(通用)
for epoch in range(num_epochs):
    for X, y in data_iter:
        loss = criterion(net(X), y)
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
```

**Cosine 学习率调度 + warmup**:

```python
from torch.optim.lr_scheduler import CosineAnnealingLR, LinearLR

optimizer = optim.AdamW(net.parameters(), lr=1e-3)
warmup = LinearLR(optimizer, start_factor=0.1, total_iters=5)   # 5 epoch warmup
cosine = CosineAnnealingLR(optimizer, T_max=num_epochs - 5)     # 余弦衰减
scheduler = optim.lr_scheduler.SequentialLR(optimizer, [warmup, cosine], milestones=[5])

for epoch in range(num_epochs):
    for X, y in data_iter:
        # ... 训练 ...
        pass
    scheduler.step()
```

**Adam 的状态检查**:

```python
# 看每个参数的动量和方差
for name, p in net.named_parameters():
    state = optimizer.state[p]
    if 'exp_avg' in state:  # 动量 m
        print(f"{name}: |m|={state['exp_avg'].norm():.4f}, |v|={state['exp_avg_sq'].norm():.4f}")
```

## Reference Tables

**优化器选择决策**:

| 场景 | 推荐 | 学习率 |
|---|---|---|
| 入门/CV 小数据 | SGD+momentum | 0.01-0.1 |
| NLP/Transformer | AdamW | 1e-4 ~ 5e-5 + warmup |
| 通用默认 | Adam | 1e-3 |
| 大批量训练 | LARS/LAMB | 线性增大 lr |
| RNN | Adam(可能调 β₂) | 1e-3 |
| GAN | Adam(β₁=0.5) | 1e-4 ~ 2e-4 |

**批量大小对照 (LeeDL 表 3.1)**:

| 维度 | 小批量 | 大批量 |
|---|---|---|
| 一次更新时间(并行) | 相同 | 相同(批量不太大时) |
| 一个回合时间 | 更慢 | 更快 |
| 梯度 | 有噪声 | 稳定 |
| 优化 | 更好 | 更差 |
| 泛化 | 更好 | 更差 |

**学习率调度策略对照**:

| 策略 | 公式 | 适用 |
|---|---|---|
| Step decay | 每 N epoch ×0.1 | CV 经典 |
| Cosine | `η_t = η_max·(1+cos(πt/T))/2` | 现代默认 |
| Warmup + Cosine | 前 N 步线性升,后余弦降 | Transformer |
| Exponential | `η_t = η_0·γ^t` | 不推荐 |
| Reduce-on-plateau | 验证 loss 停滞时降 | 不稳的实验 |

## Worked Example

**为什么 AdaGrad 后期会"卡死"?** -- 推导 σ_t 的行为。

AdaGrad 的累积分母:
$$\sigma_t^i = \sqrt{\frac{1}{t+1} \sum_{j=0}^{t} (g_j^i)^2}$$

**问题**: σ_t 单调累积所有历史梯度平方,在稠密参数上会**单调增长** -> 学习率 `η/σ_t` 单调下降 -> 后期几乎不更新。

**例子**: 训练 100k 步,某参数梯度一直 ~0.1。则:
- t=10: σ ≈ √((10·0.01)/11) ≈ 0.095,有效 lr ≈ η/0.095 ≈ 10.5η
- t=100k: σ ≈ √((100k·0.01)/100k) ≈ 0.1,有效 lr ≈ η/0.1 = 10η(稳)

**看起来还行?** 问题在于不同参数:
- 稠密参数(梯度频繁):σ 大 -> lr 小,合理
- 稀疏参数(梯度罕见):σ 小 -> lr 大,看起来合理;但偶尔来一次大梯度,被 σ 平方累积放大,后续 lr 永远小

**RMSProp 的修正**:
$$\sigma_t = \sqrt{\alpha \cdot \sigma_{t-1}^2 + (1-\alpha) \cdot g_t^2}$$

用 EMA 替代累积,遗忘老梯度,**学习率能跟上当前梯度尺度**。这就是 RMSProp/Adam 几乎取代 AdaGrad 的原因。

## Key Takeaways

1. 训练"卡住"常是震荡而非临界点;看梯度范数而非只看 loss
2. 小批量噪声是免费正则--帮助逃出 sharp minima,跑到 flat minima
3. 动量 = 历史梯度的指数加权,起低通滤波作用
4. 自适应学习率给每个参数不同 lr,RMSProp 用 EMA 解决 AdaGrad 单调衰减问题
5. Adam = 动量 + RMSProp + 偏差修正,是 NLP/通用默认
6. AdamW 用于 Transformer,带正确的权重衰减
7. 学习率是最重要超参;Transformer 必须 warmup;cosine decay 是现代默认
