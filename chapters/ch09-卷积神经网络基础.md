# Chapter 09: 卷积神经网络基础

## Core Idea
全连接处理图像参数爆炸。CNN 通过**两个观察 -> 两个简化**专门为图像设计: ① **观察 1: 检测模式不需要整张图** -> 感受野(receptive field),每个神经元只看局部 ② **观察 2: 同样的模式会出现在不同位置** -> 参数共享,不同感受野的神经元共用权重。这两个简化合起来就是**卷积层**。再加汇聚层下采样,LeNet 拼出第一个完整 CNN。

## Frameworks Introduced

- **感受野 (Receptive Field) (LeeDL)**
  - When to use: 处理高维输入(图像、音频)时减少参数
  - How: 每个神经元只看输入的一个局部区域(如 3×3)。**核大小 (kernel size)** = 感受野的高×宽;**深度** = 输入通道数(默认全通道)。典型核大小 3×3

- **步幅与填充 (Stride & Padding) (LeeDL + d2l)**
  - When to use: 控制输出空间尺寸
  - How: **步幅** = 感受野每次移动的格数(典型 1 或 2);**填充** = 边界补 0(zero padding),让边界模式也能被检测。**步幅 > 1 时感受野必须重叠**,否则跨感受野的模式会丢失

- **参数共享 (Parameter Sharing) (LeeDL)**
  - When to use: 同一模式可能出现在任何位置
  - How: 不同感受野的神经元共用同一组权重(滤波器 filter)。第 1 个感受野的第 i 个神经元和第 2 个感受野的第 i 个神经元参数相同。**参数共享 + 感受野 = 卷积层**

- **卷积运算 = 互相关 (d2l)**
  - When to use: 实际计算卷积层
  - How: 卷积核在输入上滑动,每位置做按元素乘再求和,得到输出的一格。**严格说这是互相关(cross-correlation),不是数学卷积**--但 DL 习惯叫卷积

- **多输入多输出通道 (d2l)**
  - When to use: RGB 图像(3 通道输入)、多特征图输出
  - How: 输入有 c_in 通道时,卷积核也是 c_in 通道(深度匹配),按通道卷积后相加得 1 个输出通道;要 c_out 个输出通道就用 c_out 个不同的核,每个产出 1 个输出通道。**输出通道数 = 该层滤波器数量**

- **1×1 卷积 (d2l)**
  - When to use: 调整通道数(降维/升维)、跨通道信息融合
  - How: 核大小 1×1,等效于在每个像素位置做全连接。**不改变空间尺寸,只改变通道数**

- **汇聚层 (Pooling) (d2l)**
  - When to use: 下采样空间尺寸,降低计算量+获得平移不变性
  - How: 在每个通道上独立做最大值(max pooling)或平均值(avg pooling)。**无参数**;通常 kernel=2, stride=2,尺寸减半

- **LeNet (d2l)**
  - When to use: 理解 CNN 的最小完整例子
  - How: Conv(5×5,6) -> Pool -> Conv(5×5,16) -> Pool -> FC(120) -> FC(84) -> FC(10)。**conv 提取特征 + FC 做分类**的经典骨架

## Key Concepts

- **图像 = 三维张量**: H × W × C(高×宽×通道)
- **感受野 (receptive field)**: 神经元看到的局部区域
- **核大小 (kernel size)**: 感受野的空间尺寸(如 3×3)
- **步幅 (stride)**: 感受野每次移动的格数
- **填充 (padding)**: 边界补 0(常用)或其他值
- **滤波器 (filter) / 卷积核 (kernel)**: 一组共享的权重
- **特征图 (feature map)**: 卷积层的输出
- **通道 (channel)**: 特征图的深度维度
- **多输入通道**: 输入的通道数(如 RGB=3)
- **多输出通道**: 该层滤波器数量
- **1×1 卷积**: 跨通道融合,改变通道数
- **汇聚 (pooling)**: 空间下采样,无参数
- **最大汇聚 (max pool)**: 取窗口最大值
- **平均汇聚 (avg pool)**: 取窗口平均值
- **平移不变性 (translation invariance)**: 模式出现在哪都能被检测
- **局部性 (locality)**: 模式只在局部范围内相关

## Mental Models

- **CNN 是"先验注入"**: 全连接什么都能学但易过拟合;CNN 通过感受野+参数共享注入"图像模式是局部的 + 平移不变"的先验,虽灵活性低但对图像更有效
- **滤波器 = 模式检测器**: 每个滤波器学一个特定模式(边缘、纹理、部件),输出通道的每个值表示"这个模式在这里有多强"
- **空间尺寸不断减小,通道数不断增加**: 典型 CNN 把 H×W 慢慢缩小,把 C 慢慢增大--空间信息压缩为语义信息
- **1×1 卷积是"通道维的全连接"**: 不混合空间信息,只混合通道信息
- **Pooling 是"无参数降采样"**: 与卷积不同,没有可学参数,纯算子

## Anti-patterns

- **核大小设太大 (如 11×11) 在深网络**: 现代 CNN 偏好堆叠多个 3×3,参数量更少、表达力更强
- **步幅设太大不重叠**: 跨感受野的模式会丢失。**步幅 ≤ 核大小的一半**通常安全
- **不填充导致边界信息丢失**: 边界模式无人检测。**用 same padding**(输出尺寸 = 输入尺寸)
- **混淆卷积核深度与输入通道**: 核的深度 = 输入通道数,核的数量 = 输出通道数
- **Pool 跨通道**: pooling 是按通道独立做的,不要跨通道 pool
- **Pool 后接 Pool 无卷积**: 信息损失过大,中间应有卷积提取特征
- **第一层用 1×1 卷积**: 第一层应该混合空间信息,用 3×3 或更大;1×1 用于中间层调整通道

## Code Examples

**卷积层基本使用**:

```python
import torch
from torch import nn

# 1. 基本卷积层: 输入 3 通道, 输出 16 通道, 3x3 核, padding=1 保持尺寸
conv = nn.Conv2d(in_channels=3, out_channels=16, kernel_size=3, 
                 stride=1, padding=1)
X = torch.randn(1, 3, 32, 32)  # (batch, C, H, W)
print(conv(X).shape)  # torch.Size([1, 16, 32, 32])

# 2. 步幅 2 减半尺寸
conv2 = nn.Conv2d(16, 32, kernel_size=3, stride=2, padding=1)
print(conv2(conv(X)).shape)  # torch.Size([1, 32, 16, 16])

# 3. 1x1 卷积改通道数
conv1x1 = nn.Conv2d(32, 8, kernel_size=1)
print(conv1x1(conv2(conv(X))).shape)  # torch.Size([1, 8, 16, 16])
```

**汇聚层**:

```python
# 最大池化: kernel=2, stride=2 -> 尺寸减半
pool = nn.MaxPool2d(kernel_size=2, stride=2)
X = torch.randn(1, 16, 32, 32)
print(pool(X).shape)  # torch.Size([1, 16, 16, 16])

# 自适应池化: 输出固定尺寸(用于全局平均池化)
gap = nn.AdaptiveAvgPool2d((1, 1))
print(gap(X).shape)  # torch.Size([1, 16, 1, 1]) - 全局平均池化
```

**LeNet 实现**:

```python
net = nn.Sequential(
    nn.Conv2d(1, 6, kernel_size=5, padding=2), nn.Sigmoid(),
    nn.AvgPool2d(kernel_size=2, stride=2),
    nn.Conv2d(6, 16, kernel_size=5), nn.Sigmoid(),
    nn.AvgPool2d(kernel_size=2, stride=2),
    nn.Flatten(),
    nn.Linear(16*5*5, 120), nn.Sigmoid(),
    nn.Linear(120, 84), nn.Sigmoid(),
    nn.Linear(84, 10)
)
```
- **What it demonstrates**: 经典 CNN 骨架: Conv-Pool-Conv-Pool-FC-FC-FC。**注意**: LeNet 用 sigmoid(年代早),现代 CNN 用 ReLU

## Reference Tables

**输出尺寸公式**:

```
输出 H = ⌊(H_in + 2*padding - kernel_size) / stride⌋ + 1
输出 W = ⌊(W_in + 2*padding - kernel_size) / stride⌋ + 1
```

**典型尺寸变化(kernel=3, stride=1, pad=1 + pool k=2 s=2)**:

| 层 | 输入 | 输出 |
|---|---|---|
| Conv(k=3,s=1,p=1) | 3×32×32 | 16×32×32 (尺寸不变,通道变) |
| MaxPool(k=2,s=2) | 16×32×32 | 16×16×16 (尺寸减半) |
| Conv(k=3,s=1,p=1) | 16×16×16 | 32×16×16 |
| MaxPool(k=2,s=2) | 32×16×16 | 32×8×8 |

**层类型对照**:

| 层 | 有参数? | 作用 |
|---|---|---|
| Conv2d | 是 (k×k× Cin× Cout) | 提取空间特征 |
| 1×1 Conv | 是 (Cin× Cout) | 跨通道融合 |
| MaxPool2d | 否 | 空间下采样 |
| AdaptiveAvgPool2d | 否 | 输出固定尺寸 |
| BatchNorm2d | 是 (γ, β) | 标准化中间层 |
| Linear | 是 | 全连接分类 |

## Worked Example

**为什么 CNN 比全连接参数少这么多?** -- 算一笔账。

设 100×100×3 的图像,第一层 1000 个神经元:

**全连接**: 每个神经元看全部输入
- 参数 = 100×100×3 × 1000 = **3×10⁷**(3000 万)

**CNN(3×3 感受野, 16 个滤波器)**:
- 每个滤波器参数 = 3×3×3 = 27
- 16 个滤波器 = 27×16 = **432**

**参数比**: 3000 万 / 432 ≈ **7 万倍**!

**关键洞察**:
1. 感受野让每个神经元只看 3×3×3 而非整张图--参数从 30000 降到 27
2. 参数共享让所有位置共用同一组滤波器--只增加 16 个滤波器就够
3. CNN 的"模型偏差"(灵活性低)对图像反而**更好**--因为图像确实满足局部性+平移不变性
4. 这就是 LeeDL 强调的: **正确的先验 = 更少参数 + 更好泛化**

## Key Takeaways

1. CNN 由两个观察导出: 局部性(感受野) + 平移不变性(参数共享)
2. 卷积层 = 感受野 + 参数共享;滤波器 = 共享权重的模式检测器
3. 输出尺寸 = ⌊(H + 2p - k)/s⌋ + 1;掌握这个公式能预测任何架构
4. 多输出通道 = 多个滤波器;1×1 卷积只混合通道不混合空间
5. Pooling 无参数下采样;现代 CNN 末端用全局平均池化(GAP)
6. LeNet 是最小 CNN 骨架: Conv-Pool-Conv-Pool-FC-FC-FC
7. CNN 不是"通用更强的网络",而是"对图像先验正确的网络"--换领域要重新思考先验
