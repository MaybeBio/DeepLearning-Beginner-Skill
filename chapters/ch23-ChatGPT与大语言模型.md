# Chapter 23: ChatGPT 与大语言模型

## Core Idea
**ChatGPT** 本质是一个**做文字接龙的大型语言模型 (LLM)**--输入前文,输出下一词的概率分布,采样后拼回输入继续。**三个常见误解**: ① 不是罐头回复(每次随机采样);② 不是网络搜索(训练时联网,测试时不联);③ 不是简单函数(含 1700 亿+参数)。ChatGPT 训练**三步骤**: ① **预训练(自监督)**: 用海量网络文本学文字接龙,得"基石模型";② **监督微调**: 人类老师给正确输入输出对;③ **强化学习 (RLHF + PPO)**: 人类给反馈(好/坏)而非答案,适合人类也不知答案的任务(如写诗)。**关键技术**: 预训练让模型学到跨语言、跨任务通用能力--教英文阅读理解,中文阅读也能裸考(78% vs 直接中文训 78%)。ChatGPT 带来新研究: 提示工程、神经编辑(改错不破坏)、AI 生成检测、机器反学习(遗忘隐私)。

## Frameworks Introduced

- **ChatGPT = 文字接龙函数 (LeeDL)**
  - When to use: 理解 ChatGPT 本质
  - How:
    - 输入: 当前问题 + 历史对话
    - 输出: 下一词的概率分布
    - 采样: 按概率随机选词(每次答案不同)
    - 自回归: 选出的词拼回输入,继续预测下一词
    - 结束: 采样到结束符号停止
  - 关键: 1700 亿+参数的复杂函数,非罐头、非搜索

- **训练 vs 测试 (LeeDL)**
  - When to use: 澄清"网络搜索"误解
  - How:
    - **训练**: 可联网爬数据学文字接龙(像备考看书)
    - **测试**: 不联网,凭记忆生成(像考场不能翻书)
  - 关键: 训练用网络,测试不用--ChatGPT 答案是"想出来的"

- **ChatGPT 三步骤 (LeeDL)**
  - When to use: 训练 ChatGPT
  - How:
    1. **预训练(自监督)**: 网络文本做文字接龙,无人工标注,得基石模型
    2. **监督微调**: 人类老师给输入输出对(如"中国第一高峰?" -> "珠穆朗玛峰")
    3. **强化学习(RLHF)**: 人类给反馈(好/坏),PPO 算法优化
  - 关键: 预训练提供海量知识,微调对齐任务,RLHF 对齐人类偏好

- **预训练/自监督/基石模型 (LeeDL)**
  - When to use: 理解大模型基础
  - How:
    - **预训练**: 大量无标注数据学通用表示
    - **自监督学习**: 数据自身造标签(前半句 -> 后半句)
    - **基石模型 (foundation model)**: 预训练得模型,可微调到各种任务
  - 三者指同一事物的不同角度
  - 关键: 网络每段文字都可是训练数据,无痛生成无穷成对数据

- **GPT 系列演进 (LeeDL)**
  - When to use: 理解规模扩张
  - How:
    - **GPT-1 (2018)**: 1.17 亿参数,1GB 数据
    - **GPT-2 (2019)**: 15 亿参数(10×),40GB 数据(40×),能回答问题、摘要
    - **GPT-3 (2020)**: 1750 亿参数(100×),570GB 数据(读哈利波特 30 万遍),能写代码
    - **GPT-3.5**: GPT-3 微调版,定义不明确
    - **ChatGPT (2022)**: GPT-3.5 + 监督微调 + RLHF
  - 关键: 规模扩张带来涌现能力

- **多语言零样本迁移 (LeeDL)**
  - When to use: 理解预训练的跨语言能力
  - How:
    - Multi-BERT 在 104 种语言上预训练
    - 英文阅读理解微调 -> 中文阅读裸考 78%(直接中文训也 78%)
    - 中文预训练 + 中文微调 -> 89%
  - 关键: 模型把所有语言内化为同一种"自己的语言"

- **RLHF 强化学习人类反馈 (LeeDL)**
  - When to use: 人类难给答案但能判好坏时
  - How:
    - 监督学习: 人类给答案(辛苦)
    - RLHF: 人类给反馈(好/坏,省力)
    - 用 PPO 算法优化策略
  - **适用场景**:
    - 人类偷懒,反馈比答案易收集
    - 人类不知答案(如"写诗赞美 AI")--能判断好坏但写不出来
  - 关键: RLHF 让模型对齐人类偏好,不只是模仿答案

- **提示工程 Prompting (LeeDL)**
  - When to use: 让 ChatGPT 按期望行事
  - How:
    - "催眠"模型: "请想象你是我的朋友"
    - 指定语言: "用中文回答"
    - 鼓励互动: "请试着跟我聊聊,反问我问题"
    - 启动: "现在我们开始"
  - 关键: 精准提示 = 调教模型角色和行为

- **ChatGPT 新研究方向 (LeeDL)**
  - When to use: 探索 LLM 未来
  - How:
    - **提示工程**: 系统化自动找催眠指令
    - **神经编辑 (neural editing)**: 改一个错不破坏其他答案
    - **AI 生成检测**: 训分类器区分 AI/人类内容
    - **机器反学习 (machine unlearning)**: 让模型遗忘隐私数据
  - 关键: ChatGPT 带来新研究问题,而非终结 NLP

## Key Concepts

- **ChatGPT**: OpenAI 2022 发布的对话模型
- **文字接龙 (text continuation)**: 预测下一词

- **大语言模型 (LLM)**: 大规模语言模型
- **预训练 (pre-training)**: 自监督学通用表示

- **基石模型 (foundation model)**: 预训练得模型
- **微调 (fine-tuning)**: 任务适配
- **RLHF**: 强化学习人类反馈
- **PPO**: Proximal Policy Optimization,RLHF 算法
- **GPT-1/2/3/3.5**: GPT 系列演进
- **多语言迁移**: 教一语言会其他语言
- **提示工程 (prompting)**: 设计输入引导模型
- **催眠 (hypnosis)**: 角色扮演式提示
- **神经编辑 (neural editing)**: 修改模型特定知识
- **AI 生成检测**: 判断内容是否 AI 生成
- **机器反学习 (machine unlearning)**: 让模型遗忘
- **涌现能力 (emergent ability)**: 规模到一定程度出现新能力
- **DRCD**: 台湾中文阅读理解语料库
- **Multi-BERT**: 104 语言预训练 BERT

## Mental Models

- **ChatGPT = 文字接龙**: 输入前文,输出下一词概率,采样拼接
- **训练 = 备考**: 可联网爬数据;**测试 = 考场**: 不联网凭记忆
- **预训练 = 学常识**: 海量数据学语言规律;**微调 = 学专业**: 任务数据学具体技能
- **RLHF = 老师改作业**: 不给标准答案,只判好坏
- **多语言 = 内化为同一种语言**: 模型不分中英文,只分语义
- **规模 = 涌现**: 10×、100× 规模带来新能力(写代码、推理)
- **提示 = 催眠**: 设定角色让模型按要求行事
- **改错 = 神经编辑**: 改一处不破坏其他(难,因黑盒)
- **反学习 = 选择性遗忘**: 删隐私不删能力
- **基石模型 = 一切基础**: 微调出各种应用

## Anti-patterns

- **以为 ChatGPT 是罐头回复**: 每次随机采样,答案不同。**理解文字接龙**
- **以为 ChatGPT 联网搜索**: 训练联网,测试不联。**答案是生成的**
- **只监督学习训 ChatGPT**: 数据有限,能力受限。**必须预训练 + RLHF**
- **预训练用小数据**: 学不到通用能力。**GPT-3 用 570GB**
- **RLHF 人类给答案**: 辛苦且难收集。**给反馈即可**
- **不写提示直接问**: 模型不知角色。**用"请想象你是..."催眠**
- **改错直接训新答案**: 可能破坏其他知识。**用神经编辑**
- **期望 ChatGPT 全对**: 可能幻觉,2021 后知识有限。**核实答案**
- **绕弯问隐私**: 角色扮演可骗模型泄露。**需机器反学习**
- **禁用 ChatGPT**: 是工具,应学会用。**类比计算器、搜索引擎**

## Code Examples

**GPT 文字接龙(简化)**:

```python
import torch
import torch.nn.functional as F

def generate_text(model, prompt, max_new_tokens=100, temperature=1.0):
    """GPT 文字接龙生成"""
    tokens = tokenize(prompt)
    for _ in range(max_new_tokens):
        with torch.no_grad():
            logits = model(torch.tensor([tokens]))  # (1, L, vocab)
        
        # 取最后一个位置的 logits
        next_logits = logits[0, -1] / temperature  # 温度控制随机性
        
        # 采样(非取最大,保留随机性)
        probs = F.softmax(next_logits, dim=-1)
        next_token = torch.multinomial(probs, 1).item()
        
        if next_token == EOS_TOKEN:
            break
        tokens.append(next_token)
    
    return detokenize(tokens)

# 温度 T 的影响
# T < 1: 更确定(集中高概率词)
# T = 1: 原分布
# T > 1: 更随机(分布平滑)
```

## Worked Example

**为什么 ChatGPT 能多语言?** -- Multi-BERT 零样本迁移案例。

**实验**(DRCD 中文阅读理解):
- 任务: 读中文文章,答中文问题
- 训练方式对比:
  1. **无预训练 + 中文微调**: 78%
  2. **中文预训练 + 中文微调**: 89%
  3. **104 语言预训练 + 英文微调**: 78%(中文裸考)

**神奇现象**:
- 方式 3 只教英文阅读理解,中文阅读裸考得 78%
- 与方式 1(直接中文训)相当
- **教一种语言会其他语言**

**解释**:
- 104 语言预训练让模型把所有语言内化为"自己的语言"
- 模型不分中英文,只分语义
- 英文阅读理解能力 = 通用阅读理解能力,可迁移到中文

**ChatGPT 多语言原理**:
- 预训练时见过多语言
- 教一种语言的任务,其他语言同任务也会
- 不需要翻译引擎

**关键洞察**:
1. **预训练 = 无痛数据**: 网络每段文字都可是训练数据
2. **规模 = 涌现**: GPT-3 1750 亿参数才写代码,GPT-2 15 亿不能
3. **RLHF 必要性**: 监督学习对齐答案,RLHF 对齐偏好(更友好、安全)
4. **提示工程**: 同模型,不同提示,行为差异巨大
5. **黑盒难题**: 改一个错可能破坏其他(神经编辑挑战)
6. **隐私风险**: 角色扮演可绕过安全限制泄露隐私
7. **工具论**: ChatGPT 像计算器、搜索引擎,应学会用而非禁用

**幻觉问题**:
- ChatGPT 训练数据截止 2021,2022 后知识有限
- 问"2022 世界杯冠军": 拒答(人类老师教过"2022"敏感)
- 问"最近世界杯冠军": 答 2018 法国(停留在 2021 前)
- **神经编辑需求**: 改这一处不破坏其他世界杯知识

## Key Takeaways

1. ChatGPT 本质是**文字接龙函数**: 输入前文,输出下一词概率,采样自回归
2. 三个误解: 非罐头(随机采样)、非搜索(测试不联网)、非简单(1700 亿+参数)
3. 训练三步骤: **预训练(自监督)+ 监督微调 + RLHF(PPO)**
4. **预训练 = 自监督 = 基石模型**: 同一事物不同角度,海量数据学通用表示
5. GPT 系列规模扩张: GPT-1(1.17 亿) -> GPT-3(1750 亿),规模带来涌现
6. **多语言零样本迁移**: 104 语言预训练,教英文任务,中文同任务也会
7. **RLHF 优势**: 人类给反馈而非答案,省力且适合人类不知答案的任务
8. **提示工程**: "催眠"模型设定角色,精准提需求
9. ChatGPT 带来新研究: 提示工程、神经编辑、AI 检测、机器反学习
10. **神经编辑挑战**: 改一处错不破坏其他(黑盒模型难)
11. **机器反学习**: 选择性遗忘隐私数据(角色扮演可绕过安全)
12. ChatGPT 是工具,应学会用而非禁用,提升人类整体水平
