# Chapter 06: 实践方法论

## Core Idea
训练效果差时不要瞎调参,按决策树定位问题:**先看训练损失**--大则是模型偏差或优化问题;**再看测试损失**--训练小但测试大则是过拟合;**若加数据没用**则是不匹配。这套方法论让你不靠玄学调试,而是用证据定位故障层。

## Frameworks Introduced

- **LeeDL 训练故障决策树**
  - When to use: 任何模型训练效果差时
  - How:
    1. **训练损失大** -> 是模型偏差(模型太简单,函数族里没好解)还是优化问题(梯度下降没找到好解)?
    2. **训练小、测试大** -> 过拟合(模型灵活性过大,记住了训练集 peculiarities)
    3. **训练小、测试大 + 加数据没用** -> 不匹配(训练/测试分布不同)

- **如何区分模型偏差 vs 优化问题 (LeeDL)**
  - When to use: 训练损失大时
  - How: **跑一个更深的模型**。若深模型损失 >= 浅模型 -> 优化问题(深模型灵活性更大却没压得更低);若深模型损失更低 -> 浅模型是模型偏差,该变大

- **过拟合的两种解法 (LeeDL)**
  - When to use: 训练损失小但测试损失大
  - How: ① **增加数据**(最有效,包括数据增强) ② **限制模型**(更少参数、参数共享、早停、正则化、Dropout、CNN 等结构性先验)
  - 关键: **限制要刚好**--太少 -> 模型偏差;太多 -> 欠拟合

- **k 折交叉验证 (LeeDL + d2l)**
  - When to use: 数据有限,选模型/超参时
  - How: 把训练集切 k 份,每次拿 1 份当验证集、其余 k-1 份训练,重复 k 次取平均。**用验证集选模型,不要用公开测试集选**--会在测试集上过拟合

- **训练误差 vs 泛化误差 (d2l)**
  - When to use: 理解模型质量的两个维度
  - How: 训练误差衡量"学会了吗";泛化误差衡量"换数据还行吗"。**好的模型是泛化误差低**,而非训练误差低

- **不匹配 vs 过拟合 (LeeDL)**
  - When to use: 加数据后测试损失没降
  - How: 不匹配是训练/测试分布不同(如 2020 训练、2021 测试,而 2021 数据特性变了)。**加数据无效**,要靠对数据生成过程的理解--重新设计训练集、用领域自适应等

- **模型复杂度的双刃剑 (LeeDL)**
  - When to use: 选择模型容量时
  - How: 复杂度 -> 0 时欠拟合(模型偏差);复杂度 -> ∞ 时过拟合。**最优复杂度在中间**--训练损失可降,测试损失先降后升,U 形曲线

## Key Concepts

- **模型偏差 (model bias)**: 模型函数族太小,根本不包含好解--"海里没针"
- **优化问题 (optimization issue)**: 函数族里有好解,但梯度下降没找到--"针在海里但捞不到"
- **过拟合 (overfitting)**: 训练损失低、测试损失高;模型记住了训练数据的噪声/peculiarities
- **欠拟合 (underfitting)**: 训练损失就高;模型太简单或优化不到位
- **不匹配 (mismatch)**: 训练/测试分布不同;加数据无效
- **训练误差 (training error)**: 在训练集上的损失
- **泛化误差 (generalization error)**: 在新数据上的期望损失
- **验证集 (validation set)**: 从训练集划出来选模型/超参的部分
- **k 折交叉验证 (k-fold CV)**: 数据切 k 份轮流当验证集,平均结果选模型
- **数据增强 (data augmentation)**: 用领域知识造新数据(如图像左右翻转)
- **早停 (early stopping)**: 监控验证损失,开始上升就停
- **正则化 (regularization)**: 限制参数大小/复杂度(L2、L1)
- **Dropout**: 训练时随机丢神经元,等效于 ensemble + 正则化
- **结构性先验**: 用架构本身限制模型(如 CNN 比全连接限制更大,但对图像更有效)

## Mental Models

- **大海捞针三态**: ① 海里没针(模型偏差) ② 针在海里但捞不到(优化问题) ③ 针捞到了但放到别的水里失效(过拟合/不匹配)
- **过拟合 = 自由度 > 数据约束**: 3 个点拟 100 次多项式必然过拟合,因为自由度远超约束
- **不匹配不是过拟合**: 过拟合能靠加数据救,不匹配加数据没用--这是核心区别
- **CNN 是"先验"**: 全连接灵活但易过拟合;CNN 通过卷积限制灵活性,反而对图像更好--**正确的限制 = 领域知识的注入**
- **不要用测试集调参**: 用测试集选模型=把测试集变成验证集,真正测试时会失败
- **复杂度 U 形**: 训练损失随复杂度单调降;测试损失 U 形。最优在 U 底

## Anti-patterns

- **训练损失大就加层**: 可能是优化问题不是模型偏差。**先跑深模型验证**
- **测试损失大就加数据**: 可能是不匹配。**先判断训练/测试是否同分布**
- **用测试集选模型**: Kaggle 公开榜单前几名经常私人榜暴跌,因为他们在公开测试集上过拟合
- **限制过度**: 模型太简单 -> 模型偏差又来了。限制要正好
- **瞎做数据增强**: 上下颠倒图像可能造出不真实数据,反而有害。**增强要符合数据分布**
- **不切验证集**: 用训练损失当模型选择信号 -> 必然过拟合
- **k 折太小**: k=2 时验证集太大、训练数据少;k=10 是常见折中

## Reference Tables

**LeeDL 训练故障定位表**:

| 现象 | 诊断 | 解法 |
|---|---|---|
| 训练损失大 + 深模型没压更低 | 优化问题 | 改进优化(更好初始化、Adam、学习率调度、BN) |
| 训练损失大 + 深模型能压低 | 模型偏差 | 增大模型(更多层/参数、更多特征) |
| 训练损失小 + 测试损失大 + 加数据有效 | 过拟合 | 增加数据、限制模型(正则/Dropout/早停/CNN先验) |
| 训练损失小 + 测试损失大 + 加数据无效 | 不匹配 | 重设计训练集、领域自适应、理解数据生成过程 |

**模型限制手段对照**:

| 手段 | 机制 | 何时用 |
|---|---|---|
| 减少参数/神经元 | 直接降容量 | 过拟合明显 |
| 参数共享 (CNN) | 结构先验 | 数据有空间结构 |
| 早停 | 训练时间正则 | 验证损失开始上升 |
| L2 正则 (权重衰减) | 限制参数大小 | 通用,默认开 |
| Dropout | 训练时随机失活 | 全连接层 |
| 数据增强 | 等效扩数据 | 图像/音频 |

## Worked Example

**LeeDL 5 层网络诊断优化问题** -- 用深模型反推故障层。

观察 1-5 层网络的训练损失:
| 网络 | 1 层 | 2 层 | 3 层 | 4 层 | 5 层 |
|---|---|---|---|---|---|
| 训练损失 | 280 | 180 | 140 | 100 | **340** |

**分析**:
- 1→4 层损失递降,说明更深 = 灵活性更大 = 能压更低损失
- 5 层损失突然飙到 340,远高于 4 层的 100

**诊断**: 5 层网络灵活性肯定 >= 4 层(函数族包含 4 层能表达的),但它没找到好解。**这是优化问题**,不是模型偏差。

**关键洞察**:
1. 判断模型偏差 vs 优化的方法: **比较深浅模型**--深模型若没压更低,则是优化问题
2. 深网络的优化更难(梯度消失、鞍点),所以"更深更好"只在优化能跟上的前提下成立
3. 解法不是"加层",而是改进优化(参见 Ch 07): 更好的初始化、BN、残差连接、Adam 等

## Key Takeaways

1. 训练效果差先看训练损失,再看测试损失,按决策树定位故障
2. 训练损失大时,用深浅模型对比判断是模型偏差还是优化问题
3. 过拟合靠加数据/限制模型;不匹配靠理解数据分布,加数据无效
4. 用验证集(或 k 折)选模型,绝不用测试集调参
5. 模型复杂度 U 形--最优在训练损失可降、测试损失最低的中庸点
6. 限制模型 = 注入领域先验;CNN 限制灵活性但对图像更好
7. 数据增强要符合数据分布,瞎造数据反而有害
