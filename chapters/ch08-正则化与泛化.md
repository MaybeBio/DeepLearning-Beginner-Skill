# Chapter 08: 正则化与泛化

## Core Idea
深网有极大灵活性,甚至能记住随机标签--所以必须主动**限制模型**让它泛化。三种主流正则化: ① **权重衰减 (L2)** 拉小参数范数,平滑函数 ② **Dropout** 训练时随机失活,等效 ensemble + 注入噪声 ③ **批量归一化 (BN)** 把每层输入标准化,平滑误差表面。三者机制不同,常组合使用。

## Frameworks Introduced

- **权重衰减 / L2 正则 (d2l)**
  - When to use: 默认开启(尤其在全连接层、CV 任务)
  - How: 损失加 `(λ/2)‖w‖²`,λ 是正则强度。梯度下降更新变为 `w ← (1-ηλ)w - η·∇L`(每步先把 w 衰减一点)。**偏置通常不正则**
  - 直觉: 大权重 = 复杂函数;限制权重范数 -> 偏好分散用所有特征而非依赖少数

- **L2 vs L1 (d2l)**
  - When to use: 选正则类型
  - How: L2 惩罚大分量、偏好权重均匀分布;L1 产生稀疏权重(特征选择)。**DL 默认 L2**

- **Dropout (d2l)**
  - When to use: 全连接层正则化(CNN/RNN/Transformer 用得更精细)
  - How: 训练时每层以概率 p 把神经元置 0,**剩余的乘以 1/(1-p)** 保持期望不变。测试时不 dropout,直接用全部神经元。直觉: 防止"共适应性",强迫每个神经元都能独立工作

- **Dropout 的概率论解释 (d2l)**
  - When to use: 理解为什么 dropout 有效
  - How: Bishop 1995 证明"对输入加噪声训练"等价于 Tikhonov 正则化。Dropout 把这个想法推广到每层--注入噪声 = 要求函数对扰动鲁棒 = 平滑函数

- **批量归一化 BN (LeeDL)**
  - When to use: 深网训练的标准组件(尤其 CV)
  - How: 在每层激活前对 z 做 `(z - μ_batch) / σ_batch`,然后用可学习参数 γ, β 做仿射变换 `γ·ẑ + β`。**μ, σ 用当前 batch 的统计量**(所以叫"批量"归一化)
  - 直觉: 把每层输入标准化到 0 均值 1 方差,**平滑误差表面**(让不同方向梯度尺度一致),允许更大学习率

- **BN 的训练/测试差异 (LeeDL)**
  - When to use: 部署 BN 模型时
  - How: 训练用 batch 统计量;测试用训练时的**移动平均** μ, σ。所以 `model.eval()` 必须调,否则 BN 行为错

- **特征归一化 (LeeDL)**
  - When to use: 输入特征尺度差异大时(必做)
  - How: 对每个特征维度算 `(x - μ_i) / σ_i`(Z-score)。**输入归一化是 BN 的简化版**,只对原始输入做

## Key Concepts

- **权重衰减 (weight decay)**: L2 正则,损失加 `(λ/2)‖w‖²`
- **正则化强度 λ**: 超参,λ 大 -> 限制强;通常 1e-4 ~ 1e-2
- **L1 正则**: 损失加 `λ‖w‖₁`,产生稀疏
- **岭回归 / 套索回归**: L2/L1 正则的线性回归别称
- **Dropout 概率 p**: 失活概率,典型 0.1-0.5
- **共适应性 (co-adaptation)**: 神经元间过度依赖,Dropout 破坏之
- **批量归一化 (BN)**: 每层输入用 batch 统计量标准化
- **可学习仿射 γ, β**: BN 后的缩放和平移,恢复表达能力
- **训练/推理模式**: `model.train()` 用 batch 统计;`model.eval()` 用移动平均
- **移动平均 (moving average)**: 测试时用的 μ, σ 估计
- **特征归一化 (Z-score)**: 输入特征的标准化 `(x-μ)/σ`
- **内部协变量偏移 (internal covariate shift)**: BN 论文提出的概念(后证非真正原因),指训练中深层输入分布变化
- **早停 (early stopping)**: 验证损失开始上升时停止训练
- **数据增强 (data augmentation)**: 通过变换扩数据(已在 Ch06 介绍)

## Mental Models

- **大权重 = 复杂函数**: L2 把权重往 0 拉,等效偏好简单函数
- **Dropout = 训练时做 ensemble**: 每次dropout 配置等效一个子网络;测试时用全部=平均所有子网络
- **Dropout 缩放保持期望**: `1/(1-p)` 是关键--不缩放会导致测试时数值偏小
- **BN 不是为了"内部协变量偏移"**: 真正原因是**平滑误差表面,改善优化地形**,让梯度更稳、能用更大 lr
- **BN 把"批量"变成计算单元**: batch 内样本变得相关(因为共享 μ, σ),所以 batch 不能太小
- **正则化是"限制"不是"惩罚"**: 目的是让模型学到鲁棒特征,不是单纯降低参数

## Anti-patterns

- **Dropout 用了不调 `model.eval()`**: 推理时还会 dropout,结果不稳。**推理必须 eval()**
- **Dropout 概率太高 (如 0.8)**: 训练信号过弱,模型学不到东西
- **BN batch 太小 (< 8)**: μ, σ 估计噪声大,训练不稳
- **BN 用了不调 `model.eval()`**: 推理用 batch 统计,不同 batch 结果不同--致命
- **权重衰减加到偏置**: 偏置不正则,否则欠拟合。`weight_decay` 默认只对 weight
- **L1 用作"通用正则"**: L1 主要用于特征选择,通用正则用 L2
- **Dropout 用在 CNN 输出层附近**: CNN 一般用 spatial dropout 或 BN,普通 dropout 在卷积层效果差
- **Transformer 用 dropout 替代 attention dropout**: Transformer 多处 dropout(embedding、attention、residual),不能简化为一种

## Code Examples

**Dropout 层实现**:

```python
import torch
from torch import nn

# 从零实现
def dropout_layer(X, dropout):
    assert 0 <= dropout <= 1
    if dropout == 1: return torch.zeros_like(X)
    if dropout == 0: return X
    mask = (torch.rand(X.shape) > dropout).float()
    return mask * X / (1.0 - dropout)  # 关键: 缩放保持期望

# nn 内置
drop = nn.Dropout(0.5)
X = torch.randn(5, 5)
print(drop(X))  # 训练时约一半 0,其余乘以 2

# 切换训练/推理模式
net.train()  # 启用 dropout
net.eval()   # 关闭 dropout(直接返回 X)
```

```python
# 1. 通过 optimizer 的 weight_decay
optimizer = torch.optim.AdamW(net.parameters(), lr=1e-3, weight_decay=0.01)
```

## Worked Example

**为什么 Dropout 要乘 `1/(1-p)`?** -- 推导期望保持不变。

设原始激活 h,Dropout 后 h':
- 以概率 p:h' = 0
- 以概率 (1-p):h' = ?

要求 `E[h'] = h`,即 `p·0 + (1-p)·h' = h`,解得 `h' = h/(1-p)`。

**所以 Dropout 实现是**:
```python
mask = (rand < 1-p)  # 保留比例 1-p
h' = mask * h / (1-p)  # 缩放
```

**测试时为什么不 dropout?**
- 训练时已经把保留神经元放大 1/(1-p),所以训练时 E[h'] = h
- 测试时直接用全部神经元,期望就是 h--与训练期望一致
- 若测试时还 dropout,会引入随机性,预测不稳

**关键洞察**: Dropout 的 `1/(1-p)` 是"反向 dropout"或"inverted dropout"的标准实现,让测试时无需任何调整--这种实现细节是工程化的关键。

## Key Takeaways

1. 深网灵活性极大,不主动正则就会过拟合
2. 权重衰减(L2)是默认正则;AdamW 是正确实现
3. Dropout 训练失活+缩放,测试时关闭;`model.eval()` 必须调
4. BN 标准化中间层,平滑误差表面,允许更大学习率--是深网训练的基石
5. BN 训练用 batch 统计、推理用移动平均--必须切换模式
6. 正则化是"限制"而非"惩罚",目的是泛化而非压参数
7. 多种正则化常组合使用:AdamW(weight decay) + BN + Dropout
